Using Explorer

In Windows 95, the Program Manager and File Manager from Windows 3.1 combined into a new animal called Explorer. Explorer could almost be your Desktop, really, because anything you might want to do to a file or program, and any configuration changes you want to make to your operating environment, can be done right from Explorer.

NOTE:  Actually, Explorer is the shell for Windows 2000 as well as a separate utility for browsing local resources on your computer.

In Windows 98, Explorer added some active content that uses HyperText Markup Language (HTML) files to describe the contents of individual folders. This helped to decrease the separation of your computer from the Internet. Windows 2000 has carried on that idea, but Microsoft changed the emphasis in the GUI away from the Internet and more toward ease of use. You can enable the same functionality that Windows 98 includes, but the default configuration of Windows 2000 has more separation between your computer and the Internet.

Opening vs. Exploring

One thing you’ll notice right away when you use Explorer is a difference between opening a Desktop object and exploring a Desktop object. Figure 3.1 shows the difference. Opening a Desktop object simply opens a window that shows only the contents of the object or the folder. When you open the program called My Computer, for example, you’ll see its contents, but nothing else: no toolbar, no double panes. However, if you explore the same program (that is, if you right-click the My Computer icon, either on the Desktop or in a file list where it appears) and choose Explore from the menu that pops up, the window that opens proudly displays a toolbar full of useful buttons and a right pane for displaying the contents of whatever you select in the left pane. These are the telltale features of Explorer. If you already know where you want to go, that is, the location of the file or object you want to access, Open is the way to go. But, as the name suggests, Explorer functions best when you want to “explore” the computer for the file or application you want to use.

Branches

It’s easy to think of Windows Explorer as the filing system of Windows 2000. It may be more accurate, though, to describe it as a window to the filing system of Windows 2000. The views in Windows Explorer depict an upside down tree with its root being My Computer. It branches down from there showing the various connected drives and other resources. The simplest way to access this view is to right-click My Computer on the Desktop and select Explore from the context menu. When you open Explorer, you’ll notice two different panes: On the left is a view of the tree structure of your system; on the right is a group of icons and names representing all of the main branches or resources of your computer. The following illustration shows this initial view.

Unlike NT 4’s Explorer, when you open the Windows Explorer from the Start menu in Windows 2000, you will see the My Documents folder highlighted in the left pane. This is the default starting location for browsing the resources attached to your local computer. When you open Windows Explorer by right-clicking My Computer and choosing Explore, you will see that My Computer is highlighted in the left pane.

TIP:  Each item that appears in Windows Explorer is considered an object by Windows 2000. Every object has its own properties, many of which you can modify to suit your needs. This is the same regardless of whether you’re working with a file, folder, program, or a network computer. Although Windows Explorer is often thought of as a method to view or locate objects, you could just as easily think of it as the glue that holds this diverse environment together.
