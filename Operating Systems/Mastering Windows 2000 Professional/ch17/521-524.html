Security

Security is one of Windows 2000’s A-number-1 raisons d’être. NT was designed to fulfill the needs of people requiring high levels of security, and NT is one of the only microcomputer operating systems that has been reviewed for U.S. government’s C2 security standard. Windows 2000 also incorporates this deep-seated need for security.

Here are some of the ways in which Windows 2000 implements security:

•  Windows 2000 requires users to log on to a workstation.
•  Windows 2000 defines user rights that control a user’s actions.
•  Windows 2000 uses file and printer permissions to control access.
•  Windows 2000 encrypts security information over networks.

Windows 2000 Requires Users to Log On to a Workstation

Windows 9x (for comparison against Windows 2000) has a login screen that appears when you turn it on, but the login procedure is not really secure, for a couple of reasons. First, you can bypass it by pressing Esc. Second, you can always boot a Windows 9x computer with a floppy and run the regedit.exe program to export the Registry, a set of files that controls Windows 9x’s configuration.

Windows 2000, in contrast, doesn’t let you do anything on your computer until you log on, identifying yourself by name, password, and whether you want to be acknowledged by a local Windows 2000 domain. (Windows 2000 domains are methods of centrally storing user account information.)

Windows 2000 Defines User Rights That Control a User’s Actions

Have you ever loaned your machine to someone, only to get it back with all your settings changed? Or, worse yet, to find it loaded with a new piece of software that messed up your .dll files or program groups?

The good news is that Windows 2000 divides the range of things that you can do with a computer into user rights. The User Manager enables you to restrict what someone can do on your system. Giving someone else an account on your system doesn’t necessarily mean giving them the keys to your system; it may mean giving them access to only a subset of your computer’s capabilities.

The bad news is that you don’t have the same fine control of user rights in Windows 2000 Professional that you do in Server. The good news is that Windows 2000 enables you to define a local user account that can use the computer and run common programs, but cannot install new programs. Members of the Users group can run programs, and save data to their own folders, but they can’t install programs or modify system files.

Windows 2000 Uses File and Printer Permissions to Control Access

Like its bigger brother Windows 2000 Server, Windows 2000 Professional lets you point to any given directory and remove a user’s access to that directory. Believe it or not, you can create a local account on a Windows 2000 machine that, at the same time that it lets someone get on your computer and run a particular program, also prevents that user from doing anything else.

When you examine a file or directory with Explorer, you can right-click a directory or file and one of the buttons you’ll see is Permissions.

Printers, as well as directories and files, have permissions associated with them. You can prevent specific people from using your locally attached printers, and of course, domain network administrators can control which network printers those people can access.

Windows 2000 Encrypts Security Information over Networks

As we depend on networks more and more, security becomes increasingly important. For example, getting onto the World Wide Web and buying flowers, clothes, books, software, and airline tickets is becoming routine. Well, I hope I’m not the first to tell you this, but every transaction that you make over the Web involves transmitting secure information: passwords, account numbers, maybe credit-card numbers. These are things that should remain confidential. Unfortunately, network applications called sniffers can record and display every byte that passes by on the Net.

NOTE:  Windows 2000 Server includes a sniffer called Network Monitor. Windows 2000 Professional does not include this program. But this version is limited to traffic to and from your computer. The full version of Network Monitor, which enables you to see everything that passes over the physical network, is available only in Microsoft System Management Server (SMS).

What keeps someone from sniffing the Net and stealing private information? Well, actually, nothing; so the cybershoppers had better encrypt the data before they send it. A number of encrypting protocols are available, and Windows 2000 includes one of its own. With encryption, if you happen to be sniffing the network while someone logs on, you’ll see that someone has logged on but you will not be able to snatch their password or other secure information.

NOTE:  Please note that this security applies only to NT or Windows 2000 and Microsoft networking; Internet communications such as FTP or Web browsing are not secure unless you’re using a specific Internet security protocol, such as Netscape’s Secure Sockets Layer (SSL). Never assume that your Internet communications are secure, even when you use a Windows 2000 workstation.

Windows 2000 Offers a Fault-Tolerant File System

Part of security is knowing that your data is secure. Both Windows 2000 Server and Windows 2000 Professional support a high-performance file system called the New Technology File System (NTFS) that includes the ability to recover from many events (some power failures or software crashes, for example) that would destroy or damage data on DOS machines.

NOTE:  Fault tolerance is available only on Windows 2000 Servers, but you can take it a step further and install Digital NT Clusters, a technology that uses two NT Server machines to act like one machine. If one of the servers fails, the other one picks up quickly. Microsoft has also developed their own version of clustering, implemented in Windows 2000 Advanced Server Edition and Windows 2000 Data Center Server.

Windows 2000 Will Optionally Log All Security Events

Want to keep track of everything that happens on your computer? It’s possible. Windows 2000 includes a Security log that will, at your option, log many events, including the following:

•  Who logged on to your machine, when the person logged on, and when he or she logged off
•  What files were used
•  When others attempted and failed to log on to your system
•  Whether the person who logged on to your system changed any user rights or permissions (if he or she had administrative privileges)

All that information goes in the Security log, one of the three logs you can find in the Event Viewer.
