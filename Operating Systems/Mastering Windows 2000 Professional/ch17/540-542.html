Make Sure You Have the Hardware

I’ve already said that you should buy hardware from the HCL whenever possible, so I’m not going to repeat myself. But something that’s just as important is to buy enough horsepower.

All of Windows 2000’s powers come at a price: speed. Run Windows 9x and Windows 2000 on the same system, and you can’t help but notice the difference in performance between the two. Here are a few suggestions about how to spend your money to assemble a decent Windows 2000 machine.

Processor

If you’re definitely committed to Windows 2000, then consider buying Pentium II- or Pentium III-based systems, rather than Pentium systems. Recall that the Pentium Pro through Pentium III chips are especially tuned to run 32-bit programs particularly well, and they offer that performance without the great cost of RISC-based systems.

No matter what processor you buy, get the most speed you can afford. I’m not going to tell you not to run Windows 2000 on anything below 300MHz. But if all other things are equal, more megahertz means better performance.

NOTE:  Early versions of Windows NT ran on a 386 processor, but Microsoft stopped supporting NT on a 386 as of version 4. Windows 2000 supports the Pentium as the minimum processor requirement.

Memory

I’ve run Windows 2000 on a 333MHz computer and been disappointed by its sluggish performance. I’ve also run it on a 133MHz server and been satisfied with its performance.

The difference? The 333MHz computer had 32MB of RAM. The server had 64MB of RAM.

I haven’t actually found the point at which Windows 2000 stops thirsting for memory. For example, the computer on my desktop contains 128MB of RAM, and even it shuffles things from memory onto disk now and then.

Thankfully, memory prices have dropped (and risen, then dropped again) and today there aren’t many reasons not to have enough memory. And while I still can’t say what enough memory is for Windows 2000, I can say that today most Windows 2000 users are running on computers with at least 128MB of RAM. Just check out the catalog from your favorite supplier and you will notice that most of the current systems are coming equipped with 128MB of RAM.

NOTE:  You can’t just put any arbitrary amount of RAM into a computer; the memory sockets in most Pentiums allow only a few memory configurations: 8MB, 16MB, 24MB, 32MB, 40MB, 48MB, 64MB, 72MB, 80MB, 96MB, or 128MB. Newer motherboards support much higher amounts of RAM, but still have specific configurations.

As the price falls, people will say, “What the heck, as long as I’m upgrading, I’ll go for more memory.” Many folks will jump from 16MB all the way up to 64MB or 96MB, skipping 32MB altogether. Many will try to increase their system’s RAM all the way up to 128MB or even 256MB—and then they may get an unpleasant surprise.

Part of a computer’s configuration is contained in a chip commonly known as the Complementary Metal Oxide Semiconductor (CMOS) chip. Calling this specific chip the CMOS is quite misleading—many other chips in a PC could also be called CMOS chips—but that’s the common term. Anyway, the standard CMOS configuration first appeared in 1984 with the IBM AT. The CMOS sets aside 2 bytes, which are defined as the amount of kilobytes of RAM above the first megabyte. The largest value that a 2-byte integer can hold is 64KB-1, or 65,535. So the greatest amount of kilobytes above the first megabyte on a system with a standard CMOS is 65,535KB, or 64 megabytes.

What this bit of mathematics means is that no matter how many memory sockets are on your computer’s motherboard, no matter how much memory you put on your computer, it will recognize only the first 65MB (the 64MB above the first megabyte and then the first megabyte). What’s the answer? Make very certain when you buy the computer that it isn’t afflicted with an ancient CMOS—fortunately, this isn’t very common anymore. Talk to the firm’s technical support people and be sure that (1) they understand the CMOS problem and (2) their computer doesn’t have the problem.

TIP:  The bottom line on memory is this: On a Windows 2000 machine, the more memory, the merrier.

Video Card

Video cards are one of the most difficult types of hardware to choose for Windows 2000. First of all, what are you looking for? Games? Productivity? OK, admit it, you want a little of both. Today’s games often use high-end 3-D technologies that make them very exciting. Unfortunately, finding the drivers for the 3-D video cards can be hard with NT. Again, thanks to the unified driver model, Windows 2000 will use the same drivers as Windows 98. This means that the new video cards will very likely have driver support for Windows 2000 right out of the box.

Your best bet is to find out which chipsets are providing the type of performance you want, which cards have that chipset, and finally which manufacturers are providing Windows 2000 drivers.

Summary

Well, by now you know something of why you’d use Windows 2000 (or why you wouldn’t) and what to look for when choosing software and hardware. In the next chapter, you’ll learn about Windows 2000’s structure and architecture.
