Browsing—An Overview

When entering the Network Neighborhood, you are looking at a list of resources that are available across the network. In essence, you are “window shopping” (pardon the pun), or browsing the network. The Browse list, which is the list of resources that Network Neighborhood provides, simply shows you what is being shared across the network. The Browse list does not determine whether you can access these resources; that would be the responsibility of the permissions that are set in each machine, as we discussed earlier in the chapter.

In Microsoft networking, some computers have the task of maintaining this list and providing it to other workstations. The machines that perform this function are called browse masters. The unique aspect of the Microsoft browse master concept is that no one computer is fixed as the browse master. Instead, one computer is automatically “elected” as the browse master, in a manner that we’ll soon describe. (By the way, we also have backup master browsers, or machines that back up the browse master.)

NOTE:  In some Microsoft literature, browse masters are called master browsers, so you may see either term.

For example, you see a Browse list when you’re in My Network Places. Open the My Network Places folder and double click Computers Near Me, and it will look something like Figure 24.17.

The My Network Places window shows you the machines that are nearby (we’ll define “nearby” in a minute) and offers you access to the entire network. The machines Holly, Starbug, Blackwolf, and so on are all servers in the sense that they offer resources—shared printers or directories; Spiritwolf is an NT Server.

Understanding How Browsing Works

Chances are that you’ve used My Network Places dozens of times and never wondered how it worked; most likely, it just worked.

But if you ever saw the flashlight shine back and forth for a few minutes, followed by a blank folder or a “Network is not available” message, then you probably wondered just how My Network Places works—or doesn’t work.

Browsing is an example of a client-server software system. Your Windows 2000 Professional computer runs the client software, which interrogates the browse master.

The browse master runs the other half of the operation, which is the browse server software. As we’ve already told you, you needn’t anoint one PC as the browse master. Actually, the PCs all get together and jointly determine which of their number should be the browse master.

Electing a Master Browser

If there is no master browser, then the PCs hold an election to determine which computer would best be master browser. Elections are held when the master browser is powered down gracefully and other times as required.

NOTE:  Consequently, if the master browser is turned off, then it may take a good while (almost an hour in some cases) before another one becomes active.

If you’re not a master browser, then you’re a backup browser (browser backups in some Microsoft documents), a potential browser, or you’ve opted out of the whole election process. (We’ll show you how to do that in a minute.) Backup browsers are important because they are the ones giving you the Browse list. The general process that occurs when you open My Network Places is this:

1.  Your computer sends out a broadcast to locate the master browser.
2.  The master browser responds with the list of backup browsers.
3.  Your computer chooses a backup browser at random from the list and contacts that computer asking for the Browse list.
4.  The selected backup browser responds with the Browse list.

When an election occurs, the master browser is chosen with a scoring system that works like this:

•  The master browser is chosen from among the first four computers to respond to the election. The computer with the highest criteria among the first four will win.
•  NT Servers beat NT Workstations, which beat everything else (Windows 9x, Windows for Workgroups clients, or LAN Manager servers).
•  The Primary Domain Controller (which holds the master copy of the user database) will always be the master browser for the domain. Backup domain controllers will always be at least a backup browser. In Windows 2000, all domain controllers are equal if you are using Active Directory, so a Windows 2000 domain controller will always be at least a backup browser.
•  In the case of a tie, then the election goes to the first computer that responded. Although Microsoft isn’t too clear on this, machine type plays a role—Pentiums are preferred over 486s.
•  If no NT or Windows 2000 machines are responding, then the election takes place among the Windows 9x workstations that have their MaintainServerList setting set to Auto or Yes.

That last part is what you should be worried about.

Browsing in General

Thus far we have seen that browsing is a method whereby servers on the network tell a computer called the master browser who they are and what they offer to the users of the network. The browsing service in Microsoft enterprise networks makes it possible for a workstation to see what the network has to offer. A few specifics about browsing:

•  The master browser designates one backup browser for about every 15 computers.
•  If you run multiple transport protocols, then each transport protocol needs its own set of browsers.
•  Backup browsers verify their database with the master browser every 15 minutes.
•  Every 12 minutes the master browser contacts the domain master browser (Domain Controller) to exchange Browse lists.
•  Servers first announce their existence to the master browser; then they re-announce their existence periodically. Eventually, they “settle down” to announcing themselves only once every 12 minutes on every installed protocol.

NOTE:  If you want to find out which computer is the master browser on your Windows 2000 Server-based network, then you can use the browmon.exe program that comes with the Windows 2000 Resource Kit.
