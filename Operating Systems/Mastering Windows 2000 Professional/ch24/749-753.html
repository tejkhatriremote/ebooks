Assigning Share-Level and File-Level Permissions

From the beginning, you need to be aware that you can assign file permissions only to files and directories that are located on an NTFS partition, but that share-level permissions can be applied to shares on either FAT or NTFS. Here’s how to assign permissions:

1.  Click the file or directory that you would like to assign permissions to.
2.  Click the Security menu option, then press the Add button.

NOTE:  Windows 2000 is smart enough to realize that if you have highlighted a directory, you are getting ready to assign directory-level permissions; and if you have highlighted a file, you are getting ready to assign file-level permissions.
3.  Make your selections from the displayed list of groups that you can add to the permissions list.

NOTE:  You can select groups that are built on your machine, groups that are part of the domain that you logged on to, or groups that are members of any domains that are considered trusted domains to your domain. If you would prefer to select individual users to add to the permissions list, click the Show Users button and scroll down the list; the individual users now appear.
4.  After the groups and users appear in the Add Names box, you can assign their access permissions by clicking OK to return to the Permissions dialog box, as pictured in Figure 24.5.

Directory and File Auditing

In addition to assigning file and directory permissions, Windows 2000 Professional enables you to keep track of who accessed a file and when a person accessed a file. This function is handled through auditing. The auditing feature in Windows 2000 Professional enables you to select the individual or group that you would like to audit. If you would like to perform a general audit of whenever anyone accesses the file or directory, select the Everyone group for your audit.

NOTE:  To make auditing available on your Windows 2000 Professional computer, you must first use the Computer Management console to enable auditing. On the Policies menu, select Auditing. Click Audit These Events and select the event you wish to track. Beware of enabling too much auditing because it will fill the Security log and use more system resources. Audit only what you need to audit.

To enable the auditing of a file or directory, select the file or directory, right-click the icon, and select Properties. From the Security Properties dialog box, select Advanced and then click the Auditing tab to bring up the dialog box shown in Figure 24.6.


Figure 24.6  The Auditing tab for a folder

Click the Add button to display a list of users and groups that you can choose to audit. You will then select the events that you would like to audit in the Auditing Entries box.

By default, the directory audit will audit all the files at the top-level directory that you are auditing. If you want to audit all the subdirectories, click the top check box on the Directory Auditing screen and Windows 2000 will propagate your request to the correct subdirectories.

NOTE:  Only an administrator can set up auditing and view the audit logs.

Understanding Ownership

Ownership—what a confusing concept. Ownership is a process by which you can take exclusive control over a file or directory; and you can do all of this with a click of a button. But before you get power drunk with the possibilities, let’s take a closer look at what being the owner of a file really means.

Defining Ownership

Now, having worked with NT since its inception, we don’t mind telling you that the whole idea of a directory or file’s “owner” seemed a bit confusing until we finally figured out the definition. Here’s a definition—and from this point on, let us shorten the term file or directory to object:

Minasi’s Definition of an Owner  An object’s owner is a user who can always modify that object’s permissions.

Ordinarily, only an administrator can control settings such as an object’s permissions. But you want your users to be able to control objects in their own area, their own home directory, without having to involve you at every turn. For instance, suppose you want to give another user access to a directory in your home directory. Rather than having to seek out an administrator and ask the administrator to extend access permissions to another user, you as the owner can change the permissions directly. Ownership enables users to be mini-administrators, rulers of their small fiefdoms.

You can see who owns an object by right-clicking the object and choosing Properties. You’ll then see a dialog box with several tabs, one of which is Security. Click the Security tab, click the Advanced button, and you’ll see an Owner tab. Click that, and you’ll see who owns the object, as shown in Figure 24.7.
