Using Net Use: Connecting to Other Drives and Printers

After you’ve browsed the network with Net View, you can connect to all the available goodies (or disconnect from those you don’t want) with the Net Use command. Use this command to connect to network resources as drives D through Z and printer ports LPT1 through LPT9.

NOTE:  The Help file on this command claims that you can use only printer ports LPT1 through LPT3. Technically, this statement isn’t true, but the Help file probably puts it this way because some MS-DOS applications are not able to access printer ports with numbers higher than 3.

To get information about the workstation’s current connections, type Net Use without options, and you’ll see something like Figure 24.23.

Using Long Filenames in UNCs

If you wanted to connect to a directory called Wpfiles on a server and to make that your E drive, you’d substitute E: for LPT1: and Wpfiles for HP4M in the preceding example. You get to specify the port name or drive letter that you want to connect a resource to, but you’re restricted to drive letters d through z and ports lpt1 through lpt9. Also, if the computer that you’re getting the resource from has a blank character in its name (that is, has two words in it), you must put the name in quotation marks, as in:

   "\\eisa server"

If a password (let’s say it’s “artuser”) is attached to the resource that you’re trying to connect to, you need to include that in your connection command, like this: 

   net use lpt1: \\ted\hp4m artuser

Or if you want the computer to prompt you for the password so that it isn’t displayed on the screen, append an asterisk: 

   net use lpt1: \\ted\hp4m *

Creating a Drive Mapping for Your Home Directory

To connect to your home directory (the directory on the server that has been assigned to you, assuming that there is one), type: 

   net use /home

with the (optional) password on the end as explained above.

If you want to make the connection for another user, rather than for yourself, add the user’s name (Frank) to the end of the line, like this:

   net use lpt1: \\ted\hp4m user:frank

Passwords go before the user’s name in the statement. If the user for whom you are making the connection is in another domain, the user part of the statement looks like this: 

   user:domainname/frank

where domainname is the name of that user’s home domain.

Connecting to a Resource in Another Domain

If you want to connect to a resource in a domain that is not your usual one, you must first log on to that domain. If your domain and the other domain don’t have a trust relationship with each other, you need to create a user account for yourself on the second domain. After you’ve logged on to the proper domain, the process is the same as described above.

Using Other Switches

No matter what kind of connection you make, you can make it persistent (that is, remake it every time you connect to the network), by adding the switch /persistent:yes to the end of the line. If you don’t want it to be persistent, use /persistent:no instead.

If you don’t specify one or the other, the default is whatever you chose last. If you want to make all future connections persistent, type:

   net use /persistent:yes

(Or type :no if you want all future connections to be temporary.) Typing /persistent by itself at the end of the line won’t do anything.

To disconnect from a resource, type:

   net use devicename /delete

where devicename is the connection (such as D or LPT1). You don’t have to provide a password or say anything about persistency to disconnect from a resource.

Creating Hardware Profiles

If you use a laptop for business, you likely have encountered the following problem. When you bring your laptop into the office, you would like your computer to recognize the NIC, maybe an external monitor and keyboard. But when you take your laptop on the road, you want it to know that you do not currently have any of the “office” devices attached. Most laptops get their information by timing out on each device, which adds five minutes to the boot process and wastes time, not to mention wasting valuable battery life.

A feature of Windows 2000 Professional that was designed to alleviate this problem is a hardware profile. Windows 2000 enables you to create a profile of the hardware that is attached to your machine when you are in the office and specify a different hardware profile for when you are on the road. And the fun doesn’t stop there. You can create profiles for all the places in which you use your computer.

To create a hardware profile, open the Control Panel and double-click System (see Figure 24.25).


Figure 24.25  System Properties dialog box

Click the Hardware tab and press the Hardware Profiles button (see Figure 24.26) to see the available profiles.


Figure 24.26  Hardware Profiles in the System Properties dialog box

To create a new hardware profile, choose a configuration from the Available Hardware Profiles list and click Copy. This step creates a copy of the selected profile and enables you to give the copy a new name. We’ll show you how to configure these profiles in a few moments.

To play with the docking properties of this profile, click Properties. The next screen, as shown in Figure 24.27, is the Profile 1 Properties dialog box where you can specify whether the system is going to be attached to a docking station when it is using this profile. Note that a docking station is a box that sits on your desk (usually at the office) that you plug your laptop into. You use it because you can attach peripherals to it such as a larger monitor and keyboard. Then, when you come into the office, you simply plug your laptop into the docking station and you can use the larger monitor and keyboard instead of the smaller ones on the laptop. However, when you do this, your laptop needs to know that sometimes it’s using the laptop screen, whereas other times it’s using the larger monitor. You tell the laptop this by configuring the docking properties.
