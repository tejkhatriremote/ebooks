Lengths of Different Network Names

For bizarre historical reasons, machine names, people names, and passwords are established according to different rules in a Microsoft network.

NOTE:  Machine names can be up to 15 characters long. You can include spaces, but we recommend against it; we’ve run into some bugs in NetBIOS software when handling machines whose names contain blanks. If you are using TCP/IP as your network protocol, your computer name can be up to 63 characters in length and may contain only upper- and lowercase letters and numbers. You may also use the hyphen (-) in the name.

Passwords can contain up to 14 characters if your Windows 2000 computer must coexist with Windows 9x computers, and up to 127 characters in a “pure” Windows 2000 environment. You can use non-letter characters, so you can have a password like “eyes. Blue” or “1e907bK.” This makes your passwords even more secure because passwords that contain one or more numbers are harder to guess. Unfortunately, there doesn’t appear to be a simple way of forcing a user to choose a password with non-character values. Uppercase and lowercase count; password and PASSWORD are two different passwords and they can’t be substituted for each other. See Table 24.1 for a clear list of how long passwords and names can be.

Table 24.1 Password and Name Length Limits
Element Character Length Case sensitivity
Passwords 14 characters with Windows 9x on the same network 127 in windows 2000 Case sensitive
User names 20 characters Not case sensitive
Machine names with TCP/IP installed 63 characters Not case sensitive
Machine names without TCP/IP installed 15 characters Not case sensitive

Sharing Files under Windows 2000 Professional

Windows 2000’s GUI simplifies sharing directories and using shared directories. For example, suppose you want to share the Eudora directory on your computer. Open My Computer, open the drive Eudora is stored in, and right-click the Eudora folder. One of the options on the menu is Sharing; choose it to display the Sharing tab of the dialog box (see Figure 24.1). Implement sharing by clicking the radio button labeled Share This Folder.


Figure 24.1  The Sharing tab in the Properties dialog box. In this case, the user is planning to share a Eudora e-mail folder

A share name is generated for you, but you are not limited to that name. The Comment text box, shown in Figure 24.1, enables you to briefly describe the share. You can also specify the maximum number of users who can access this share simultaneously. This feature can assist you with software licensing. For example, if you know that you have only 10 licenses of your Eudora e-mail program, and the licensing states that it is a 10-user concurrent license, you can set the number of connections to this share to “Allow 10 Users.” Thus you can enforce the licensing of any program.

NOTE:  Windows 2000 Professional is hard-coded to a limit of 10 inbound user sessions. You can set a limit of fewer than 10, but Windows 2000 won’t allow more than 10 at one time.

If a client workstation tries to attach to a share that is already being used by the maximum number of allowable users, it will receive a warning message. The exact error message varies from operating system to operating system, but a DOS workstation would receive the following message: 

   Error 71: This request is not accepted by the network. The server may
   have run out of resources necessary to process your request. Try the
   request again. If the error persists, contact your network
   administrator. For more information, type NET HELP 71 at the command
   prompt.

Windows 9x workstations are more terse, saying:

   An extended error has occurred.

Windows for Workgroups workstations also believe in keeping it brief, with the following rather uninformative warning: 

   This request is not accepted by the network.

NT or Windows 2000 clients are a bit more chatty and descriptive: 

   No more connections can be made to this remote computer at this time
   because there are already as many connections as the computer allows.

Now, before you start relying on this feature, let us tell you about a problem in NT Workstation 3.1 and 3.5 and Windows for Workgroups machines. The user-maximum feature seemed not to work. You could create a share, give it a maximum number of users of 1, and then fire up 20 Windows for Workgroups machines—and they all could get onto the share. Our experience with Windows 2000 leads us to believe that Windows 2000 Professional does enforce the user maximum, but test it before you build a security system around this feature.

Now that you can share directories, what if you want to share a directory with one person only or with only a specific group of people? Windows 2000 sports a sophisticated two-layer security approach to user access of shared directories and files.
