One of the most powerful features of Windows 2000 Professional is its capability to attach to and become part of a networking environment. In this chapter we examine many of the decisions that you will need to make in order to get the networking features to run reliably in that environment.

The whole idea of a network is to share things: share space on a large disk drive, share a particular file on that disk drive, and share a printer. As an example, let’s consider a small office that needs to do some sharing.

In our office, Jennifer has more storage capacity on her machine than Joe does on his, but the office laser printer is attached to Joe’s PC. Jennifer and Joe work on the office accounting system, so they need to share the accounting files—or they’ll have to pass floppies back and forth. Because Jennifer has more disk space, they put the accounting files on her machine. So, the network problems that we need to solve are:

1.  Sharing Joe’s printer with Jennifer
2.  Sharing Jennifer’s disk with Joe

Let’s solve their problem with a basic Microsoft network. With this type of network, Jennifer puts her hard disk on the network and Joe puts his printer on the network. Assuming that both computers are running Windows 95, here’s how to get Joe onto Jennifer’s disk and Jennifer onto Joe’s printer:

1.  Jennifer tells the Windows 95 networking software, “Offer the Acctng subdirectory on my C drive to anyone who wants it. Call it Acctng.” In Microsoft enterprise networking terminology, Acctng becomes the share name of that drive on Jennifer’s machine (GTW09), and it’s the name that others will use to access the drive over the network. In a few pages we’ll show you exactly how to share such a resource on the network so others can access it, but for now, remember these important facts: A machine (named GTW09) is sharing a resource called Acctng with anyone on that network who’s able to use it.

NOTE:  Here’s an important concept in Microsoft networking: You must name each machine in the network, whether it is a server or a workstation. You also must name each user (in our example, Joe and Jennifer). Because the PCs need names, we may as well name the PCs with their inventory numbers, which in this example are DELL05 and GTW09.

TIP:  Naming machines after their users is a bad idea, because PCs may be reassigned to other users.
2.  Joe then tells the networking software on his PC, “Attach me to the Acctng resource on Jennifer’s machine.”
3.  Joe, meanwhile, runs his network software and tells it to share the printer on his LPT1 port, giving it a name—again, a share name—of JOLASER. Joe’s machine is called DELL05, so the UNC name of that printer will be \\DELL05\JOLASER.
4.  Jennifer then tells her networking software to attach JOLASER on Joe’s machine to her LPT1 port.

From now on, whenever Jennifer tells an application program to print to a LaserJet on LPT1, the network software will intercept the printed output and will direct it over the network to Joe’s machine. The networking software on Joe’s machine will then print the information on Joe’s printer.

We know we left out some of the “how do we do this?” information; it’s coming later, we promise. We just wanted to illustrate the following concepts:

•  Machines and people in a Microsoft network have names.
•  Shared resources in those networks also have names.
•  You specify network resource names with UNC names, in the format \\servername\sharename.
•  One way (and the most common way) to share network resources is to give them bogus local names, such as D or LPT1.
Naming a Network Resource

For years, PC workstation operating systems haven’t understood networking, so we’ve had to introduce network resources in the back door by showing these resources as logical drives or, in the terminology of some networks, mapped drives. We might say that the Acctng resource is mapped to local drive D for Joe. Joe doesn’t know whether Acctng is all of Jennifer’s drive or just part of it. Joe’s networking software then says something like, “You’re now attached to Acctng on Jennifer’s machine. It will appear to you as local drive D.”

Actually, we should mention that Windows 2000 Professional doesn’t require drive letter mapping. We mention it because virtually every previous PC network operating system required it and because most applications to this day can still address network resources only by letter names. Again, Joe doesn’t have a D drive on his machine; he only has network software that takes read and write requests for a mythical drive D and reformulates those requests into network communication with Jennifer’s machine.

And, while we’re on the subject, the actual name of this network resource is \\gtw09\ acctng. Notice the \\gtw09\acctng term; it’s called a Universal Naming Convention name, or a UNC name. The two backslashes are a warning that the name following is a machine name, and the backslash after that refers to the share name acctng rather than the directory name.

Understanding Networking Concepts

A fundamental term in Microsoft networking is a server; any machine that shares resources is a server.

Network Server and Workstation Types

Various types of machines can operate as servers in the Microsoft world. Let’s name them now:

•  Windows 98 workstations
•  Windows 95 workstations
•  Windows for Workgroups machines
•  NT Workstation computers
•  NT Server computers
•  Windows 2000 Professional computers
•  Windows 2000 Server computers
•  LAN Manager servers
•  OS/2 LAN Servers
•  DOS-based machines running LANtastic
•  OS/2 Warp Connect machines

The machines that can act only as workstations are those that are running one of the following:

•  Windows 3.x
•  DOS
•  OS/2 1.x and 2.x
