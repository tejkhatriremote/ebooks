Creating User Profiles

For every new user you create, Windows 2000 creates a default profile called ntuser.dat, which lives in the C:\Winnt\Profiles\Username (for example, Jughead) subfolder. If you log on as this new user, Windows 2000 Professional displays an un-configured, vanilla Desktop. The cool thing is that Windows 2000 will store any changes you make to the environment, and they’ll be there the next time that user logs on. Although a profile is created for all users, changes to the Desktop apply only to local users because users connecting via your LAN are primarily using file and print services.

Suppose you’ve created a new user account and now you want to configure that user’s environment. Here’s what you need to do:

1.  In the Local Users and Groups console, select the name of the user whose profile you want to modify and double-click it. Windows 2000 displays the User Properties dialog box.
2.  Click the Member Of tab.
3.  If this user doesn’t already belong to the Administrators group, click the Add button, select the Administrators group and click Add, and then click OK.
4.  To change the user’s password, right-click the account in the right pane of Local Users and Groups, and select Set Password from the context menu. Set the user’s password to something you’ll use to log on locally as that user, and then type the password a second time to confirm.
5.  Click OK to close the Set Password dialog box.
6.  Exit Windows 2000 by clicking the Start button, choosing Shut Down, and selecting Log Off.
7.  Press Ctrl+Alt+Delete, and at the logon prompt, type the user’s name and password. Click OK.
8.  You’re now logged on as that user, so go ahead and make whatever changes are necessary (Desktop wallpaper, shortcuts to disk drives, applications, and so on).
9.  When you’ve finished, log off as that user and log back on using your usual account (one with administrator-level access).

TIP:  After setting up a new user’s environment, be sure to log back on to the system as Administrator and remove that user from the Administrators Group, unless you want that user to have administrator-level access.

You can also change a user’s environment by using the User Properties dialog box. Click the Profile button, and Windows 2000 displays the User Environment Profile dialog box. Although you can change a user’s environment using the steps outlined above, you can also set a pointer to an existing profile that lives on another system on your network. The advantages of this approach are economy of hard-disk space and a central location where your network administrator can change profiles once and have those changes apply to all systems on the network. The one disadvantage to this method is that the system must have an uninterrupted network connection. If a user’s profile is on an unavailable network, he or she won’t see the usual environment when logging on.

Suppose you want to set up an account for a user whose profile lives on another machine. To set a profile path that points to that existing profile, you’d use the following format:

   \\Servername\Profile Folder\Username

So, a typical profile path might look something like this: 

   \\MissionControl\UserProfiles\Jughead

NOTE:  By default, a user profile lives in the C:\Winnt\Profiles\Username subdirectory and is called ntuser.dat. So, if you don’t specify a profile path, Windows 2000 will look there to read the user’s profile information.

The two types of roaming user profiles are personal and mandatory.

Personal profiles
Can be locally based or server based and are updated with any changes the user has made when the user logs off. By default, all profiles are personal profiles.
Mandatory profiles
Can be locally based or server based but they aren’t updated when the user logs off. If you have a specific environment that you want users to see and you don’t want them to change it, change their profile to mandatory.
