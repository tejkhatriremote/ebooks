Creating Striped and Spanned Volumes

Windows 2000 can create large volumes that span several hard drives, which can be useful if you work with large amounts of data. Windows 2000 can also create a striped volume, which consists of two or more disks “striped” together; stripe sets enable you to access large data files faster. In other words, when you tell an application to write to disk, data is written to all disks in the stripe set simultaneously, with each one receiving part of the file. The result is faster throughput for both reads and writes, although your hardware costs increase and you don’t effectively add to your storage capacity.

NOTE:  You may have heard the term RAID, which stands for Redundant Array of Inexpensive Disks. RAID technology is usually used only on servers that process a great deal of data or that get many data requests from many users. There are different RAID levels, and each has its benefits and liabilities. The Windows 2000 striped volume is considered RAID 0—data is striped across at least two drives. The benefit is increased performance. The liability is added risk: If one of the drives fails, the entire volume will be lost if it hasn’t been backed up. Also, most file servers have hardware RAID disk controllers, but Windows 2000’s implementation of RAID 0 is done in “software.” Your CPU, therefore, has to do the work of splitting and reassembling files that are read from or written to the striped volume.

TIP:  Disk striping is the fastest partitioning scheme available in Windows 2000. It can use either FAT or NTFS but cannot contain the system or boot partition. In addition, in Windows 2000, striped volumes can only be created on dynamic disks.

Windows 2000 can also create a spanned volume, which groups several formatted disks to create a single, larger volume that Windows 2000 sees as one unit. This technique is more commonly known as spanning of disks, or parts of disks. For example it offers the advantage of letting you create a single, very large data drive for general storage rather than having to use each drive individually. Here’s how to create a volume set:

1.  The drives or drive parts you’re including should contain no partition information. They should read Free Space.
2.  Hold down the Ctrl key and select the drives or parts of drives you want to include in the new volume set.
3.  Next, from the Action menu, chose Create Volume Set.
4.  Windows 2000 asks for the volume set size. If you decide you want the size to be less than the sum total of the included drives, Windows 2000 will try to distribute the volume set evenly across its member drives.
5.  To format the new volume set partition, from the Partition menu, select Commit Changes Now.
6.  Next, select the volume set and right-click it—or click the Action menu—and select Format. Windows 2000 prompts for which file system you’d like to use. Select NTFS and leave the QuickFormat box unchecked. Windows 2000 now formats the new volume set. Depending on the volume set size, this process may take some time.
7.  To save these new changes to your configuration, run the Windows Backup utility to create a new ERD.

Creating a striped volume is similar to creating a spanned volume, but there is a consideration: If you dual-boot your system, you won’t be able to see this striped volume when running Windows 3.1 or Windows 9x. Here’s how to create a striped volume:

1.  The drives or drive parts you’re including should contain no partition information. They should read Free Space.
2.  Hold down the Ctrl key and select the drives or parts of drives you want to include in the new striped volume.
3.  Next, from the Action menu, choose Create Striped Volume.
4.  Windows 2000 asks for the striped volume size. If you decide you want the striped volume size to be less than the sum total of the included drives, Windows 2000 will try to distribute the striped volume evenly across its member drives.
6.  Next, select the striped volume and right-click it—or click the Action menu—and select Format. Windows 2000 asks which file system you’d like to use. Select NTFS and leave the QuickFormat box unchecked. Windows 2000 will now format the new striped volume. Depending on the striped volume size, this process may take some time.
7.  To save these new changes to your configuration, run the Windows Backup utility and create a new ERD.

TIP:  A Windows 2000 striped volume is non-fault-tolerant: If one drive goes south, the whole stripe set is history. Be sure to back up stripe sets regularly.

Using Dynamic Volumes

Dynamic volumes are a new concept in Windows 2000. Traditional partition types stored their information in a partition table that was located in the first physical sector of the hard disk. It was this location, and its limited storage space for partition information, that limited us to four partitions per physical disk. The partition table stores information for the size and location of each partition, and is used to find the bootable partition during the startup procedure.

Dynamic volumes store their partition information in the data portion of the drive, and thus they are not limited in the amount of information they can store. To convert a basic disk to a dynamic disk, right-click the disk in the Disk Management tool and select Upgrade to Dynamic Disk from the context menu. You will have to reboot to perform the conversion. After the restart, you will be able to create or modify dynamic volumes on the disk, including expanding the volume across multiple disks.
