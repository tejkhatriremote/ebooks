Setting File/Directory Access

Although you can set a great many parameters in Users and Passwords, you cannot completely set up file and directory access rights here. You can, however, complete half the task. Here’s how to grant access for a group of users to a subfolder or even specific files:

1.  Create a new group and then add those users to that new group.
2.  Use Explorer or double-click the My Computer icon to view the subfolder(s) or file(s) for which you want to assign special access.
3.  Right-click it/them, and select Properties.
4.  In the Properties dialog box, select the Security tab.
5.  By default, the Everyone group has Full Control permissions; so you may want to remove this access if the files contain private information. To do so, highlight the Everyone group and click the Remove button.
6.  Click the Add button to open the Select Users or Groups dialog box, and pick the users or groups that you want to have permission to this folder or file. When you have added all of the accounts to the bottom pane, click OK to return to the Security dialog.
7.  Assign access to the group(s), specify the level of access, and click OK.
8.  Click OK again to close the folder’s or file’s properties sheet.

For more details on access levels, see Chapter 19.

You can set your system’s Account Policy to make unwelcome logons much more difficult. In the Local Security Policy console, expand the Account Policies console tree and click either Password Policy, or on Account Lockout Policy as shown in Figure 30.20. You will have to set the individual policies here, just like setting the user rights earlier in this chapter.

It’s a good idea to plug as many security “holes” as possible, not the least of which have to do with passwords. Table 30.8 describes how to set password options in the Account Policy console so that your system is more secure.

Account lockouts are a good idea because otherwise someone with a password-guessing program can hack at your system all day and may eventually gain access. It’s a good idea to set a lockout after five logon attempts. The Reset Count After option specifies the maximum number of minutes that can occur between any two incorrect logon attempts before lockout results. Set this parameter to five minutes. For Lockout Duration, select Forever to cause locked accounts to remain locked until an administrator unlocks them. Using these parameters gives additional security and discourages unwanted visitors.

Running System Information

When everything on your system works, new devices install painlessly, and services work right the first time, life is good. If only it were always so. Alas, the general headaches of new hardware installation follow—cracking the box open, finding an available I/O slot, dealing with the mysterious force that makes your system’s box much easier to rip open than it is to put back together.... Then, after you restart the system, there may be I/O resource conflicts or services may fail for whatever reason. The fun never ends…

Fortunately, Windows 2000 provides some tools to ease system troubleshooting. Chief among them is System Information (see Figure 30.21), which lets you see system information ranging from detailed display driver information to the services that are installed and the other services or devices on which they depend.

By the way, even though System Information delivers the most information, several other system tools will be helpful when trouble arises.

The first to mention is the Windows 2000 Task Manager, which you can launch by right-clicking the Taskbar and selecting Task Manager. In addition to showing which applications you have open, Task Manager displays all processes that are running and enables you to terminate specific processes. What’s more, it delivers constantly updated performance data, CPU usage, and detailed memory statistics.
