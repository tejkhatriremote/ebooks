The process of creating a new group account is very similar to creating a new user account. Local groups are very useful for assigning permissions to resources. To create a new group account, you must use the Local Users and Groups icon in the Computer Management console, and follow these steps:

1.  Open the Computer Management console in Administrative Tools, and expand System Tools Local Users and Groups.
2.  Right-click the Groups folder and select Create Group from the context menu. This will open the Create Group dialog shown in Figure 19.9.


Figure 19.9  The Create Group dialog box lets you make a new group account and add members to it

3.  Type in a name for the group in the space provided. The name can contain any numbers or letters up to 256 characters. The name must be unique in the local database.
4.  Enter some text in the Description field that will describe the membership and purpose of this group.
5.  Click the Add button to open the Select Users, Computers, or Groups dialog box shown in Figure 19.10. This dialog box can add users from the local computer or domain, and global groups from the domain. Use the Look In drop-down list box to select the correct database, either the local computer or a domain from which you will add users or groups.


Figure 19.10  The Select Users, Computers, or Groups dialog box

6.  Add users or groups from the displayed list by double-clicking them or by typing in a user or group name in the Name field and clicking the Add button. The names that you have selected should appear in the lower pane under You Have Selected the Following Objects. When you are satisfied with the list of names, click the OK button to close the Select Users or Groups dialog box and return to the Create Group dialog box.
7.  Click the Create button to make the group account.

Waiting for Changes to Take Effect

Suppose you give yourself some nifty new rights—perhaps you add yourself to the Administrators group. When does this change take effect? Well, remember, Windows 2000 grants you the ability to do things based on your SAT. The SAT contains all the things that you can set in User Manager, such as username, password, the groups you belong to, and what rights you have.

But when is the SAT created? When you log on. So for any new information to show up on your SAT, you must log off and then log back on. Please note that you do not have to reboot, only log off.

Making the Locks

We have mentioned Security Descriptors quite frequently in this chapter. They are lists of people who are authorized to access objects, and what those people can do with those objects—read them, delete them, and so on. Let’s zero in now on exactly how you can examine an SD, modify it, and use it to your advantage.

Understanding SDs, ACLs, ACEs, and Permissions

The truth is that you can’t directly view or control an SD; you can only control a part of it. We’ve already said that an SD is a list of people (or, actually, a list of people and of groups) with varying degrees of access to an object, but what we haven’t said is that there are people on that list that you can’t remove. Windows 2000 allows you to view and change the people on the SD list, but it doesn’t show you the ones that it won’t let you change.

The subset of people whose access you can change is called the access control list or, inevitably, the ACL. (Seems like the security people at Microsoft are particularly fond of abbreviations.) While we’re at it, here’s one more Microsoft term: access control entry, or ACE—it’s the term for an item in an ACL. So if an ACL contains three items (Joe can read, Mary can read, Paul can read and write the object), then the ACL contains three ACEs. The ACL is also sometimes called the permissions list.

Modifying File Permissions

To attach ACLs to files and directories on a particular drive, you must have formatted that drive as NTFS instead of as FAT or FAT32. You can examine the ACL for a particular file by opening Explorer or My Computer, working your way down to a file, and right-clicking the file. You’ll see a Security tab in the Properties dialog box shown in Figure 19.11. (Again, you will not see the Security tab if the drive is formatted as FAT or FAT32.)


Figure 19.11  Security tab on a file showing the list of users with permissions to the file

File permissions is another name for the security information stored by Windows 2000. Note that this permissions list/ACL contains just one entry, a group called Everyone that has Full Control of the file (see Figure 19.12). You can add or delete people from the permissions list with this dialog box. You could remove the Everyone group with the Remove button; you do that, but now you need to give someone access to this file, or it’s of no value. So you click Add to display the Select Users, Computers, or Groups dialog box, as shown in Figure 19.10.


Figure 19.12  Access Control Settings dialog box

The dialog box shown in Figure 19.10 displays the same groups as the Local Users and Groups console—with a few additions: the Everyone group that we’ve already referred to, as well as INTERACTIVE, which just means anyone logged on this machine, and NETWORK, which means anyone accessing this machine over the network. (These groups are visible when you scroll down the list.) The list also contains a group called SYSTEM, which is the operating system itself, but you can’t see it unless you scroll down; ditto for the Users group, which is the regular old Users group that you met in the Local Users and Groups discussion.

When you have added the names that you want to grant permissions to, click the OK button to close the dialog box and return to the Security dialog box for the file.
