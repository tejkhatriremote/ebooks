You can set the following parameters for this connection from the Modem Configuration dialog box:

Maximum Speed
Sets the maximum transmission speed for the modem connection, just as the name implies. Keep this setting high, as your modem can occasionally go faster than its rated speed if the line conditions are particularly good.
Modem Protocol
This setting is dependent on the type of modem installed. It enables you to choose whether the modem will use error correction, force error correction, or disable error correction altogether.
Enable Hardware Flow Control
Enables handshaking between the modem and the computer so that if the modem falls behind in receiving, it can halt the flow of data rather than just register an overflow error. In short, keeping this option enabled allows the modem to slow its eating speed rather than choke.
Enable Modem Error Control
Enables error checking through cyclical redundancy checks (CRCs). This option increases modem efficiency because it eliminates the need for start and stop bits (bits that signal the beginning and end of data transmission).
Enable Modem Compression
Enables your modem to compress the data stream using the V.42 bis compression algorithm. This option is not checked by default, but we recommend checking it. Not all modems support compression, which is probably why this setting is disabled by default. Most high-speed modems do support compression nowadays, however, so it’s probably a safe bet to enable compression. Just as data-compression programs such as Stacker work better on some kinds of files than others, the amount of compression that takes place depends on how much redundancy is in the data transmission, which varies with the type of transmission.
Enable Modem Speaker
Enables the modem to work silently or to keep you informed of every beep and squawk. This setting is pretty self-explanatory: With a check in the box, every squeal and grunt from the modem is broadcast through its speaker.
Initialization
This section contains two settings that control the startup of the connection. The first option asks if you want to Show Terminal Window when you connect. This would enable you to enter special commands when connecting. The other option, Run Script, lets you specify a script that contains all of those commands you would run from the terminal window during connection.

Options Tab

The Options tab of the connection Properties dialog box sets the basic behavior of the computer while it’s trying to connect. This tab is shown in Figure 26.4.


Figure 26.4  The Options tab lets you control dialing options.

The configuration settings include:

Display Progress While Connecting
Keeps you informed of the progress of the connection while the modem is dialing the server.
Prompt For Name and Password, Certificate, Etc.
Specifies whether you will be prompted for credentials before connecting; this includes your user name, password, and certificate if you are using something like a smart card for additional security.
Prompt For and Include Windows Logon Domain
This will eliminate one of the most annoying glitches when dialing the Internet. In NT 4, you would be prompted for a user name, password, and domain name every time you dialed your connection. If you were dialing an ISP and supplying a domain name, the ISP (frequently a Unix computer) would hang up because it didn’t understand the domain name. If you are dialing an ISP, you can simply uncheck this box to prevent this from happening.
Prompt For Phone Number
Determines whether you will be prompted for the phone number prior to dialing the connection.
Redialing Options
This area of the tab includes settings that control how the line will be redialed if it is busy or there is no answer. You can make settings that determine how many redial attempts will be made, how many seconds will pass between each attempt, and idle time before the modem will hang up.
X.25
This button opens the properties sheet for your X.25 connection.

Security Tab

The Security tab of the connection Properties dialog box is shown in Figure 26.5.


Figure 26.5  The Security tab in the connection Properties lets you determine the type of encryption for logon.

The Security tab contains the following settings:

Security Options
Contains settings for the different types of logon validation that are supported by Windows 2000. The Typical Settings include the standard validation schemes ranging from Allow Unsecured Password to Use Smartcard. The Advanced Settings includes the ability to further customize the encryption protocols used to log on to this connection.
Interactive Logon and Scripting
Includes settings to enable you to open a terminal window to perform your logon, and to use a logon script.

The Advanced Settings enables you to select multiple forms of logon encryption as shown in Figure 26.6. Two of these options, PAP and CHAP, are used commonly by Microsoft operating systems and are discussed below.

Password Authentication Protocol (PAP)  The first level of security is established during the connection. If you have selected PPP as your server type on both the client and the server, then you can utilize a technology known as the Password Authentication Protocol (PAP). Before PAP, the server, client, and user held the following conversation:

Server (to Client): Do you use PAP?
Client (to Server): No.
Server (to Client): What is the user’s name?
Client (to User): What is your logon name?
User (to Client): Frank.
Client (to Server): The user says his name is Frank.
Server (to Client): Great, what is the user’s password?
Client (to User): What is your logon password?
User (to Client): Doghouse.
Client (to Server): The client says his password is doghouse.
Server (to Client): Thank you.

This conversation would take place every time the user wanted to log on to the network remotely; no encryption of information is being sent back and forth. With some network servers, you may have to create a script file to automate this conversation. In either case, the network administrator has to understand how to create the script file. Scripts in theory are very straightforward, but the actual syntax can vary from hardware device to hardware device and from network operating system to network operating system.
