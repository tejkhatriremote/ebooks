Without software, your computer is nothing but an expensive hunk of plastic and metal. It takes software to make your computer come alive and do something useful. An operating system, such as Windows 2000 Professional, serves as the platform for running your application software. Although it is most helpful to learn how to use your favorite applications so that you can be productive with them, it is also important to learn how your software interacts with your operating system. This knowledge will not only help make you a happier and more informed computer user, it will also help you diagnose and solve problems—and perhaps become the envy of your office.

Eliminating Programs You Won’t Need Anymore

When you make the move to Windows 2000 Professional, you won’t need to take all your old DOS and Windows programs with you. Some just won’t work and others have been replaced by utilities that are built into Windows 2000 Professional. In fact, many of your Windows 95 utilities can be left behind since they have been replaced by new built-in utilities. In this section, we’ll discuss which programs you won’t need anymore, why you don’t need them, and what Windows 2000 Professional has replaced them with, if anything at all.

TSRs

Terminate and Stay Resident programs (TSRs) are small programs that DOS would run at bootup and then leave in memory for as long as DOS was in operation. They were a mainstay in the days of DOS. As a rule, TSRs would provide functionality not found in DOS. The most common TSRs were those that enabled DOS to use memory more effectively, to load support for your sound card, to use CD-ROM drives or other Small Computer System Interface (SCSI) devices, or to connect to a network.

You don’t need TSRs under Windows 2000 for a number of reasons. The most important among them is that they won’t work. Even if you had DOS installed and running in dual-boot mode with NT, all aspects of DOS are ignored once NT boots. Windows 2000 would not even be aware of your CONFIG.sys file, if it existed. Not that it matters, anyway, the functions that most TSRs were designed for are already present in NT.

The functionality of TSRs is replaced by the Windows 2000 services. If you need to support a program or hardware device with a TSR under DOS or even Windows 95, there is likely to be a counterpart for Windows 2000 that runs as a service.

Disk Utilities

Hard disks need ongoing care to keep them running smoothly and happily. This is still true under Windows 2000, but because of the way Windows 2000 interacts with your computer’s hardware, including hard disks, you can’t use your old disk utilities from DOS or Windows 3.1. Windows 2000 provides its own set of tools for such purposes, and there are third-party, 32-bit utility programs developed to provide even more features than the ones that come with Windows 2000. Just be sure that the utility you choose is compatible with the file systems you are using. Many of the utilities that were designed for Windows NT 4 can still be used under Windows 2000, but you may find that the built-in utilities work fine.

Windows 2000 includes new utilities for such tasks as defragmenting your hard disks and checking for errors. The defragmenter utility is a new addition to Windows 2000, something that you would have to have purchased separately in NT 4.

Memory Utilities

In the world of DOS, it was a constant battle to work around the 640K memory limit, as it rapidly became apparent that increasingly sophisticated software needed increasingly more memory. The most common DOS utilities were frequently packaged together as programs, such as QEMM, that worked as TSRs and enabled programs to load into memory regions beyond 640K. Windows 3.x was able to take advantage of these programs as well. Windows 2000, however, uses a completely different memory system—a “flat” memory model—and can accommodate far, far more memory than 640K. In fact, there is almost no practical limit on the amount of memory that Windows 2000 can use, so there is no need for memory management utilities.

In theory, you can install up to 4GB of Random Access Memory (RAM) in Windows 2000 Professional, although very few of us will approach this limit for a few years at least. This limit is because of a hardware limitation on the PC-compatible platform. Windows 2000 itself should easily be able to address more memory than that, once the hardware platforms provide support for larger amounts of memory.
