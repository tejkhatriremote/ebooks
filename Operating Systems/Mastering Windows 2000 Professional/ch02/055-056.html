Special Topic: Running Programs in Dual-Boot Mode with Windows 9x

If hard-drive space is not an issue, you might do well to consider running your system in dual-boot mode, with both Windows 2000 and Windows 9x, because doing so lets you take advantage of the strengths of both operating systems. For example, you could switch to Windows 9x to play the games that Windows 2000 won’t permit and then switch to Windows 2000 to use your word processing program to write about the games in your award-winning hint book. Another use for this arrangement would be to check out how well your favorite applications run under two different systems: You could run them first under 9x and then switch to Windows 2000 and see if it really makes any difference to the way they work.

Unfortunately, it takes a little work to get the programs you installed under Windows 98 operating and running smoothly under Windows 2000. The main obstacle to being immediately able to run all your preinstalled programs is that you now have two system folders. In general, Windows system folders contain all the .dlls and other essential program files that enable your programs to run in the first place. Windows 2000 and Windows 9x (and Windows 3.1, for that matter) each have separate system folders. They are set in the path statements for each operating system. The problem is, if you’ve installed Windows 2000 on a system that was already running Windows 9x, all your previously installed applications have their .dlls in your Windows 9x system folder. When you try to run those same programs under Windows 2000, Windows 2000 looks for them in its own system folder and of course can’t find them there.

The quick solution to the system folder discrepancy problem is to add your Windows 9x system folder to your Windows 2000 path. By doing that, Windows 2000 looks in both its own and Windows 9x’s system folders. The procedure is discussed in Chapter 20, “Installing Windows 2000 (and Automating Installation).”

But the best way to have programs available to you in both Windows 9x and Windows 2000 is to reinstall the programs in Windows 2000. This isn’t as bad as it sounds. You should choose to install the program in the same folder where it is already installed. Setup programs won’t overwrite any data; they just copy the files that are missing. In the case of the application files, none are missing in the program folder, but all are missing from Windows 2000’s system folder. By reinstalling the application, you are installing the system files Windows 2000 needs to run the application and making the correct Registry entries so the program will run properly.

WARNING:  If you’re jumping ahead of the discussion here, be careful: Don’t copy the contents of one system folder into another. Despite the similar natures of Windows 2000 and Windows 9x, they often use different versions of the same files. Replacing even one file with the wrong version can prevent either operating system from running with full functionality, or perhaps even booting at all! Turn to Chapter 20 to read more about the process.

Summary

Running all your favorite applications (and even the programs you don’t like) is easier under Windows 2000 than in previous versions of Windows. More than being easier, it’s more convenient. You now have a variety of ways of running and opening files, and with shortcuts, you can put the buttons that you click right where you want them.

Even though Windows 2000 is a 32-bit operating system, it still runs 16-bit Windows and DOS programs as well or even better than Windows 3.1 or DOS. As you’ll recall, this is because Windows 2000 uses a separate memory space for 16-bit Windows and DOS programs, protecting them from each other and keeping your operating system safe from crashing programs. Remember, if a program does go errant on you, you can put it out of its misery with a swift Ctrl+Alt+Delete key combination.

In short, Windows 2000 enables you to run your programs—almost all of them—better, faster, and more easily.
