Sending and Receiving Data

After you’ve made a successful connection, you can begin to transfer data between the two computers. What you do now depends entirely on what the other computer expects from you. If you are calling an information service, BBS, or a mainframe computer, you typically have to sign on to the remote system by typing your name and possibly a password. If you are calling a friend’s or associate’s computer, you can just begin typing whatever you want to say. In any case, after the initial connection is made, there are several ways to begin transferring data between the two computers. The next sections describe these techniques and how to use them.

Communicating in Interactive Mode

The simplest way to communicate information is directly from your keyboard. As mentioned earlier, after you’re connected to the other computer, everything you type is automatically sent to that computer. Conversely, characters typed at the other computer are sent to your computer and show up on your screen. Sending and receiving data this way is called working in interactive or terminal mode. Communications sessions often begin in terminal mode, with each person typing to the other’s screen.

Terminal mode is often used, too, when connecting to many of the information services and e-mail services that are interactive. With these, you type certain commands to the host computer, and it responds by sending you some data. Understanding How HyperTerminal Displays and Stores Text

As you receive or type each new line of text, the cursor moves down a line in the HyperTerminal window. After the cursor reaches the bottom of the window, each new line shifts the previous text up so the oldest line disappears from view. This is just as text behaves in a word processing program such as WordPad (see Chapter 9, “Using WordPad for Simple Word Processing”).

But the older text isn’t gone forever. HyperTerminal remembers the last 500 lines in its backscroll buffer. To see text that has scrolled off the screen, use the vertical scroll bar or the standard cursor-movement keys. Then scroll back down to continue interacting with the other computer. (You can scroll through previous text when you’re not connected, too.)

Capturing Text

At times you’ll want a way to save data you see on your screen while you’re working in terminal mode so you can work with it later. You can capture incoming text at any time during a communications session with the Receive Text File command on the Transfer menu and save it in a disk file for later reading, printing, or editing. Here is the basic procedure for capturing text:

1.  Choose Transfer Capture Text.
2.  A small dialog box appears, asking you to name the file in which you want the captured text stored. Type the name (you can use the Browse button to select a new directory) but don’t press Enter yet.
3.  Click OK. The file opens. If the status bar is visible, the “Capture” message becomes highlighted, indicating the capture is in progress.
4.  Continue with your session. All incoming text, along with whatever you type as well, is captured in the file you chose. When you want to stop capturing text, choose Transfer Capture Text again to display a new submenu (available only when a capture is in progress). Here, you should choose either

•  Stop, to close the file.
•  Or Pause, to temporarily discontinue capturing text while leaving the file open for more. To resume the capture, choose Transfer Capture Text Resume.

By the way, after you stop a capture, you can add or append text to the same file by simply restarting the capture process and specifying the file again. If you don’t want the new text to go into the previous capture file, you must enter a different filename when you restart the capture process.

Capturing Selected Portions of Text

During text capturing there may be sections of text you don’t want to save interspersed with portions you do. There are two ways to selectively capture portions of the text that appear in the HyperTerminal window:

•  You can use the Pause feature of the Capture command to stop the capture process whenever you’re receiving information you don’t want to keep. When you expect more “good” information, use Resume.
•  You can dispense with the Capture command and just allow HyperTerminal to store the text in its backscroll buffer. You can then scroll through the buffer to copy the information you want to keep into a document in a text-editing program such as Windows Notepad. See “Manipulating Text in the HyperTerminal Window” later in this chapter for details.
