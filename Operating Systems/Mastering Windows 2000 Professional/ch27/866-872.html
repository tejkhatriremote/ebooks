LAN Cards

You need to take care of a few things before you install your network card. Make sure that what you do now doesn’t get you into trouble later. Now is the time to be sure that

•  Your system doesn’t have any interrupt request (IRQ) or input/output (I/O) address conflicts.
•  You’ve written down the settings for all your hardware.
•  The card works in a stand-alone mode.

Then install the driver for your card, as follows:

1.  Open the Control Panel and double-click the Add/Remove Hardware icon.
2.  Click Next to begin the wizard.
3.  Click Add/Troubleshoot a Device, and then click Next.
4.  Windows 2000 will search for any Plug-and-Play hardware you may have added to your system. When it is done, the wizard will display a list of devices you may want to troubleshoot. At the top of the list is an option to Add a New Device. Select that option and click Next.
5.  Select the first option to Search for New Hardware and click Next. This allows the wizard to search for all new hardware in the computer. Or, you could pick the second option, No I Want to Select the Device From a List, and specify the manufacturer and model of the network card as shown in Figure 27.16. When installing a network card you may be prompted to supply the TCP/IP networking information as shown in Figure 27.17.


Figure 27.16  Select your network adapter card’s make and model.


Figure 27.17  Microsoft TCP/IP Properties dialog box

Here you can set your computer’s IP address or let Windows 2000 automatically assign an address for you. If you know your computer’s address, fill it in here. However, if you’re not sure what your computer’s IP address should be, ask your network administrator. We know that’s not what you wanted to hear, but it’s better than screwing up your computer and who knows how many others. Don’t put in an address unless you’re absolutely sure it’s valid and not being used by another system.

If your network uses DHCP, select the DHCP option to automatically assign an address. Click OK to continue.

EIDE and IDE Adapters

Generally, if you have an IDE or EIDE hard drive, you’ll have Plug-and-Play support. After you configure your drives and install them on your system, Windows 2000 will automatically detect Plug-and-Play compatible devices.

The IDE driver, atapi.sys, controls all IDE and EIDE devices on a PCI machine. AT Attachment Protocol Interface (ATAPI) is the specification that defines the IDE drive interface. AT refers to the IBM/PC personal computer and its bus architecture.

If you have a hard drive that Windows 2000 won’t be able to detect, Windows 2000 may think that it’s a SCSI drive and use a SCSI driver for it. If you don’t have a SCSI adapter driver installed, install the SCSI adapter following the instructions in the “SCSI Cards and Devices” section earlier in this chapter. Reboot, and Windows 2000 should be able to detect your hard drive.

The ATAPI driver in Windows 2000 will also work with UltraDMA hard drives, though you will probably want to install the driver specific to your disk controller for maximum performance. Be sure that the driver was designed for Windows 2000, however, or you may be troubleshooting a blue screen later.

PCMCIA Adapters

Just like the tape driver, this driver is automatically detected when Windows 2000 is loaded. If you’re not sure if a PCMCIA adapter card exists on your system, check in Device Manager to see if there is an entry beneath PCMCIA Adapters as shown in Figure 27.18.


Figure 27.18  Device Manager shows the installed PCMCIA adapters.

In general, the PC Card entry in Device Manager is for monitoring the status of PCMCIA devices and changing the driver for the PCMCIA socket. Most PCMCIA card drivers cannot be configured using this method. You should always use the proper icon in Control Panel for changing a PCMCIA device. For example, if you are trying to change the settings for a PCMCIA network adapter, Windows 2000 will tell you to use the Network and Dial-Up Connections applet instead.

NOTE:  The terms PCMCIA and PC Card refer to the same type of devices. The terms are used interchangeably today.

NOTE:  If you want to change the PCMCIA socket settings, you can do so by double-clicking the entry in Device Manager and clicking the Driver or Resource tabs, provided that the PCMCIA driver supports this type of configuration. If you modify the settings this way, you’ll see a dialog box similar to the one shown in Figures 27.19 and 27.20.

Summary

In this chapter, you learned how to add hardware drivers in Windows 2000 Professional. We began by discussing the configuration of display adapters and showed you how to restart Windows 2000 safely if the incorrect driver is used by using the VGA mode start-up option or the Last Known Good configuration. We looked at installing support for sound cards, modems, and PCMCIA devices. We also talked about modifying the driver settings for your disk controllers.
