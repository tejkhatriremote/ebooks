[<SnmpParameters>]  [<SnmpParameters>] is used to list parameters for SNMP installation.

•  Accept_CommunityName = <community name> [, <community name>, <community name>]

Use this key to identify up to three community names that the SNMP service computer accepts traps from.

•  Send_Authentication = yes | no

Specifies whether an authentication trap should be sent when an unauthorized community or host requests information.

•  Any_Host = yes | no

Use this key to indicate whether the SNMP service computer should accept packets from any host.

•  Limit_host = <host name> [, <host name>, <host name>]

Up to three host names can be indicated. This key can only be used when Any_Host = no.

•  Community_Name = <community name>

Indicates the community name for the computer.

•  Traps = <IP addresses> | <IPX addresses>

Indicates that traps should be sent to up to three IP or IPX addresses.

•  Contact_Name = <name>

Indicates the computer user’s name.

•  Location = <computer location>

Indicates the physical location of the computer.

•  Service = Physical, Applications, Datalink, Internet, End-to-End

Indicates the type of SNMP service to be installed. Any combination of the above services can be listed as long as the elements are separated by commas.

[<RasParameters>]  The [<RasParameters>] key lists the parameters needed to install the RAS service.

•  PortSections = <PortSectionNames>

Defines the name of the port section.

•  DialoutProtocols = TCPIP | IPX | NETBEUI | ALL

Indicates the network protocols that RAS uses when dialing out. ALL indicates to install all three protocols.

•  DialinProtocols = TCPIP | IPX | NETBEUI | ALL

Indicates the network protocols that RAS uses when dialing in. ALL indicates to install all three protocols.

•  NetBEUIClientAccess = Network | ThisComputer

Default is Network.

•  TcpipClientAccess = Network | ThisComputer

Default is Network.

•  UseDHCP = yes | no

Default is yes.

•  StaticAddressBegin = <IP address>

This key needs to be listed if UseDHCP = no.

•  StaticAddressEnd = <IP address>

This key needs to be listed if UseDHCP = no.

•  ExcludeAddress = <IPAddress1 - IPAddress2>

Use this key if a range of IP addresses is being assigned manually. StaticAddressBegin and StaticAddressEnd must be also listed to use this key.

•  ClientCanRequestIPAdress = yes | no

Default is no.

•  IpxClientAccess = Network | ThisComputer

Default is Network.

•  AutomaticNetworkNumbers = yes | no

Default is no.

•  NetworkNumberFrom = <IPX net number>

Valid numbers range from 1 to 0xFFFFFFFE. You must use this key if AutomaticNetworkNumbers = no.

•  AssignSameNetworkNumber = yes | no

Default is yes.

•  ClientsCanRequestIpxNodeNumber = yes | no

Default is no.

[<PortSectionNames>]  [<PortSectionNames>] lists the parameters for the COM port you will use for RAS.

•  PortName = COM1 | COM2 | COM3-COM25

Indicates the port to be configured for this port section.

•  DeviceType = Modem

Implies the device type that RAS should install. As of this writing, the only available option is Modem.

•  DeviceName = <device name>

Indicates the name of the device being installed.

•  PortUsage = DialIn | DialOut | DialInOut

Indicates the dialing properties for the port named in the PortName key.

To run Setup in unattended mode for new installations, use the WINNT32 or WINNT command with the /unattend parameter and specify the location of an answer file. The WINNT32 command will work only with 32-bit operating systems such as Windows NT 4, Windows 9x, or OS/2. For 16-bit operating systems like MS-DOS or Windows 3.x, you should use the WINNT command instead.

The unattended setup command syntax is as follows:

   WINNT32 /unattend:<answer_filename> /s:<source>
•  /unattend indicates unattended Setup mode.
•  <answer_filename> includes the location of the answer file.
•  /s indicates that the installation files are located on a drive other than the CD-ROM. This parameter is optional.
•  <source> is the location of the distribution files.

A Sample unattend.txt file

Since that was a somewhat lengthy explanation of batch files, we’ll give a short example of what we mean. Here is a sample file that should enable you to perform an unattended installation using the bare minimum of parameters. This file will install Windows 2000 Professional on your machine and set up your display without installing the network. 

   [Unattended]
   Display = 0
   OemPreinstall = No
   NoWaitAfterTextMode = 1
   FileSystem = LeaveAlone
   ExtendOemPartition = 0
   ConfirmHardware = No
   NtUpgrade = No
   Win31Upgrade = No
   OverwriteOemFilesOnUpgrade = Yes
   TargetPath = *
   ComputerType = Retail

   [GuiUnattended]
   TimeZone = "(GMT-05:00) Eastern Time (US & Canada)"
   Display = 0

   [UserData]
   Display = 0
   FullName = "Mark Minasi"
   OrgName = "TechTeach International"
   ComputerName = P101
   ProductID = ""

   [Display]
   Display = 0
   ConfigureAtLogon = 0
   BitsPerPel = 8
   XResolution = 640
   YResolution = 480
   VRefresh = 60
   AutoConfirm = 1

   [Network]
   Display = 0
   JoinDomain = ORION
