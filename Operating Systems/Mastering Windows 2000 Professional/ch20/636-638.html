Creating a Windows 2000 Boot Disk

What will happen if one of your boot files is damaged or missing? Certainly, it will be impossible to load Windows 2000, and unless you have a copy of these files in your root directory, you’re out of luck. Fortunately, it’s relatively easy to rebuild the boot files.

To make sure that you never have this problem, you should create a Windows 2000 boot floppy. A boot floppy is a floppy disk that has the minimum operating-system boot files that you need to start a faulty system. Sometimes you can lose or damage an important boot file, which can prevent the system from booting. For example, one of us recently edited our boot.ini file and rebooted the machine. An error must have been made in editing the file because when the system came back up, this message appeared:

   Windows NT could not start because the following file was missing or corrupt:
   <winnt root>\system32\ntoskrnl.exe.re Profile/Last Known Good Menu
   Please reinstall a copy of the above file.

Because something in the file was corrupt, the system couldn’t locate the <winnt root> and thus couldn’t find the necessary file to continue the startup process. If a command is out of place or misspelled, the system won’t boot. As it turned out, boot.ini was the only file damaged on the workstation, so it would have been nice to have a floppy around that contained a valid boot.ini. That boot floppy would have been enough to start the workstation and then the hard disk would have taken over.

One good solution is to use an emergency repair disk, but we didn’t have one at the time. However, as stated earlier, an emergency repair disk is specific to a particular machine, but a bootable Windows 2000 floppy is generic. If you really want to be able to boot Windows 2000, you’ll make yourself a generic Windows 2000 boot floppy.

NOTE:  A Windows 2000 boot floppy will not boot the computer to an A: prompt. It will only serve to replace the files in the root of your C drive if they are missing. You must still have a bootable version of Windows 2000 on the computer.

Having said that, the following procedure explains how to create a generic Windows 2000 boot floppy:

1.  Format a floppy either using Windows 2000 Explorer or from a command line under Windows 2000. In Explorer, right-click the drive letter and then click Format.

Do not use a DOS-formatted floppy because a DOS-formatted floppy looks for the DOS boot files IO.SYS and MSDOS.SYS; a Windows 2000-formatted floppy looks for the Windows 2000 boot file ntldr.

You’re going to copy a few files from the root directory of your workstation to the floppy in the A drive. The files are hidden, however, so you have to tell Explorer to show the hidden files.

2.  Click View Options and click the radio button to Show All Files.
3.  Copy the following files from the server’s root in your workstation’s root directory to the floppy disk:

ntldr
ntdetect.com
boot.ini
ntbootdd.sys (Copy this file only if your workstation boots from a SCSI drive attached to a controller without an active BIOS; if not, then you probably won’t even have this file in the root directory.)

Use this boot floppy to bypass your problem files and start your operating system.

Installing Fonts on Windows 2000

Windows 2000 Professional contains only the most frequently used fonts. If you want to use fonts from other operating systems, such as Windows 9x or Windows 3.1, here’s a word of warning: Not all of the fonts that are available in Windows 9x are available in Windows 2000. Fortunately, Windows 2000 supports the most common fonts, such as Arial, Times New Roman, and Courier.

To install fonts, you need to

1.  Go to the Fonts directory in the Control Panel.
2.  Click File Install New Font to display the Add Fonts dialog box.
3.  Change directories to the path where the new font files are located. All of the available fonts for Windows 2000 will be listed, as shown in Figure 20.7.


Figure 20.7  Add Fonts dialog box

4.  Click Select All or select the fonts you want.
5.  Click Install Now.

TIP:  If the fonts are located on the domain, you may wish to create shortcuts to the files instead of copying the files to your Fonts directory. To do so, make sure that the Copy Fonts to Fonts Folder box is unchecked. If it is checked, the files will be put in your system directory.

If a font already resides in the Fonts directory, you won’t be allowed to reinstall it and you’ll receive a message stating that you’ll have to remove the old file before installing the new one, as shown below. If you receive this warning, click OK to skip past this file. Your new fonts should now be installed.
