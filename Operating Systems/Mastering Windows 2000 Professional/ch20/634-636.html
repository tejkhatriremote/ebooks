Obtaining Debugging Information

If you have started the boot process but are unable to load Windows 2000 after choosing your operating system, the problem may be with one or more of the drivers that Windows 2000 needs to run properly. If this is the case, you can find out which drivers failed during the boot by adding an /SOS parameter to the end of your Windows 2000 version line in the boot.ini file. This parameter enables you to view the drivers that Windows 2000 attempts to load during the boot.

TIP:  The /SOS parameter may be annoying if you use it all the time, but it’s helpful when trying to diagnose problems.

Another parameter you may wish to use is /MAXMEM=n, which enables you to fool Windows 2000 into thinking that it has less memory when you want to test your system’s memory. The range of values for n can be anywhere between 8 and 16 (the default). Reducing the value of n reduces the amount of physical memory and the total space for page files. The main result of changing this parameter is that applications run more slowly because of the increased amount of disk access by the Virtual Memory Manager, depending on the amount of memory available.

You can also add several other switches to your boot.ini file:

/NODEBUG
Does not collect or transmit debugging information collected during startup. This setting is the default.
/DEBUG
Enables debugging with a debugger such as I386KD. The last standard COM port found will be used unless a different port is specified using the /DEBUGPORT=COMx parameter.
/DEBUGPORT=COMx
Sends output to a specific COM port.
/BAUDRATE=nnnnn
Specifies the baud rate at which data is sent through the COM port; used for remote debugging purposes. The default for nnnnn is 19200.
/CRASHDEBUG
Enables a certain amount of stackable pages to be accessible via the debugger. For remote debugging, the default number of stackable pages is 9600. The pages are accessible only after the system encounters a fatal error and begins checking for bugs.
/NoSerialMice
Disables the serial mouse detection on certain COM ports. This switch excludes ports that are used by an Uninterruptible Power Supply (UPS) system because the detection signal causes some UPS units to power down, thereby turning off the computer. You can use multiple /NoSerialMice parameters. The format is /NoSerialMice- [COMx | COMx,COMy,COMz,…]. If /NoSerialMice is specified without parameters, serial mouse detection will be disabled on all the serial ports.

Booting Windows 2000 in Safe Mode

If you’ve spent any time working with Windows 9x, you’re no doubt familiar with Safe Mode. Any time the system won’t boot due to a newly installed driver or minor hardware problems, you could choose to boot into Safe Mode, which would load only the drivers necessary to start the operating system. Most of the time, this would allow you to boot the computer and begin making repairs.

The great news for NT users is that Windows 2000 includes a Safe Mode boot option, too. To use Safe Mode on Windows 2000, simply press the F8 key when you see the boot menu. This presents a screen with the following options to choose from:

Safe Mode
Boots Windows 2000 with only the basic drivers needed to load the operating system. Uses basic drivers for mouse, keyboard, display (VGA), mass storage, default system services and drivers, and absolutely no network components. Creates a log file to track the progress of the system boot.
Safe Mode With Networking
As the name implies, this option loads the Safe Mode configuration, but also enables the network devices and services. Creates a log file to track the system boot.
Safe Mode With Command Prompt
This option loads Windows 2000 in Safe Mode, and after logging on, a command prompt is opened instead of the normal desktop. Creates a log file to track the progress of the system boot.
Enable Boot Logging
Boots Windows 2000 with all of the normal files and drivers, but writes their status to a log file at each step. This log file, like the log files in the Safe Mode options, is located in the Windows 2000 folder and is named ntbtlog.txt.
Enable VGA Mode
This option is an old favorite with NT users. The Enable VGA Mode boots Windows 2000 using a plain vanilla VGA 16-color 640×480 display driver for the highest level of compatibility with video cards.
Last Known Good Configuration
Boots Windows 2000 using the last Registry configuration that is known to actually work. Last Known Good is a way of undoing problems created by installing incorrect drivers, or possibly by Registry corruption. It will not, however, fix problems caused by corrupted or missing files.
Debugging Mode
This option boots Windows 2000 while sending debug information through a serial port to another computer. It’s useful in the case of a persistent Blue Screen or Stop error.
