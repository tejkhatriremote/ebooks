Changing Your Computer’s Name

Earlier in this chapter, we described the process to change your domain or workgroup membership using the Network Identification Wizard. In this section we’ll take a look at the other method of changing this information using the Network Identification tab of System Properties.

If you want to change your computer’s name, go to the Control Panel and then follow these steps:

1.  Click the System applet and click the Network Identification tab, as shown in Figure 20.3.


Figure 20.3  Network Identification tab

2.  Click the Advanced button.
3.  Change the name of your computer and press Enter. You’ll receive a message like the one shown in the Domain Username and Password dialog box shown in Figure 20.4.


Figure 20.4  Renaming a computer

4.  Click the radio button next to Workgroup and click OK. The dialog box should look like Figure 20.5.


Figure 20.5  Changing the computer from domain to workgroup member

5.  Reboot your computer and return to the Network Identification tab via the System applet in Control Panel.
6.  Click the Advanced button.
7.  Click the radio button next to Domain and enter the name of the domain. Click the OK button.
8.  Restart your computer for the settings to take effect.

The reason you have to change your workstation to a workgroup and then back to a domain is that you are changing the name of the computer on that workstation only. The domain still knows your computer by its old name, not the one that it has now. For your workstation to become part of the domain again, you have to remove the old name from the domain, change the name of the workstation, and then add the workstation, with its new name, to the domain.

The domain assigns invisible internal passwords to the workstations. The Primary Domain Controller uses these passwords to verify that a machine is who it says it is. As a result, if you reinstall Windows 2000 Professional on a machine that was already running Windows 2000 and you assign the same name to the computer that it was previously using, you’ll get the following message: “No domain controller was available to validate your logon.”

This warning is an essential security feature of Windows 2000. In essence, the domain’s reasoning is something like this: “Even though you say that you’re a machine named NTWS4 and I know of a machine named NTWS4, how do I know that you’re that machine?” Similarly, we use identification badges, numbers, or fingerprints to distinguish ourselves from one another and also to prove to others that we are who we say we are. For example, the only way we could distinguish one person named John Smith from another John Smith is to give each John Smith a unique social security number, driver’s license, and the like. Without these unique identifiers, we’d have no idea if someone was really who they said they were. The same goes for computers. A computer needs special identification in order for the domain to identify that workstation named NTWS4 from all other computers with the same name.

Because of these identification requirements, you have to follow three steps to reinstall Windows 2000 Professional on a computer:

1.  Shut down the computer on which you are going to reinstall Windows 2000.
2.  Log on the domain as an administrator, run the Users and Computers console, and delete the to-be-reinstalled workstation from the domain.
3.  Reinstall Windows 2000 Professional, specifying that you are not upgrading but rather are replacing the Windows 2000 software on the machine. In the process of reinstallation, you are given the chance to create a new machine account on the domain. Create the new machine account, and you are up and running.

Deleting a Workstation Account from a Domain

In the course of writing this book, we discovered many ways of how not to do things. One problem we had was figuring out how to delete a workstation from the domain while the workstation was still connected to the domain. We thought, “Oh, well. Maybe we can just create a new account with the same name.” Using the Users and Computers console, we created the new machine account, but neither the workstation nor the domain could recognize one another. You can guess why this message appeared. Instead of assigning the old machine account to the workstation, the domain created an entirely new account with the same name. When we then tried to access the domain from the old workstation account, we were turned down because the domain didn’t recognize the old domain account.

NOTE:  The difference between reinstalling a workstation to a domain and deleting a workstation account from a domain is that when you reinstall a workstation, the domain re-creates the machine account for the workstation under the same name. However, when the machine account is deleted while a workstation is using it, the domain has to create an entirely new account because the workstation is still using the old one.

The process for restoring a machine account is very similar to changing the computer’s name. If you have deleted a workstation account from the domain before shutting down your workstation, you need to do the following in the domain:

1.  Go to the System applet in the Control Panel and click the Network Identification tab.
2.  Click Advanced.
3.  Click the radio button next to Workgroup and click OK.
4.  Reboot and go back to the System applet.
5.  Click the Network Identification tab and click Advanced.
6.  Click the radio button next to Domain and input the name of the domain. Click OK.
7.  Reboot, and your workstation should be back as part of the domain.

Repairing Windows 2000 Professional

If you encounter a problem with getting Windows 2000 to run properly, you may have some damaged system files, or perhaps you accidentally deleted or damaged a few crucial files and don’t feel like sitting through another half-hour installation. If this is the case, repairing Windows 2000 is much more convenient than reinstalling it because the repair process is fairly easy to complete.

Before attempting to repair Windows 2000, make sure that you have two things: a Windows 2000 CD-ROM and an emergency repair disk. (Now is the time to use the emergency repair disk you made when you first installed Windows 2000.) What, you didn’t make one? Well, you can also use the Backup utility on your workstation to create a new Emergency Repair Disk. See the next section on how to do this.

NOTE:  Keep in mind that the emergency repair disk will work only on the workstation from which it was created. The disk contains the configuration and Registry information for a specific machine. Therefore, it can’t be used to repair other workstations.
