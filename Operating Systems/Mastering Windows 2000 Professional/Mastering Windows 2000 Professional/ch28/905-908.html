Tips for Running Diagnostics

If you’re going to run diagnostics, make it a serious test. Most diagnostic tests of disks or memory have a slow mode and a fast mode. To do an exhaustive test of your PC’s hardware, choose the slow mode and be prepared to wait (perhaps all night for a good memory test to see if your SIMMs are working properly, or even a couple of days to test a 1GB+ hard disk).

A Final Memory Management Note for DOS Programs

Windows 2000 supports the MEM statement that you may be familiar with from DOS. The DOS MEM statement shows you the items that you have loaded in the first megabyte of memory and where these devices and TSRs have loaded in memory. Because of the memory requirements of many DOS applications, you will want to have more than 550MB of conventional memory free. As you look at the display of the MEM statement in the following printout, you will note that it is showing 582.5MB as the largest executable program size.

You won’t run into this barrier with 32-bit programs. They use the memory structure that Windows 2000 provides and can be up to 4GB in size (which is more than 4 billion bytes).

Summary

The issues that we have discussed in this chapter deal with how to manipulate the DOS environment to give maximum support to your DOS and Win16 applications. We looked at the control that you have over DOS applications via the PIF files, and we showed you how Windows 2000 supports Win16 applications by opening them up in a separate memory space. We concluded the discussion by looking at the memory management issues involved with running a DOS application under Windows 2000 Professional.

That’s about it for running legacy applications under Windows 2000. Now go grab that software and get to it!
