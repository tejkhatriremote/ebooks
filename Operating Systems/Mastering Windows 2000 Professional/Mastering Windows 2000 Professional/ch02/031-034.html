Even Some Windows 95 Apps Won’t Run

Yes, it’s true. The sad fact is that not all 32-bit Windows applications will run under Windows 2000. When Microsoft originally designed the requirements for creating Windows 95 programs, one of the specifications was that 32-bit applications that were designed for Windows 95 had to run under Windows NT as well. Microsoft later changed that requirement to state that programs don’t have to run on Windows NT—but if they don’t, they’re required to “degrade gracefully.” This means they aren’t allowed to freeze the operating system when they fail.

Additionally, many application programmers just plain haven’t figured out the correct way to write a 32-bit application, so they cobble together some of the 16-bit programs that they’ve already written and understand and wrap a bit of 32-bit code around it, calling the result a Windows 95 application. Before you know it, Windows 95 is out of memory and you end up with a Windows 95 application that doesn’t run under Windows 2000.

Finally, although Windows 9x certainly isn’t the benchmark of quality for operating systems, it has one great strength when compared to NT and most other operating systems: its hardware support. Windows 9x’s Plug-and-Play compatibility makes installing new hardware considerably easier than it ever was; NT traded that strength for its own strength of providing security against invalid hardware access. Windows 2000 has attempted to merge the strengths of both Windows 98 and NT into one operating system. It provides both the stability of NT when supporting applications, and the ease of Plug-and-Play support for installing hardware.

NOTE:  Not all 32-bit Windows 95 programs will run on Windows 2000. Be sure to ask if an application is NT and/or Windows 2000 compliant before you buy.

Running Programs

When you’re using NT, the question isn’t what program to run, it’s deciding how to run it. Under Windows 3.x or the previous versions of NT (version 3.5x) there were two ways to run an application: You either double-clicked the program icon, or you typed the program name on a command line. Both of these approaches work under Windows 2000 as well, but now you have many more options.

Before we go any further, though, you must familiarize yourself with the main tools that Windows 2000 provides on its Desktop because these tools offer you the options that we present in the rest of this chapter.

But First…The User’s View of the Desktop

In previous versions of Windows, Program Manager was the Desktop, for all practical purposes. Sure, you could minimize Program Manager and look at the Desktop behind it, but you couldn’t do anything with the Desktop except make it look pretty. In Windows 2000, the Desktop has become a fully integrated and essential part of the operating system. You can drag programs, files, and system objects onto it in order to organize them or launch them. (And, as in the old days of Windows, you can also customize the way the Desktop looks.)

In this overview section, we present the Desktop tools that every end user should learn to use. These are your interface tools; they are fundamental to your work with Windows 2000. For more detailed information on advanced and custom options available with any of the new Windows elements listed in this section, please see Chapter 4, “Setting Object Properties.”

TIP:  If you already know Windows 98, you’ll have an advantage when you begin to navigate around a Windows 2000 Desktop. The Taskbar, the Start button, Explorer, and My Computer are four elements that should look familiar. But when you open the Control Panel, it will look very different.

Taskbar

Located at the bottom of your screen (you can move it anytime you want to), the Taskbar is Windows 2000’s information system. It constantly tells you what applications and files you have open, and whether you are printing, have e-mail, or have sound. Most important, the Taskbar (see the following illustration) is the permanent home of the Start button. Click this button to pop open the Start menu, discussed in the next section.

Start Menu

As the name implies, everything on your computer can be initiated from the Start menu. Go to the Start button to pop open the Start menu, and you can launch any program, configure your computer, and access your most recently used documents, all with a click of your mouse—and all without ever stopping the program you are using. The Start menu is also quite customizable: You can drag icons for your most frequently used applications or files to the Start button, which turns them into menu items that you can click. Right-Click Menus

Throughout the work you’ve done with Windows, you may have noticed that you have two mouse buttons and that most of the time the right one (well, the one that isn’t set up as your primary button) is useless. Windows 2000’s new interface rectifies that situation by assigning the right-click action to pop up some kind of context-sensitive menu for almost every file, folder, drive, or any other object you’ll encounter. Context-sensitive means that the menu is different depending on where you are in your task when you click the mouse button. Common commands include the ability to cut, copy, and paste, as well as the ability to send a document (using the Send To choice) directly to the printer or attach it to an e-mail message. In addition, almost every right-click menu offers a Properties choice, which is usually your fastest route to discovering the location of a program file in the path and folder hierarchy, and for adjusting the way the program or file behaves.
