Running Programs by Accessing the Files They Create

You don’t necessarily need to find a program to launch it. You can also start a program by opening a file or document that has been associated with that program. For example, if you want to launch Microsoft Word, you can double-click a Word document in Explorer or in the Search utility’s Results window. This opens Microsoft Word and also loads the file you double-clicked.

If you already have a program running that has been associated with the types of documents it creates, you have even more methods of opening its associated documents. From Explorer or from the list resulting from a Search operation, you can do whichever of the following seems most convenient:

Double-click the document icon
Opens the document.
Drag the document’s icon to the program’s menu bar
Opens the document in its own document window within the program.
Drag the document’s icon into the program’s workspace
Inserts the document into the currently open document. (Unlike the second approach, this action doesn’t simply open it into its own document window.)

TIP:  There’s even a fourth technique for opening a file from Explorer or a Search list. If you have a program active on the Taskbar but it’s not currently visible on the Desktop (perhaps it’s hidden behind Explorer itself), you can drag the document icon to the program’s button on the Taskbar and, without unclicking, hold it over that button until the program reappears on the Desktop. At that point, you can continue dragging the icon as you would using the second or third method above. (That is, drag to the menu bar to open the document, or drag to the program’s workspace to insert it into the currently open document.)

Special Topic: Running Programs with Shortcuts

The history of humanity has been spent searching for faster, easier ways to do things. That’s why we have microwave ovens, drive-through windows, and shortcuts. In Windows 2000, you can make a shortcut to any object that exists in the operating system—files, folders, programs, hard drives, and other computers on the network. Anything. You can even make shortcuts to other shortcuts!

The shortcut appears as an icon that you can click to open the associated file or resource. Because you can put a shortcut anywhere on the Desktop, or anywhere in an Explorer or Explorer-type list, the shortcut can save you a lot of browsing to find your favorite items. Taking a mere 1K of drive space on average, a shortcut is an economical way to set up quick access to your programs.

To create a shortcut from within an Explorer or Explorer-type list, do one of the following:

•  Right-click the object you want to create a shortcut to, and drag the object to the Desktop or to the folder or drive of your choice. When you let go of the mouse button, a menu pops up with various choices. Choose the option that says Create Shortcut(s) Here. The item (file, folder, resource, whatever) remains in its original location; the shortcut appears where you dragged it.
•  Copy the item you want to create a shortcut to, then select the folder in which you want the shortcut to appear, and choose Paste Shortcut from the Edit menu.
•  Select an object and choose Create Shortcut from the File menu, then drag or copy the shortcut thus created to the Desktop, or to the folder or drive of your choice.

Anything you can do to a file you can do to a shortcut. If you send a shortcut via e-mail to another MS Exchange Mail user, the shortcut will appear as an active shortcut in the e-mail message. The recipient will be able to double-click the shortcut and access the original file, or they will copy the shortcut to their Desktop or anywhere else in their computer. This is a great way to distribute information across a network!

Running Programs by Using Run

You can also use the Start menu’s Run command for launching applications. This works just like the Run command in Windows 98 or NT 4. When you choose Run, you’ll see a dialog box like the following:

If you’re like most end users, you don’t know or care where your application files are living on your hard drive, and you might not notice how they’re spelled. For you, the Browse… button was invented.Click the Browse button within the Run dialog box, and a mini-Explorer opens to enable you to search for the program you want, just as if you were searching for anything else on your hard drive. (The mini-Explorer is shown on the facing page.) After you find the file, launch it as you would in the real Explorer.

NOTE:  If you are one of those people who remember precisely where your files are and how they’re spelled, you could type the path and the name of the application you want to run in the Run dialog box. But you have to know the full path, meaning you have to know what drive and folder the file is in. For most people, it’s a lot easier to use the Run dialog box’s Browse button.

Here are some added benefits of the Run menu. First, in addition to using it for running applications, you can use it to open and explore the contents of any disk device attached to your computer. For example, if you have two floppy drives and you want to see the contents of your B drive, you can go to the Run menu, type B: and press Enter. The mini-Explorer window opens and shows you the contents of the disk or drive.

Second, the Run command remembers what it has done for you in the past. Each time you use Run, it adds every file, folder, or drive that you launch or open to its drop-down list. Let’s say you just ran your 3-D ray tracing program, either by typing the application’s path and filename or by using the Run dialog box’s Browse feature. You would never again need to type that program’s path or browse for it, because after the first time you run it by means of the Run tool, you can just choose it from Run’s drop-down list.

If you are familiar with the Run command from NT 4, you will remember the option to Run in Separate Memory Space. This option appears to be missing in Windows 2000, though the ability to start a 16-bit Windows application from the command prompt is still there. Hopefully this will be resolved by the time the finished product is released.
