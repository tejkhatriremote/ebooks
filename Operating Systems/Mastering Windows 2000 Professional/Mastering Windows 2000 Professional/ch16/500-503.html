Installing Peer Web Services

Microsoft’s Peer Web Services made their debut in NT Workstation 4. Certainly plenty of great Web Server software packages are available for NT, and the Peer Web Services were not touted as being on equal par with full-featured, heavy duty servers. But it remains to be seen how NT’s inclusion of Peer Web Services will affect the producers of other Web server software down the road. One of the greatest uses for Peer Web Services is to test your Web pages during development. Another possible use is to host a personal Web site on your company’s Intranet.

NOTE:  Peer Web Services (PWS) are actually a limited version of Microsoft’s Internet Information Server (IIS). In this case limited refers only to the number of inbound connections the server can support. PWS is limited to 10 inbound connections (the same as Windows NT Workstation). When you are ready to upgrade to newer versions of PWS, you apply exactly the same files used to upgrade IIS on NT Server.

Windows 2000 Professional includes Internet Information Services (IIS), which are identical to the services included in Windows 2000 Server, except that they are limited in the number of inbound connections that will be accepted. The services provided in IIS include the following:

File Transfer Protocol (FTP) Server
Provides the ability to act as an FTP server to receive or distribute files through the File Transfer Protocol.
FrontPage 2000 Extensions
This service enables you to manage your Web sites using Microsoft’s FrontPage 2000 or Visual InterDev authoring tools. With these extensions installed you can easily publish your Web content directly from the authoring tool to the Web server.
Personal Web Manager
This tool gives you a graphical interface to manage your Web site on Windows 2000 Professional.
Simple Message Transport Protocol (SMTP)
This gives Windows 2000 Professional the ability to act as a workgroup Internet mail server. Install this service if you want to use Outlook Express to send and receive e-mail within your workgroup.
Visual InterDev Remote Application Deployment Support
Enables the deployment of custom applications from your Web server.
World Wide Web (WWW) Server
Enables you to publish pages using the Hypertext Markup Language (HTML), or Active Server Pages (ASP).

Included with the Web server are several great-looking demo Web pages. Interestingly enough, the documentation for Peer Web Services is also presented as a set of Web pages, which you can view from any Web browser. Of course after the Web server is operational, you can even view this documentation from other stations on the network (assuming you are on a network).

As an aside, Microsoft announced a while ago that all their Help files are converting to HTML format. This has been implemented in Windows 2000, and has resulted in a much more convenient Help tool. When you consider how much easier HTML files are to create than Windows Help files, and how much more formatting and richness of graphics and other content can be incorporated using HTML, this is certainly a step forward and a welcome example of how everyone can benefit by adopting a well-designed and open standard.

Without further ado, let’s get IIS installed and running!

To Install the Internet Information Services, do the following:

1.  Go into Control Panel and start the Add/Remove Programs applet.
2.  Click Add/Remove Windows Components to start the Windows Components Wizard.
3.  Click Next to proceed to the second page of the wizard where you can select from the list of Windows components to install. Highlight Internet Information Services and click the Details button. The Details page for Internet Information Services is displayed in Figure 16.13.


Figure 16.13  The details for the Internet Information Services enable you to select individual services to install.

4.  Select the individual services you wish to install and click OK when you are satisfied. You can easily add or remove them later if you change your mind.
5.  Back on the Windows Components page, click Next to install the files for IIS.
6.  The Configuring Components page displays a progress bar as the files are copied into place and the Registry is updated. You may be prompted to provide the CD-ROM if the system needs a file that was not pre-copied to the hard drive during installation.
7.  When the file copy is complete, click Finish to exit the wizard. Click Close on the Add/Remove Programs applet to close it. IIS is now installed on your computer.

When your system restarts, click Start Settings Control Panel Administrative Tools, then double-click Internet Information Services. This application lets you start and stop (or pause and resume) Web and FTP servers on your system and on any other Windows 2000 systems to which you have been given Web-services administration access. It also lets you view and change service settings, logs of service activity, and directories used by each service.

If any of your services are not currently running (particularly the Web service), try starting them. You can easily do this by single-clicking a particular service and pressing the VCR-style play button (or by right-clicking the service to bring up a pop-up menu, and then selecting Start). After you get the Web server service running, try launching your Web browser. In the browser’s URL space, simply type the computer name for your station. No slashes or www’s are necessary. Press Enter, and the default page for your new Web peer server should be displayed. There will likely be a link to the online Peer Web Services documentation. Be sure to look at this. It is quite well done and covers everything from installation to security.
