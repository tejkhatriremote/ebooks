What’s Not in Windows 2000?

The following is a very short list of features not available in Windows 2000.

No MSN Support

The much-talked-about Microsoft Network (MSN) software that was a key part of Windows 95 is not available in Windows 2000. However, you can still use MSN as an Internet service provider (ISP), for use with Microsoft IE or the Web browser of your choice. Look for an MSN client to be released for Windows 2000 in the near future.

No Support for MIPS or PowerPC Chips

Windows 2000 has discontinued support for the MIPS and PowerPC processors. These platforms haven’t been very well represented in the marketplace, and the development effort put into the approximately 36 million lines of code for Windows 2000 demands that only the most prevalent systems be supported.

Why Choose Windows 2000 over Windows 98?

Given that their interfaces are so similar, it would be a fair question to ask why you should choose one over the other. The answer is pretty straightforward. Windows 2000 may look like Windows 98, but it does many things that Windows 98 can’t. For starters, Windows 2000 is a secure operating system. Just try and log on to a Windows 2000 computer without the correct password. You can’t do it. In Windows 98? Just press the Esc key and you’re in! The lesson here is, if you have data that absolutely, positively needs to be kept secret, Windows 2000 is the operating system for you.

As discussed elsewhere in this book, Windows 2000 is also the most complete 32-bit environment you can buy. If you need an operating system that won’t crash and won’t shut down on you, Windows 2000 is the one for you.

How different is Windows 2000 from Windows 98? Both use the same style interface; both are 32-bit operating systems. But after that, the similarity ends. Although they are both 32-bit operating systems, they implement their 32-bitness in much different ways. Table 1.1 lists the differences, which are explained below in greater detail.

HAL and Plug and Play

When Plug and Play was introduced with Windows 95, most users quickly accepted it as one of the best things since sliced bread. After years of tweaking features such as Interrupt Requests and Direct Memory Access with jumpers or switches, the concept of installing a new adapter card in your computer and having the operating system simply say “Hey, I notice you have a new card there…Would you like me to install and configure the driver for you?” was a huge step forward. Plug and Play depends on the operating system continuously polling the Basic Input/Output Services (BIOS) of the computer to determine what hardware is present. So for Plug and Play to work, your computer must have a Plug-and-Play BIOS to gather the basic information from the devices, such as which hardware resources they need. Plug and Play also depends on a Plug-and-Play operating system to know what to do with information from the devices. Windows 95 did that fairly well, as does Windows 98. The Plug-and-Play agents in Windows 9x poll the BIOS to see what hardware is present and what interrupts it needs, and then communicate that information back to the operating system to keep the hardware and software functioning in harmony. This process presumes a certain type of BIOS and a certain type of central processing unit (CPU), namely a Pentium processor or another member of the x86 processor family.

NOTE:  The term x86 is shorthand for the class of computers that are backward compatible with the Intel 8086 chip. These include computers using the 8088, 80286, 80386, 80486, or 80586 (Pentium), 80686 (Pentium Pro, or Pentium II or III) chips. For the sake of this discussion, you can also assume that x86 refers to the Intel-compatible CPUs from manufacturers such as Cyrix and AMD.

Unfortunately for Windows NT users, Plug and Play wasn’t so easy to implement in NT. Windows NT uses the Hardware Abstraction Layer (HAL) to provide an interface between the operating system kernel and the hardware in the computer. The HAL is what makes it possible to port NT to so many different computer types, so it’s a good thing to have, but the downside is that makes adding Plug and Play to the operating system very difficult.
