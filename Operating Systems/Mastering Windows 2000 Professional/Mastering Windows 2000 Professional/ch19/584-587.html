Creating User Groups

Under the Groups folder in Computer Management Local Users and Groups, you see the user groups that this machine’s SAM contains. Each user group has different functions that it can perform. The following are the default groups that any Windows 2000 Professional machine contains:

Administrators
Can do just about anything to the computer. The things that they can do that no other type of user can do include loading and unloading device drivers, managing security audit functions, and taking ownership of files and other objects.
Backup Operators
Can log on to the computer and run backups or perform restores. You might put someone in this group if you wanted him to be able to get on your system and run backups, but not to have complete administrative control.
Guests
Have very minimal access to network resources.
Power Users
Have a few more capabilities than do regular users. They can create new printer and file shares, change the system time, force the system to shut down from another system, and change priorities of processes in the system. They can’t run backups, load or unload device drivers, or take ownership.
Replicator
Enables your computer to receive replicated files from an NT Server machine. We won’t discuss that process here, as you can’t do it without a Windows 2000 Server PC.
Users
Can run programs and access data on a computer, shut it down, and access data on the computer from over the network.

Adding or deleting someone from a group is fairly simple. Just double-click that group, and you’ll get a list of who’s in it. You can then add or delete someone from the group through the graphical user interface.

Understanding User Rights

But what’s this about shutting down the machine or loading and unloading drivers? Well, actually, the notion of a user right is an integral part of how Windows 2000 security works. Basically, the difference between regular old users and administrators lies in the kinds of actions that they can perform; for example, administrators can create new user accounts but regular old users cannot. In Windows 2000 terminology, the ability to perform a particular function is a user right. Most user rights are pretty self-explanatory, but a few can use some clarification. User rights in Windows 2000 Professional are found in the Local Security Policy console in Administrative Tools. Once you’ve opened that console, expand the Local Policies User Rights Assignment console tree, and then double-click the right you want to change.

If you have the right to access this computer from the network, then you can use data on this computer from another computer on the network (provided, of course, that you have access to that particular piece of data). You’ll note that Everyone—that’s the name of a special pre-created group that includes, well, everyone—can access data from this computer over the network. Here’s a list some of the rights and, where necessary, what they mean and what they’re good for:

Back Up Files and Directories
Run backup utilities.
Change the System Time
For some reason, not just anybody can change the system clock; it’s a right. (Of course, you could always reboot the computer in DOS or go straight to the setup program in CMOS to reset the time, so it’s not a very airtight security feature.)
Force Shutdown from a Remote System
Some utilities let you pick a Windows 2000 machine and force it to shut down, even though you’re not logged on to that machine. (One such utility comes with the Resource Kit.) Because you wouldn’t want just anybody doing a forced shutdown, Microsoft made this a right.
Load and Unload Device Drivers
A device driver is not only a video driver or SCSI driver; a device driver may be part of a software application or operating system subsystem. Without this right, you’ll often be unable to install new software and you’ll usually be unable to change drivers or add and remove parts of the operating system.
Log on Locally
Sit down at the computer and log on.
Manage Auditing and Security Log
You can optionally turn on a Windows 2000 Security log, which will report every single action that woke up the SRM, the LSA, or any other part of the security subsystems in Windows 2000. You can learn more about the log by clicking Policy Auditing. In general, we don’t recommend using the Security log because the output is quite cryptic and can be huge; logging all security events can fill up your hard disk quickly, and the CPU overhead of keeping track of the log will slow down your computer. You can’t enable any security logging unless you have this right.
Restore Files and Directories
As the name states.
Take Ownership of Files or Other Objects
If you have this right, then you can seize control of any file, directory, or other object even if you’re not supposed to have access to it. This right is obviously quite powerful, which is why only administrators have it.

NOTE:  The user right to Take Ownership of Files or Other Objects is the secret to the administrator’s power. You can do whatever you like to keep an administrator out of your data, but remember that the administrator can always take ownership of the file, and as owner do whatever they want to the file including changing permissions. You cannot keep an administrator out; you can only make it difficult to get in.
