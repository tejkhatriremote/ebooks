Creating a User Account

Let’s return to creating a new account. You’ll create a user account for one of the authors of this book, an account named Mark. Before you do, however, here’s a reminder: If you use a Windows 2000 domain, do not create a local user account. You don’t need to have a Mark account in both the domain’s SAM and the local machine’s SAM.

Open Control Panel and double-click the Users and Passwords icon to display the dialog shown in Figure 19.6. To create a new user account for your Windows 2000 Professional computer, do the following:


Figure 19.6  The Users and Passwords dialog box

1.  Click the Add button. The Add New User Wizard starts, prompting you to enter the username to use for the new account, and optionally, the domain in which the account resides. If you don’t specify a domain here, you will be creating a brand new account in your local SAM database. Click Next.
2.  On this tab, you must decide which type of user account you’re adding. This is really setting the group to which the account will belong. The options include the following:

•  Standard User These accounts will be members of the Power Users group. This account will be able to modify the computer by installing new software and changing system settings.
•  Restricted User This type of user can run programs on the computer, but cannot install new software. A Restricted User account is a member of the Users group.
•  Other This setting enables you to select from the whole list of groups in the SAM. It includes the above options, plus Administrators, Guests, Backup Operators, and Replicator.

3.  After selecting the group, click Finish to complete the wizard and create the user account.

To gain more control over the process of managing user accounts on Windows 2000 Professional, you will need to use the Local User Manager. This tool can be accessed two ways. First, you can open Control Panel and double-click Administrative Tools Computer Management, then expand System Tools Local Users and Groups as shown in Figure 19.7. The second way is to open the Users and Passwords applet in Control Panel, and click the Advanced button on the Advanced tab. Sounds a little redundant, doesn’t it?


Figure 19.7  You can use the Management Console to manage Users and Groups.

To add user accounts to Windows 2000 Professional using the Computer Management tool, use the following steps:

1.  Open the Computer Management icon in Control Panel Administrative Tools.
2.  Expand the System Tools icon by clicking the plus sign to the left of the icon, then expand Local Users and Groups.
3.  To add a new user account, right-click the Users folder under Local Users and Groups and select Create User from the context menu to open the Create User dialog shown in Figure 19.8.


Figure 19.8  The Create User dialog lets you define the properties of a new user account.

4.  Type a unique user name in the space provided. The user name can be up to 20 characters in length and cannot contain the following symbols: " / \ [ ] : ; | = , + * ? < >. You can use any combination of uppercase or lowercase characters or numbers for the name, but it must be unique to the database where you are creating it.
5.  Enter the person’s full name in the space provided. This becomes very important in large domains, but is less vital on a Windows 2000 Professional computer, as you are not as likely to encounter multiple people with identical or similar names. The full name enables you to distinguish between people with similar names.
6.  The next field is the Description, where you can enter a string of text that describes where the person works, who they are, what type of account this is, or other data that helps to uniquely identify this account from others on the same computer.
7.  In the Password area of the dialog, enter a password that is up to 14 characters in length. The password may contain any combination of uppercase or lowercase letters, numbers, and extended characters. The password is case-sensitive, so be very careful with the Caps Lock key on your keyboard when entering the password. When you have entered the password you want to assign to the account, enter it again in the Confirm field.
8.  Set the password options. The default option is User Must Change Password at Next Logon. This option means that only that user will know the password, which means better security. If you uncheck this option, the other two options become available. Select User Cannot Change Password if this account will be used for a service, or for someone that you do not want to give the ability to change their own password. Select Password Never Expires if this password should be considered “permanent” and not have an automatic expiration.
9.  The final option is to decide whether the account should be disabled. This is often a good idea if you want to change other properties of the account before it can be used, such as setting permissions on files and folders that this user will use. If this is the case, check the Account Disabled box.
10.  When all options are selected, click Create to complete the process of making the new user account.
