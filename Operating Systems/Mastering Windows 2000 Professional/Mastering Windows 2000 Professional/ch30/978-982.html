Because Log view shows only one data point at a time, you can change the data point by bringing up the Time Range slider in the Properties for the report, and changing the start point. Using Report View

Very similar to Chart view, Report view (see Figure 30.15) shows up-to-the minute statistics on different object counters but displays them numerically rather than graphically. If you want to watch many counters simultaneously, Report view is your best bet because Chart view gets to be a cluttered mess when you try to display too many counters simultaneously.

Running Users and Passwords

The Windows 2000 peer-to-peer networking features let you turn your PC into a “mini-server,” with which you can assign specific access rights to users and groups on your system. Of course, if your network administrator has configured your local access so that you don’t have administrative rights on it, so much for your fiefdom. But let’s assume that you’ve been given administrator-level privileges over your own system and that you want to configure it to allow close coworkers access to special subfolders that contain files for a collaborative project.

Click the Start button, and then choose Settings Control Panel Users and Passwords. Windows 2000 displays the Users and Passwords dialog box, as shown in Figure 30.16.


Figure 30.16  The Users and Passwords applet lets you create accounts for all users logging on to your system locally or from the network.

Windows 2000 sets up two accounts by default: Guest and Administrator. A guest can log on only locally (working at your machine); an administrator can make changes to your system configuration, load and unload device drivers, and even shut down your system—all from a remote location.

Before changing either of the default accounts or creating new user accounts, it’s a good idea to create a trapdoor account that has administrator-level privileges. On the outside chance that you forget the password for your default logon ID, you can use this trapdoor account to gain access to your system with Administrator privileges and rebuild your default logon user ID. To create a trapdoor account, follow these steps:

1.  On the Users tab, click the Add button. Windows displays the Add New User Wizard, as shown in Figure 30.17.
2.  In the User Name text box, enter a unique name, such as Emergency, and assign a password. Be sure to use a password that you can easily remember.
3.  In the Domain text box, enter the name of your local computer. This field is used to determine the location of the user account. You would type in the name of a domain if you want to create a local account for a domain user. If you are creating a local user account that won’t exist anywhere else, type in the name of your computer. Click the Next button.


Figure 30.17  Creating a trapdoor account may save you from being locked out of your system.

4.  Next, determine what type of user account you are creating. The three options are Standard User (Power Users group), Restricted User (Users group), or Other. For the trapdoor account, click Other and select Administrators from the list box. Click Finish to complete the process.

After you have clicked Finish to create the account, you will be able to double-click the account name in the Users and Passwords dialog box to open the Properties for that account. Once the Properties are displayed, you can set the password for the trapdoor account. Now that you’ve created this account, don’t touch it! You’ll always have an extra entrance into your system should you need it.

With Windows 2000 Professional, you can enable or disable access rights for individual users or for groups. By default, Windows 2000 has six user groups; you can, however, delete them, modify them, and add other groups with their own specific access rights. The six default groups are

•  Administrators
•  Backup Operators
•  Guests
•  Power Users
•  Replicators
•  Users

Table 30.6 lists the basic rights, what they do, and which groups have these rights.
