Windows 2000 Professional comes with new administrative tools for you to use when you back up your hard drive, add new disk volumes and check on existing ones, track system events, watch system performance, add new users and change attributes of existing users, and run general diagnostics. Let’s take a look at these tools and how they can help you keep your system running smoothly.

Running Backup

To start the Backup application, click Start, and then select Programs Accessories System Tools Backup. Windows 2000 will display the Backup window, as shown in Figure 30.1.


Figure 30.1  The Backup window

The opening tab of the Backup program gives you a choice between three wizards to make Backup operations easy. You can use the wizards to create backup and restore jobs as well as Emergency Repair Disks (ERDs). Backup is the new location for this last wizard. If, however, you are familiar with backup programs, you can create them directly in the graphical interface. Just select the Backup, Restore, or Schedule Jobs tab to go directly to the function you need. The graphical interface still supports command-line operations for use in batch files. Unlike its NT predecessor though, Windows 2000 Backup can back up to any supported device, including floppy disk, hard disk, network drive, tape drive, or even CD-ROM.

Running Disk Management

In Windows NT, you would use the Disk Administrator tool to manage disk partitions and handle all of your formatting chores. In Windows 2000, the corresponding tool is Disk Management, which is part of the Computer Management console in Administrative Tools. To reach this tool, click Start, select Settings Control Panel Administrative Tools, then double-click the Computer Management icon. Expand the Storage console tree by clicking the plus sign to the left of the Storage icon. Click Disk Management, and you will see the view shown in Figure 30.2.


Figure 30.2  You use Disk Management to create and format partitions and striped volumes.

The Disk Management tool, which is displayed in the right pane of the Computer Management console, will give you a graphical representation of the hard disks and CD-ROM devices in your computer. However, Disk Management has a variety of views, so you can customize the way information is presented to you.

NOTE:  There is an easier way to access Administrative Tools. If you are going to be using this group frequently, click Start Settings Taskbar and Start Menu. On the Advanced tab of the Taskbar and Start Menu properties sheet, click the checkbox beside Display Administrative Tools in the Start Menu Settings list box, then click OK to make the change. The Administrative Tools group will now be displayed under Programs on the Start menu.

The Disk Management console is broken up into three parts. On the left is the Tree view of the Computer Management console. The lower-right pane of the display contains the information for all of the physical disks installed in the computer, excluding floppy drives. The upper-right pane contains descriptions of those disks, including data on the amount of space used and free, the type of file system, and the health of that system.

To use Disk Management to create or delete partitions, format drives, and create stripe or volume sets, simply click the disk that you wish to modify, and select the change that you want to make from the Action menu. Or, you can right-click the drive that you want to modify, and then select the appropriate action from the Context menu.

Deciding Which File System Is Best

Windows 2000 can use the File Allocation Table (FAT) file system supported by DOS and Windows 95, the File Allocation Table 32 (FAT32) file system supported by Windows 98, and the New Technology File System (NTFS) supported by Windows NT. Each has its respective upsides and downsides. Here’s a quick rundown.

If you dual-boot your system, you need to use the FAT file system because DOS and Windows 95 cannot read a partition formatted using NTFS. FAT’s main advantages are its backward-compatibility with DOS and Windows 95 and its slightly-faster overall performance, which is due, in large part, to its lack of security features. Security features, though useful and at times invaluable, do introduce overhead that can slow disk performance. Moreover, unlike previous versions of Windows NT, FAT supports long filenames in version 4 and in Windows 2000.

FAT’s main disadvantage is that it’s a 16-bit file system with a minimum cluster size of 32KB. The result is that FAT can address partitions only up to 2GB. If your drive is larger than this, you’ll have to create two partitions to use FAT. Also, FAT has no security features. As a positive note, under Windows NT, the FAT file system has a maximum file and partition size of 4GB.

The FAT32 file system evolved from the traditional FAT file system used by DOS and Windows. One of the big benefits of having FAT32 support added to Windows 2000 is that you can easily dual-boot between Windows 98 and Windows 2000 on the same partition. This file system also allows for much larger volumes—it supports volumes of as much as 16GB in size—with smaller cluster sizes than FAT, yet it doesn’t support as large a volume size as NTFS. Another downside to FAT32 is that, like FAT, it does not provide any local security.

In contrast, NTFS is a robust, 32-bit file system with many security features, including password protection for specific files, the capability to limit access to specific files, file compression, file recovery, and the capability to address very large partitions. Although a system using NTFS will run a bit slower than one using FAT in some cases, NTFS is your best bet if you want Windows 2000’s additional features.
