Dynamic volumes come in various flavors:

Simple
A Simple dynamic volume is very similar to a traditional partition in that it exists on a single drive. It can be resized by extending the volume if it is on a dynamic disk that was created as a dynamic disk and not upgraded from a basic disk.
Spanned
This dynamic volume spans two or more disks to create one logical volume, as its name implies, but it can be extended. Spanned volumes are the same as volume sets in NT 4, but in Windows 2000 they are only available on dynamic disks.
Mirrored
This volume type is only available on Windows 2000 Server. Mirrored volumes provide fault tolerance by keeping an exact mirror image of all data in the volume. If you lose one disk, you still have an exact copy of all of the data.
Striped
Striped volumes are the fastest type of dynamic volume and yield great benefits in performance. They are not fault tolerant, however. If you lose one disk, you lose all of the data in the volume. They are the same as the traditional stripe sets available in NT, but stripe volumes are only available in Windows 2000 on dynamic disks.
RAID 5
This type of dynamic volume is only available in Windows 2000 Server. RAID 5 is also known as striping with parity. RAID 5 allows the volume to lose one hard disk without failing. It does this by combining the remaining data with the parity information to recreate the data on-the-fly.

Working with Disks and Volumes in Windows 2000

Before we go on, let’s look in more detail at three specific Windows 2000 features we’ve already mentioned—file compression, security, and scanning for bad sectors—and then at a new feature in Windows 2000—a hard-disk defragmenter.

Using File Compression in NTFS  One cool feature of the NTFS file system is its compression capability. Windows 2000 can compress entire volumes, individual subfolders (also called subdirectories), and files. The compression ratio usually winds up being about 1.5:1, or 33 percent. For example, Windows 2000 would compress a 100MB file to about 67MB. Generally, it’s better not to compress files that contain multimedia data because more often than not, they’re already using their own compression algorithm (or codec, see Chapter 7, “Running Multimedia Applications”). Also, because multimedia files are played back in real time (at a specified data rate rather than as fast as possible), the added overhead of NTFS decompression may adversely affect audio or video quality, causing breaks in the audio or frames to be skipped in the video.

Database files are probably also best left uncompressed because decompressing the database each time you run a query on it will slow performance.

NTFS compression is great for files that you need to keep around but don’t access often. It’s also good for files you access regularly if you don’t mind a slight performance hit because of the extra hard-disk real estate you gain.

Using NTFS File Security Features  NTFS file security lets you change settings for disk drives, folders, and even files. These features can be useful if your machine stores many files that you share with other users on your network as well as your private files.

If multiple users share a single workstation, issues of security and file access can be local to that machine. NTFS attributes dictate Permission, Auditing, and Ownership properties, but these issues relate more to using Windows 2000 on a network. (For information on these attributes, see Chapter 19, “Living with Windows 2000’s Strict Security.”)

Scanning for Bad Sectors  Although Windows 2000 doesn’t have a ScanDisk utility as DOS and Windows 9x do, you can still scan a disk for bad sectors. If you’re having trouble reading files from a disk, these files may be on a part of the disk that has bad sectors. You can scan a disk for bad sectors by using the Disk Administrator or by right-clicking the drive icon and selecting Properties. Let’s take a look at the latter approach:

1.  Double-click the My Computer icon, right-click the letter for the suspect hard drive, and select Properties.
2.  Next, select the Tools tab. Now you can check the disk for errors (bad sectors) or back it up. Click the Check Now button.
3.  The Check Disk dialog box contains two options: Check the first if you want Windows 2000 to automatically fix file system errors, and check the second if you want Windows 2000 to try to recover sectors gone bad. Enable both features and click OK.
4.  Windows 2000 now scans your hard disk looking for bad sectors and attempts to repair any it finds. This process is lengthy, so maybe plan a coffee break around it.

NOTE:  Close all applications before scanning a disk for bad sectors. Windows 2000 cannot perform a disk-scan if you have any other applications open because it needs exclusive use of the disk to perform its diagnostic tests.

You may remember a program called Check Disk (named CHKDSK) from the DOS/Windows 3.11 days. Windows 2000 still has Check Disk. It will scan your disk looking for bad sectors and, upon finding them, recover readable information and write it elsewhere on the disk to a .chk file. Check Disk will also mark the sector as bad so that no new data is written there. To run Check Disk from a command prompt, type CHKDSK /F and press Enter. To perform the same scan within Disk Administrator, right-click the letter of the drive to check and select Check for Errors.

Using Disk Defragmenter  Windows 2000 has included a disk defragmentation utility in its arsenal of tools to keep your computer running well. The Disk Defragmenter tool is built in to the Computer Management console in Administrative Tools.
