The settings you may need to change to connect successfully fall into the following categories:

Basic Communications
For setting the speed and format for commands and data sent between the two connected computers.
Terminal Emulation
For choosing between the types of terminals that HyperTerminal can simulate (terminals are defined later in the “Selecting a Terminal-Emulation Setting” section). Both the remote computer and your software must be set to the same terminal type to communicate correctly.

In addition, you can control a variety of settings that are a matter of preference, detailed in the “Setting Other HyperTerminal Preferences” section.

Choosing Basic Communications Settings

What we’re calling basic communications settings control the fundamental “language” of modem-to-modem communications. They determine such critical characteristics as how fast information flows between the modems, how they signal each other, and how each modem can tell one nugget of information from another.

One fundamental principle bears repeating: Most of these settings must be identical on both ends of the communications link (we note the exceptions as they come along). Because you probably have no control over the settings at the other end of the line, you’re stuck with making sure your settings match. Actually, most connections will work without any changes—HyperTerminal’s standard settings are the most popular ones, in use by most of the major commercial information services. But if you experience problems or if you know the other computer’s settings are different from your computer’s, you’ll need to alter the standard settings.

All the basic communications settings are controlled via the Properties dialog box for your modem. Access it as follows:

1.  Display the Properties dialog box for the connection you’re working with (see Figure 15.5). Select the Connect To tab to display the modem information for the connection.
2.  In the list box labeled Connect Using, choose the modem you want to use for the connection.
3.  Click the Configure button. You’ll see the Preferences dialog box for this particular connection, as shown in Figure 15.6.


Figure 15.6  The Connection Preferences dialog controls settings for the connection.

The modem Preferences dialog box has two tabbed pages: General and Advanced.

The General Tab

On the General tab of the Connection Preferences dialog box you’ll find the basic communications settings for each HyperTerminal connection you create. In this section, you will learn about the settings contained on the General tab.

Call Preferences  The three settings in this area of the General tab describe how the call will actually be dialed. They are really controlling the dialing of the phone, and when it will be automatically hung up. The settings are

Operator Assisted (Manual Dial)
Checking this box will cause Windows 2000 to prompt you to dial manually. This is especially helpful when you’re trying to dial out from a hotel room and need to give an operator instructions, or you need to dial a special number sequence to get through the switchboard. After you dial manually, listen for the other computer to answer, and then click OK on the dialog and hang up the handset. Your computer will then take over the phone connection.
Disconnect a Call If Idle for More Than
Use this setting to specify the number of minutes that your connection can be idle before Windows 2000 will automatically disconnect the call. This setting mirrors the one in the properties of your dial-up connections in Network and Dial-Up Connections in Control Panel. In the case of a conflict, the setting in Control Panel will override this one.
Cancel the Call If Not Connected Within
Specifies the number of seconds to wait for the other computer to answer the call. Not all modems support this option, so if yours appears to ignore it, you will have to consult the manual for the modem to see if there are commands that can be added to the initialization string for the modem.

Over on the Advanced tab, you’ll find the options that control the hardware settings for the connection, as well as the terminal window appearance.
