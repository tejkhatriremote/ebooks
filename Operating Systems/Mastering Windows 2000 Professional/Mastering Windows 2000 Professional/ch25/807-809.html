Understanding How Windows 2000 and NetWare Interact

A Novell NetWare network may incorporate Windows 2000 machines solely as workstations. This task is fairly simple because it requires no work on the server side at all. All you need to do to access the Novell servers directly from Windows 2000 is use the Client Service for NetWare, which is provided with Windows 2000 Professional.

Here are a few things you should know about this solution:

•  It may require dual network client software, which can be hard to manage and require additional memory.
•  It does provide access to both Novell NetWare and Windows 2000 server resources, but security and programming interfaces are not integrated.
•  It is difficult to share remote access and backup processes under this system.
•  It involves no special software installation on the servers, and so may be attractive to some network administrators.

Running NetWare and Windows 2000 Together

The idea here is simple: Your office contains both Windows 2000 servers and a NetWare server or servers. The Windows 2000 servers talk NetBIOS/NetBEUI and perhaps TCP/IP, and the Novell servers talk IPX/SPX or TCP/IP. Getting the two server families (NetWare and Windows 2000) to talk to one another is a mite tricky, goes the reasoning, so don’t bother. Instead, load both IPX/SPX and NetBIOS/NetBEUI stacks on your workstations.

NOTE:  There is a very common belief in the networking world that the Microsoft operating systems use only NetBEUI, and NetWare uses only IPX/SPX. Today you will typically find that both are coexisting quite well using TCP/IP to communicate. You need to ensure that you have at least one protocol in common among all the computers that will communicate on your network.

WARNING:  Although both operating systems can use TCP/IP to communicate over the network, the Microsoft Client Services for NetWare will work only with NWLink installed. Here again, Microsoft is fostering the belief that NetWare only uses IPX/SPX for network communication.

The Older Solution: Windows for Workgroups and Windows 95

To better understand how Windows 2000 Professional is providing NetWare support, let’s take a brief look at how Microsoft handled this problem in the past.

Windows for Workgroups ships with an IPX/SPX stack that runs in combination with NetBIOS/NetBEUI, making it possible to carry on conversations with both Windows 2000 and NetWare servers. Here are some features of the NetWare and Windows 2000 server parallel solution:

Simplicity (kind of)  You needn’t install any software on the servers. You needn’t put NWLink on your Windows 2000 machines nor install some kind of odd NetWare Loadable Module (NLM) on your NetWare servers.
Lack of drivers 1  To date, Windows for Workgroups does not ship with a NetWare redirector. The IPX/SPX protocol is in place, but the redirector is not, requiring that you run the redirector that came with Windows for Workgroups version 3.1. Microsoft doesn’t warn of this deficiency until you’re part way through the installation of Windows for Workgroups version 3.11. (Nice touch, boys and girls!) In addition, the combination of 3.11 and the old 3.1 driver is a bit wobbly.
Lack of drivers 2  If you’re running NetWare 4.x or 5.x servers, then there’s just no way that you can make the old Windows for Workgroups 3.1 NetWare drivers work reliably. You have to download a bunch of drivers from one of Novell’s online resources or from one of their support CDs. They seem to work fairly well talking to the NetWare servers but tend to eliminate some of the benefits of Workgroups. Besides, they cut off your connections to Windows 2000 Server machines periodically.
Memory management  Insufficient memory becomes a problem in that you’re loading a lot of programs on the workstation.
Sharing  Because you’re still running two different server environments, they don’t share anything. You’ve got to create separate user accounts on the Windows 2000 side of the house from the NetWare side. Remote access services aren’t shared, requiring twice as many modems. Backup services aren’t shared, so you’ve got to keep track of twice as many tapes.

TIP:  Windows 2000 greatly simplifies matters because Microsoft has written complete protocol stacks and client software for both Windows 2000 and Novell networks. Use all Microsoft client software, and you’ll be dealing with only one vendor.
