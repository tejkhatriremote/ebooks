Choosing a 32-Bit Novell Solution

Some of you may be familiar with the 32-bit drivers for NT that Novell provides. You may choose to use the Novell solution if you need to ensure driver compatibility with NetWare. Choosing this option is also preferable if you attach to different types of servers that are using IPX as their primary protocol.

TIP:  Client Service for NetWare may talk directly to Novell servers with few problems, but if you need to talk to many types of servers using the IPX protocol, you really want to use the NetWare client.

NOTE:  Remember that this is one time when you cannot use the NT 4 version of this service. Please go to Novell’s Web site, www.novell.com, to download the latest and greatest version of this software for Windows 2000.

The Client32 solution provides another nice selling point. Novell has a package of network tools they call Zero Effort Networking or Z.E.N.works. Z.E.N.works lets you gain complete control over desktop and user management from the NetWare 4.x and 5.x environments. It’s actually similar in many ways to Microsoft’s Zero Administration Windows (ZAW) initiative in that it uses policies and login scripts to manage users and computers. But in the case of Z.E.N.works, the policies are being applied based on NDS objects across all client platforms, not just one version of Windows.

The full version of Z.E.N.works provides several useful help desk functions as well, including a ticketing system for reporting problems and remote control ability. The version that is supplied with NetWare 5.x is the “light” version, which includes everything except the help desk tools, and it is supported by all of the current Client32 platforms.

Client32 integrates very well with Windows 2000, offering a single login that can be configured between the NetWare and Windows 2000 environments. Figure 25.7 shows the login screen offered by Novell’s Client32 software.

Accessing NetWare Servers from Windows 2000 Clients

As we conclude our discussion of Windows 2000 and NetWare compatibility, you should be aware of one important issue: Windows 2000 clients accessing NetWare servers. If you install the client software for NetWare that comes with Windows 2000, you will be given the option of processing the NetWare login script. The login scripts usually contain map statements such as the following: 

   map z:=den02\sys:public

This command will make the Z drive represent the public directory on a NetWare server known as DEN02. This result would occur on most DOS client machines. But Windows 2000 clients map only to the root directory, regardless of the map statements. That is to say, if you processed this same statement from a Windows 2000 client, Z drive would be DEN02\SYS:, not the “public” directory. The fix for this is easy to implement; instead of using the Map command, use the Map Root command, as follows:

   Map root z:=den02\sys:public

Now the public directory becomes the root directory of Z. If you want to get something from the public directory, simply go to Z:\ (the root directory of Z) and you will be in the public directory.
