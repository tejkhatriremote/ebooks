Configuring Windows 2000 to Run in Parallel with NetWare

Running Windows 2000 Professional in parallel with NetWare requires that the Client Service for NetWare be installed on the Windows 2000 machine. Installing the Client Service for NetWare is a straightforward procedure.

To install Client Service for NetWare, follow these steps:

1.  Right-click My Network Places on the Desktop and choose Properties.
2.  Right-click the icon for Local Area Connection and select Properties again.
3.  Click the Install button. Choose Client from the list and click the Add button to open the dialog shown in Figure 25.2.


Figure 25.2  The Select Client dialog box

4.  Select the Client Service for NetWare by double-clicking it.
5.  Type the drive and directory (if needed) at the prompt and click Continue. Windows 2000 copies the appropriate drivers to your system and makes changes to your Registry that enable the Client Service for NetWare to load the next time you start your system.

If you are loading the Client Service for NetWare and have not previously loaded the NWLink IPX/SPX compatible transport protocol, this protocol will load automatically as you install the service. The NWLink IPS/SPX Properties screen (see Figure 25.3) prompts you for the frame type that your NetWare server is using. You should contact the NetWare administrator to ensure that you select the correct frame type, even though Auto Detect frame type works most of the time.


Figure 25.3  Setting the frame type during install

NOTE:  The Client Services for NetWare will not work at all without NWLink installed. If you disable NWLink, Windows 2000 will automatically disable the Client Services for NetWare.

After you reboot the system, Windows 2000 attempts to log on to Novell for the first time, as seen in Figure 25.4.


Figure 25.4  Logging on to Novell for the first time

With the client software provided by Microsoft, you can either log on to a Novell 3.x bindery by clicking the Preferred Server radio button shown in Figure 25.4, or log on to a 4.x NDS tree by clicking the Default Tree and Context radio button. If you are going to log on to an NDS tree, do not provide the parameter codes before each option (i.e., omit the o= and the cn=). You can see an example of these selections in Figure 25.5.


Figure 25.5  Configuring the Client Service for NetWare

It is the nature of being a client in a networked environment that you may log on to one server one day and log on to another server on another day. If you are an administrator, you may switch between servers more frequently. You can change the NetWare server that you log on to with great ease. In My Computer, click the Control Panel. Now that you have installed the Client Service for NetWare, you will notice the CSNW icon in the Control Panel. By clicking this icon, you can adjust your settings and the NetWare servers or tree that you log on to, as seen in Figure 25.5.

From this screen you can specify or change the server, or log on to a NetWare 4.x/5.x NDS tree. In addition you can set the print settings. The three options are:

Add Form Feed
Adds a form feed to the end of each print job.
Notify When Printed
Displays a screen message when your print job finishes printing.
Print Banner
Adds a cover page to the beginning of each print job, specifying who printed the job and the date and time.

Any settings that you make here will apply to any Novell print queue that this Windows 2000 computer accesses.

The final option in CSNW determines whether the system logon script that resides on the NetWare server will run. This feature will run either a 3.x logon script or a 4.x/5.x logon script.

Using Novell Commands Available from Windows 2000

At the Windows 2000 Professional computer, you can open a command line and run a number of Novell commands by changing the directory to the public directory of the NetWare server. Here are some of the commands, in simple alphabetical order:

chkvol help rconsole settts
colorpal listdir remove slist
dspace map revoke syscon
fconsole ncopy rights tlist
filer ndir security userlist
flag pconsole send volinfo
flagdir psc session whoami
grant pstat setpass

You can also run NetWare-aware applications (some of them, anyway) with some support programs. Windows-based, NetWare-aware applications may require files netware.386, nwcalls.dll, and nwnetapi.dll. They are in your System32 directory.

Novell supplies a file called nwipxspx.dll that some client-server applications depend upon; Gupta SQLBase and Lotus Notes are two examples.

WARNING:  You must map a drive to the NetWare volume where these utilities are located in order to run them. Windows 2000 will not run them correctly from a UNC path.

Printing to Novell Printers

Earlier in this chapter, we described the print options that are available after you install the Client Service for NetWare. You can connect to a NetWare print queue by clicking Start Settings Printers and selecting Add Printer. This selection launches the Add Printer Wizard. You’ll be asked to select a shared printer on the network from the list provided, which looks similar to Figure 25.6.


Figure 25.6  Browsing the network for a Novell print queue

You will see a list of NetWare servers under an option called NetWare or Compatible Network. Select the server that maintains the print queue of the printer that you want to use and select the queue from the list. You will then be prompted to install the driver for the printer.

Enabling Long Filename Support on the Novell Server

To enable long filename support on the NetWare server, you need to load the OS/2 name space support. Here’s how you do it:

1.  At the NetWare server prompt, type the following:

Load os2
add name space os2 to volume sys

2.  Add the following to the startup.ncf file:

load os2

3.  Down the server (that is, bring down the server by typing Down at the server prompt), and when you see the C: prompt, copy the os2.nam file from the NetWare disks or CD-ROM to the same directory as server.exe.
4.  Type SERVER at the C: prompt to restart the NetWare server.
