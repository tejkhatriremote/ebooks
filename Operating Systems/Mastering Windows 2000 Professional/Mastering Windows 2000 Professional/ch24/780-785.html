Browsing My Network Places

Another method of attaching to network resources is to use the My Network Places. As you double-click the My Network Places icon on the Desktop, you will get a list of icons representing your network options, and one which will create a new network place. If you double-click the Computers Near Me icon you will see a list of all the computers that are sharing resources in your workgroup.

NOTE:  My Network Places shows you what resources are available across the network; it is your way of browsing the network.

At the very top of the list is an option to browse the entire network. When you click Entire Network, My Network Places displays a list of all the other domains, workgroups, and file servers in your environment.

If you have loaded multiple software clients, that is, Client Service for NetWare as well as the Client Service for Microsoft (Workstation Service), then you will see a list of networks you can browse after clicking Entire Network, as shown here.

If you select an option here, you will see only the Microsoft resources or the NetWare resources. You cannot view both sets of resources simultaneously from this window.

Mapping a Network Drive

If you will need to frequently revisit a particular network connection, you may want to consider mapping a drive to that location. Mapping a drive will assign a driver letter to a network location so that you can access the network resource easily. Many DOS applications do not accept UNCs; they require that you refer to NetWare locations by drive letter. Mapping a drive provides backward compatibility for your older (legacy) DOS applications.

To map a drive:

1.  Open My Network Places.
2.  Select the folder that you want to map.
3.  Right-click the folder.
4.  Select the Map Network Drive option.

When the Map Network Drive window opens, select the drive letter that you want to assign to the folder, as shown here. If a mapping for this drive already exists, you will be prompted to decide whether you want to overwrite the existing mapping or prefer to come back to this screen and try again.

Using the Connect As Option

You can use the Connect As option to attach to a resource with another user account. By default, when you map a network drive, you are attached to the resource using the same user account that you originally use to log on to the workstation. But what if you are attaching to another server and you do not have an account on that server? This happens a lot to administrators.

If you have an administrator called Sheila Sanders, then Sheila will have an account on her primary server that she logs on to. In addition, Sheila may need to attach to other servers that have resources that she wants to use, even though she doesn’t have an account on those servers. Another classic example would be that Sheila logs on to a Windows 2000 server as Sheila but has an account named SSanders on a NetWare server.

To resolve this issue, Sheila could log on to the Windows 2000 Server as Sheila, but when she maps a drive to a Novell server, Sheila could use the Connect As option to connect to the NetWare server as SSanders. If the Sheila password on Windows 2000 and the SSanders account on NetWare are the same, then Sheila gets direct access to the NetWare directory. If the passwords are different, Sheila will be prompted to enter a password for the NetWare server when she clicks OK to map the network drive.

Finally, if you want to remap this drive mapping every time that you log on, click the Reconnect at Logon box. Microsoft refers to this act as creating a persistent connection.

To Disconnect a Drive
1.  Go to the My Computer icon on the Desktop. (Remember, all mapped drives appear in the My Computer window.)
2.  Right-click the drive that you want to disconnect.
3.  Choose the disconnect option.

Making a Direct Connection via a UNC

Yet another way to attach to network resources is by using UNCs. An easy way to accomplish this connection is by executing Net Use statements at the command line prompt. But before you can attach to resources with the Net Use statements, you need to know the resources available on the network. We already showed you how to browse the network via the GUI, but how would you perform the same task at the command line prompt? The answer is Net View.

•  If you type the Net View command on its own, you get a list of the local servers on the domain or workgroup—that is, the services shared by the computer upon which you’re typing. This list is shown in Figure 24.19

•  If you want to see a list of the resources that a particular server is sharing with the network, you can append the name of the server that you want to look at to the command. For example, Figure 24.20 shows a server named Spiritwolf. You can view this kind of information for any server on the network.

•  If you want to look at a server in another domain (perhaps server Orion1 in domain Orion), type the following: 
   net view \\orion01 /domain:orion
