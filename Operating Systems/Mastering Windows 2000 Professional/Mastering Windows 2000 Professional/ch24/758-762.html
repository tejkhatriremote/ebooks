Base I/O Address

The base input/output address is the memory area that holds the information that needs to be manipulated. The best way to visualize the function of a base I/O address is to imagine the layout of your kitchen cupboards. Suppose you store your Tupperware containers in the cupboard above and to the right of the kitchen sink. If someone asks you for a Tupperware dish, you know exactly where to look.

The same holds true for the base I/O address for your NIC. When packets are sent to your NIC, the CPU needs to know where to find the packet—at the base I/O address.

Bus-Mastered Network Interface Cards

Before we conclude our discussion of DMA, we must introduce a special type of NIC known as a Bus-Mastered Interface Card. A Bus-Mastered card is a card that has a DMA chip on the card itself. In 1989 IBM introduced some of the first Bus-Mastered cards. IBM asserted that by putting the DMA chip on the card itself, the card could process and perform DMA transfers faster than the two DMA chips on the motherboard—and IBM was right. With its own DMA chip, a Bus-Mastered card is capable of sending data to and receiving it from RAM without additional processing intervention by the CPU. As soon as you put a DMA chip on any card, you have a Bus-Mastered card.

NOTE:  The card sitting in the BUS slot controls—that is, it masters—its own DMA transfer; hence the name, Bus-Mastered.

We have all driven behind cars that have little yellow signs with slogans such as Baby on Board. Well, as soon as you go to your nearest computer store and pick up a card that says Bus-Mastered, a little yellow sign that says DMA Chip on Board should pop into your mind.

Software Configurable Network Interface Cards

As we leave this section, we thought it would be a good idea to provide a list of some NICs that are software configurable, meaning that setting their IRQs, DMAs, and base I/O addresses is as easy as running a software routine. If you are new to setting up networks or are looking for an easier way to install your network, using software-configurable NICs is the way to go. Although there are many available on the market today, here’s a short list:

•  The Intel Ether Express Card
•  The Intel Token Express Card
•  The 3Com EtherLink III Card (3C5X9)
•  The latest cards from Thomas-Conrad
•  The latest cards from Hewlett-Packard

This tool is reached most easily by typing winmsd at the Run prompt, but you can also reach it through the Computer Management console in Administrative Tools. System Information is a useful tool for discovering what resources are in use on your computer. As you install your NIC, remember that in most cases you will have to set the NIC through the manufacturer’s software routine, jumper pins, or DIP switches and then tell Windows 2000 what the card settings actually are.

NIC Settings

When working with Windows 2000 Professional, you may need to specify the proper NIC settings. You may have to run a diagnostic utility that came with the NIC to determine this information. After you know the IRQ and base I/O port address, you will need to let Windows 2000 in on the secret by opening the Control Panel, double-clicking the Add/Remove Hardware icon, and selecting the Adapters tab. By double-clicking the NIC, you will see an NIC dialog box similar to the one below.

From this dialog box you tell Windows 2000 the information it needs to know about your Network Interface Card. You can set the IRQ, DMA, and base I/O addresses for your NIC in three ways:

•  You can set jumper pins that sit on the card.
•  You can set DIP switches that sit on the card.
•  You can set the IRQ and DMA for some cards via software.

Specifying Protocols

To install or configure network protocols you need to first open the Network and Dial-Up Connections window. You can do this a couple of different ways. First, you can use the Start menu by clicking Start Settings Network and Dial-Up Connections. The second way is to right-click My Network Places and choose Properties from the context menu.

Network and Dial-Up Connections will contain at least two icons after you’ve installed a network. It will have an icon for the Make New Connection Wizard, which will help you to create a new network or dial-up connection, and an icon for your Local Area Connection. Right-click this last icon and select Properties to display the dialog box shown in Figure 24.9. This Properties sheet has only one tab, the General tab, which gives you access to everything you need to configure this network connection.


Figure 24.9  Click the Install button and select Protocol to add additional protocols.

To add additional protocols, press the Install button and choose Protocol as the type of component to install. Figure 24.10 shows the list of protocols that are provided in Windows 2000, and the Have Disk option that enables you to install a third-party protocol. The initial list of protocols displays Microsoft-provided drivers. If you need support for an additional protocol, such as Banyan Vines or UNIX drivers, you need to get drivers that are designed for Windows 2000 from the vendor of the network operating system or application. When you receive or download these drivers, you select the Have Disk option in the Select Network Protocol window, as shown in Figure 24.10, and point the system to the directory or drive where the protocol driver resides.


Figure 24.10  Click the Have Disk option if you have the installation disk for the component you want to install.
