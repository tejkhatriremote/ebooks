Managing Profiles

Now that you have created your hardware profiles, you can begin to manage the services and devices that will be available through each profile. To manage the services that you want to load with each hardware profile, open the Control Panel Administrative Tools Computer Management System Tools and select Services. Figure 24.28 shows the Services selection box.


Figure 24.28  Services selection box

Review the list of services and their current status—Started or Stopped. Note that although none of the services in this figure are listed as stopped, they could have been. Select the service you want to manage. By way of example, we have selected the Computer Browser service.

Double-click the service name to open the Properties sheets for that service, and then click the Log On tab. The Log On tab of the Service Properties dialog box is shown in Figure 24.29.

Surviving User Profiles

Sometimes when you sit down at a Windows machine, you might be confronted with a lime green and purple color combination that a particular user swears by. And sometimes sitting at a computer system, you might be looking for the Notepad but someone has decided that the Notepad icon should be located in the Games window instead of the Accessories window. If you encounter these types of things from time to time, you will be glad to know that Windows 2000 has a solution to your problems. It is called a user profile.

Understanding Profiles

User profiles enable each user to customize the GUI to their individual needs. User profiles keep track of the following elements, among others:

•  Colors
•  Wallpaper
•  Screen savers
•  Start menu options
•  Desktop icons
•  Cursor of choice

Within the root of your computer’s hard drive is a folder called Documents and Settings, which contains the structure of the entire user profile configuration. The following subfolders are in the Documents and Settings folder:

•  A folder for each user who logs on
•  All Users folder
•  Default Users folder

User Folders

The Documents and Settings folder is created during the installation of Windows 2000. Each time a person logs on for the first time, Windows 2000 creates a separate subfolder with their logon ID. That user ID folder contains the contents of the Windows 2000 environment that is specific to the user.

The All Users Folder

If you want some information to be part of the Everyone profile when users log on, you can place this information in the All Users folder.

For example, Joe logs on to Windows 2000. He then installs a copy of snipes.exe on the computer and creates an icon for this program on the Desktop. (Please note: The function of snipes.exe does not matter; it is only relevant that Joe created an icon on the Desktop.) The snipes icon will be created in

   c:\Documents and Settings\joe\desktop\snipes.lnk

When Jennifer logs on, she will not get a copy of the snipes icon because it would not be part of her Desktop folder; it would not be part of her profile.

On the other hand, if snipes is a popular utility, you may want everyone who logs on to this machine to get a copy. All you would have to do is copy the snipes.lnk file to all of the Desktop folders for all the users who have logged on to this system—you might have to copy this file to 20 folders. And if a new person logs on to this workstation, you will have to remember to copy snipes.lnk to his or her \Documents and Settings\[userid]\desktop folder.

A better way to deal with the above scenario is to put a copy of the snipes.lnk file in the All Users folder. Every time a user logs on, Windows 2000 goes to the person’s \Documents and Settings\[userid]\desktop folder and places the appropriate items on the Desktop. Then Windows 2000 goes to the \Documents and Settings\all users\desktop folder and copies the contents of the Desktop folder to the user’s individual Desktop folder. Please note, Windows 2000 does put a physical copy of snipes.lnk in the user’s individual Profiles directory.

The example above applies to Desktop information, but the All Users folder is not limited to Desktop information, as we explain later in this chapter.

The Default User Folder

If this is the first time that you have logged on to a particular Windows 2000 machine, Windows 2000 creates a Profile folder with your individual information in it. In order to build the new Profile folder, Windows 2000 needs a model or template of information that will enable you to log on to this system for the first time. In essence, you need a default environment to start from. The default user environment is stored in the Default User folder, which stores information just like any Profile folder.
