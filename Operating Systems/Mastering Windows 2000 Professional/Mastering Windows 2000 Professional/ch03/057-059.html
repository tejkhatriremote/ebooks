Understanding Windows 2000 as a Multiuser Environment

An operating system is like your desk. Or more accurately, it’s like your whole office. If your office is cluttered, you will find it harder to find things and to be as productive as you like. The same goes for your operating system. Like other operating systems, Windows 2000 not only offers its own filing system, it pretty much forces you to use it. However, just as you may be in the habit of keeping stacks of mail in a drawer of your office filing cabinet because you never get around to buying those special hanging folders, you can misuse (or under-use) Windows 2000’s file system and not capitalize on its strengths. Fortunately, Windows 2000 has numerous tools, tricks, and enhanced usability features to keep you organized.

Under Windows 3.x, you had only two choices for organizing the contents of your hard drive: File Manager and Program Manager. Program Manager merely enabled you to arrange shortcuts to your programs. You could store files there as well, but almost nobody ever did. File Manager, on the other hand, enabled you to create and delete directories, to organize your files by means of those directories, and to launch or open programs and files by double-clicking them. Windows 2000, using a GUI that was developed from Windows 98, has taken the best features of Program Manager and File Manager, and has modified them to integrate them with the Desktop. In this chapter, you’ll see the usefulness of this approach as exemplified by the Start menu, which you can think of as an always-available list of shortcuts, and by Windows Explorer.

Using the Start Menu

Clicking the Start button brings up the Start menu, which, as the name implies, is the place to go first. From the Start menu, you can find anything you need or want on your system—and that includes anything on any machines that you have been given permissions to, through a shared or network arrangement. The Start menu is highly customizable insofar as you can decide what to list on it. You can easily modify it to present shortcuts to any program, file, or resource (such as a printer or an external tape drive). By default, it already offers a fast way to access the Help system, a command for opening the Control Panel (to change your computer’s settings), commands for launching tools such as Search and Windows Explorer (to find, analyze, or launch any file on your system), and the command for shutting down the system. The Start menu appears to the left of this paragraph.

TIP:  The Start menu can be accessed no matter what application you’re using, so you’ll always have instant access to the “controls” for your computer. If you have a Windows 95/NT keyboard, you can quickly access the Start menu by pressing the special Windows key; otherwise, you can press the Ctrl and Esc keys simultaneously.

Adding Your Favorite Programs to the Start Menu

If you copy any file, program, or system object (printers, hard drives) to the Start menu, Windows 2000 automatically creates a shortcut to that item—and because it’s on the Start menu, you can access those shortcuts anytime, no matter what other programs you are running.

To create a shortcut on the Start menu, drag the file you want onto the Start button. The next time you click the Start button, the shortcut to that file will appear on the top of the Start menu.

TIP:  If a shortcut is damaged or accidentally deleted, it won’t affect the file that it points to. If the shortcut is damaged, when you try to open it, Windows 2000 will automatically search for the object the shortcut points to.
