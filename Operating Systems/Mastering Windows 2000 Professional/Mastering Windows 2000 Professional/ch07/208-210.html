Understanding the Foundation Layer of DirectX

The Foundation layer of DirectX 7’s APIs is designed to handle the interaction of applications with today’s high performance multimedia hardware such as video accelerators, sound cards, and video capture cards. This layer of services automatically detects you computer’s hardware capabilities and sets the application’s configuration to match, giving you the best performance possible.

The Foundation layer provides support for 3D graphics and sound, 2D graphics acceleration, and input devices like joysticks and keyboards. The following components make up the Foundation layer:

DirectDraw
This API provides many of the same graphics functions as the Graphics Device Interface (GDI) in the Executive Services of Windows 2000, but actually it can process most of them much faster than GDI. DirectDraw provides extremely fast direct access to the accelerated abilities of the video adapter. It also provides vendors with a great opportunity to develop hardware for Windows 2000 as all they need to do is take advantage of DirectDraw’s hardware-independent graphics functions. DirectDraw supports 2D graphics acceleration and video codecs (compression and decompression services).
Direct3D Immediate Mode
This API gives applications a way to directly access the advanced 3D rendering ability of most new video cards without requiring extra configuration from the user. Direct3D can use all of the features of Intel’s MMX and Pentium III architecture as well as AMD’s 3Dnow! Capable processors.
DirectSound
Many of us remember multimedia programs or games where the sounds didn’t quite match up with the pictures on the screen. One of DirectSound’s best features is the ability to sequence sound and video so that this problem no longer occurs. DirectSound provides hardware-independent access to a sound card’s mixing, playback, and recording features.
DirectMusic
This is the pure music portion of DirectX. DirectMusic handles message-based sound like the Musical Instrument Digital Interface (MIDI) format. It also gives application developers the ability to design completely immersive soundtracks that can respond to the user’s activities. Immersive soundtracks contain audio tracks to create the full surround-sound experience when played back over theater style speakers.

NOTE:  MIDI stands for Musical Instruments’ Digital Interface, and it enables synthesizers made by different manufacturers to “talk” to one another with a common set of commands. A specification called General MIDI is commonly implemented on PC sound cards. General MIDI commands can be recorded and saved as a MIDI sequence, which is basically a batch file of commands to turn specific notes on and off at exact times, using instruments from different synthesizers if more than one is present. The important distinction between a MIDI sequence and an audio wave file is that a MIDI sequence doesn’t contain audio data, only instructions to be passed to a synthesizer that will produce the sounds. The main numbers you’ll probably want to know are that the General MIDI patch set consists of 128 patches (a patch is a specific instrument sound, such as that made by a clarinet), and MIDI data can be transmitted and received on 16 MIDI channels.
DirectInput
This API handles the user input from a variety of hardware such as force-feedback controllers, gamepads, joysticks, mice, and more. It includes support for the new Universal Serial Bus (USB) game controllers. It gives the application developer the ability to work with these devices in a completely hardware-independent manner.

Understanding the Media Layer of DirectX

The second layer of DirectX 7 is the Media layer. This layer is responsible for high-level functions, and works together with the Foundation layer to provide a full-featured multimedia experience.

Among the responsibilities of the Media layer are handling of animation, media streaming, and interactivity. The media streaming is one part that you probably already use. It’s the download and display of media from Internet sites that feature streaming media like movie clips, audio programs like radio, or broadcasts. The Media layer is comprised of the following API sets:

Direct3D Retained Mode
This API provides high level support for 3D graphics and animation. This is the media layer counterpart of the Direct3D Immediate mode that is part of the Foundation layer. It provides access to real-time three-dimensional graphics and animation capabilities for Internet applications, multimedia, and games.
DirectAnimation
The DirectAnimation API provides the integration and timing support for 2D and 3D images, sound, movies, text, and vector graphics. This is the piece of code that ensures that the individual components of your 3D action game all work at the same time.
DirectPlay
This API’s job is to provide that thrilling, head-to-head competition across modem, LAN, or other Internet connection. DirectPlay allows game designers to concentrate on building the game rather than dealing with the network hardware.
DirectShow
Spend any time watching movies on your computer? This API enables Windows 2000 to display MPEG, and .avi format videos, and even to play .wav audio files. DirectShow is also useful when using video capture hardware to record and edit digital video in Windows 2000.
DirectX Transform
This last API set has an odd name, but it’s very useful. DirectX Transform enables developers to build tools for editing and manipulating digital 2D or 3D images. It aids in animation, and in the development of graphics for Internet applications.
