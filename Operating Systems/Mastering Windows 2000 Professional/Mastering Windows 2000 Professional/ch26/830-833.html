Choosing Connection Protocols Supported by Dial-Up Networking

The protocol of choice for Dial-Up Networking is the Point-to-Point Protocol (PPP). Microsoft has designed most of its remote access and dial-up connectivity around this protocol. At the speeds most of us will conduct Dial-Up Networking (128Kbps or less), the primary advantages of PPP are that the connection checks errors in data and performs data compression during the transmission process. Consequently, data transfer is relatively fast and secure. Another reason that PPP is a popular protocol is that it is quickly becoming an industry-wide standard.

If you are using PPP, you can connect to a network using IPX/SPX, TCP/IP, NetBEUI, or any combination of the three. Because of its power and flexibility, PPP is the default protocol when installing Dial-Up Networking.

Novell uses software known as NetWare Connect to enable remote clients to dial up to the network. Novell generally provides its own client software to attach the remote PC to a NetWare Connect server. However, Windows 9x comes with a NetWare Connect client that enables your Windows 9x machine to attach to a NetWare Connect server directly, without going through a Microsoft gateway of any type. Even though Windows 9x machines can connect to a NetWare Connect server, the relationship is not reciprocal. NetWare Connect clients cannot dial into a Windows 9x server.

An older protocol standard is Serial Line Interface Protocol (SLIP). Unlike its counterpart, SLIP does not perform error checking or data compression while transmitting data; the modem takes on the responsibility of performing these functions. Fortunately, most 14.4Kbps and 28.8Kbps modems do perform these functions.

TIP:  Microsoft recommends that you use SLIP when dialing a Unix network from a dial-up server with TCP/IP, unless you know for certain the Unix host supports PPP.

Finally, Dial-Up Networking supports RAS as implemented by Windows for Workgroups 3.11 and Windows NT 3.1 (Windows NT 3.5x now defaults to PPP). You may see this option referred to as asynchronous NetBEUI (AsyBEUI).

So far this sounds great, but how do you install DUN?

Installing Dial-Up Networking

Installing DUN is much simpler than it was in NT 4. When you install a modem in Windows 2000, the Dial-Up Networking services are installed automatically.

Configuring the Dial-Up Networking Connection

To create and configure a DUN connection, you need to right-click My Network Places and select Properties, and then double-click Make New Connection. The Network Connection Wizard will enable you to create either a network connection or a Dial-Up Networking connection. In Figure 26.1, you can see the options available to you for the types of connections the wizard can create. For the sake of this discussion, we’ll assume that you are creating a Dial-Up to a Private Network.


Figure 26.1  The Network Connection Wizard lets you configure connections to a private network.

Each time you want to create a new dial-up connection, you will be using the wizard. After you have created the connection, you can modify the properties to include the server settings and configuration. This isn’t as easy as the method used in NT 4, perhaps, but the wizard does make short work of the process of creating the connection. Click the Finish button to complete the wizard and create the connection.

Once the connection is created, you can right-click the icon for that connection and choose Properties from the context menu to open the dialog box shown in Figure 26.2. There are five tabs in the connection Properties dialog box, which we will discuss in depth below.


Figure 26.2  The General tab from the connection Properties dialog box

General Tab

The following items are located on the General tab of the connection Properties dialog box:

Connect Using
This drop-down list box gives you access to the installed modems in your computer. Pick the modem you want to use for this connection. The Configure button just below this list box will open the Properties for the particular modem you are using.
Phone Number
Enter the number your system will dial to connect to the remote network. If this is a local call, the area code isn’t necessary. If you have to dial 9 or 8 to access an outside line, make sure you include that digit at the beginning of the phone number, though you could easily add these settings to a TAPI location later on. Sitting next to the Phone Number option is a button labeled Alternates. Click this button to enter additional phone numbers that you can use to connect to this remote network site.

NOTE:  Enter the phone number just as you would dial it on a touch-tone phone. Make sure to include outside access codes and the code that you use to turn off call waiting (if applicable).

TIP:  If you are dialing a 10-digit local number, enter your area code in the box for area code, then enter the entire 10-digit number as the phone number. This will fool Windows 2000 into dialing it as a local number.

Click the Configure button to display the Modem Configuration dialog box as shown in Figure 26.3.
