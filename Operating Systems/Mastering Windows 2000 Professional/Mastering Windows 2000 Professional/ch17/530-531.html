Problems Related to Scale

In more than 20 years of experience with microcomputer operating systems, I’ve seen a number of really good products go down in flames through no fault of their own, save for one thing: They never saw enough acceptance. Although it’s true that most Windows 9x applications run under Windows 2000, a system requires more than applications. For example, drivers must be available to run for every piece of hardware that the application needs.

Fewer Drivers Are Available

Running your video board, network card, mouse, SCSI host adapter, or scanner requires a piece of software called a driver. Someone must sit down and write a driver program for each piece of hardware in your system. Of course, Windows 2000 includes drivers right in the box for major pieces of hardware. But what about not-so-major pieces of hardware? For example, although obtaining a driver for the 3Com Ethernet board is easy enough, can you get one for the SMC Ethernet cards? And even if a major vendor, such as 3Com, comes out with a new card after Windows 2000 ships, will anyone write a Windows 2000 driver for that board?

The relatively small number of available drivers is definitely one of NT’s main weaknesses. Windows 2000 uses the same drivers as Windows 98 for most of its hardware, but not all. It includes a new version of the Network Device Interface Specification (NDIS) drivers for network cards. Now this isn’t as bad as it may sound. In this case, the driver for Windows 98 works fine, but the information (.inf) file that tells Windows 2000 how to install the driver is in a different format.

The things to check out before you commit to Windows 2000 are usually the drivers for your video card and sound card. SCSI and network card drivers are more plentiful, but you can’t be positive that the generic Sound Blaster 16 driver will work on your Sound Blaster clone. Unless you’re using a Microsoft or Logitech mouse, check for Windows 2000 mouse drivers as well. Or restrict your purchases to devices on the Microsoft Hardware Compatibility List.

Now, after all the gloom, it has to be said that driver support for Windows 2000 is getting better all the time. If in doubt, check the Web site for the company that made your hardware. There have been many cases lately where the support staff for a company tells you that a driver for NT isn’t available, and then you find one right on their Web site! The moral here is that everyone, including Microsoft, has been surprised by the rapid acceptance of NT as the operating system of choice by savvy users, and we can expect that to continue with Windows 2000.

Not All Windows 9x Apps Run

NT 4 did not include much support for Microsoft’s game API DirectX, so there were many games and other applications that would not run correctly, or not run at all under NT. Windows 2000 includes the full DirectX 7 API set, so this should no longer be a major issue for new users of Windows 2000. But there will still be issues if applications are written to look for a specific version of Windows. For instance, if a program is written to only run on Windows 95 by name, it won’t run on Windows 2000. It probably won’t even run on Windows 98 for that matter.

Additionally, many application programmers just plain haven’t figured out the correct way to write a 32-bit application, so they cobble together some of the 16-bit things that they’ve already written and understand and wrap a bit of 32-bit code around it, calling the result a Windows 95 application. Before you know it, Windows 95 is out of memory and you end up with a Windows 95 application that doesn’t run under Windows 2000.

Problems Related to Missing Windows 9x Functionality

Although Windows 9x certainly isn’t the benchmark of quality for operating systems, it has one great strength when compared to NT and most other operating systems: its hardware support. Windows 9x makes installing new hardware considerably easier than it ever was; unfortunately, NT doesn’t share that strength.
