[MassStorageDrivers]  If [MassStorageDrivers] is missing or empty, Setup will attempt to detect the mass storage device on the computer and install the appropriate drivers.

•  <MassStorageDriverDescription> = RETAIL | OEM

The values replacing <MassStorageDriverDescription> must match one of the values in the [SCSI] section of the txtsetup.sif file (for a retail driver) or txtsetup.oem (for an OEM driver). More than one <MassStorageDriverDescription> can be listed in this section.

RETAIL implies that the driver is supplied as part of the Windows 2000 Professional installation.

OEM indicates that the driver is OEM-supplied. The driver must also be listed in the [OemBootFiles] section of the unattend.txt file as well.

[DisplayDrivers]  [DisplayDrivers] is a list of the display drivers to be loaded and installed during Setup. It will be ignored unless OemPreinstall = Yes. If the section does not exist or is empty, Setup will automatically detect the display devices and install the appropriate drivers.

•  <DisplayDriverInformation> = RETAIL | OEM

Indicates the driver to be installed. <DisplayDriverInformation> is replaced by a string that matches those in the right side of the [Display] section of txtsetup.sif (for retail drivers) or txtsetup.oem (for OEM drivers). More than one driver can be listed in this section.

RETAIL implies that the driver is supplied as part of the Windows 2000 Professional installation.

OEM indicates that the driver is OEM-supplied. The driver must be listed in the [OemBootFiles] section of the unattend.txt file as well.

[KeyboardDrivers]  [KeyboardDrivers] is a list of the keyboard drivers to be loaded and installed during Setup. It will be ignored unless OemPreinstall = Yes. If the section does not exist or is empty, Setup will automatically detect the keyboard devices and install the appropriate drivers.

•  <KeyboardDriverInformation> = RETAIL | OEM

Indicates the driver to be installed. <KeyboardDriverInformation> is replaced by a string that matches those in the right side of the [Keyboard] section of txtsetup.sif (for retail drivers) or txtsetup.oem (for OEM drivers). More than one driver can be listed in this section.

RETAIL implies that the driver is supplied as part of the Windows 2000 installation.

OEM indicates that the driver is OEM-supplied. The driver must be listed in the [OemBootFiles] section of the unattend.txt file as well.

[PointingDeviceDrivers]  [PointingDeviceDrivers] is a list of the pointing device drivers to be loaded and installed during Setup. It will be ignored unless OemPreinstall = Yes. If the section does not exist or is empty, Setup will automatically detect the pointing devices and install the appropriate drivers.

•  <PointingDeviceDriverInformation> = RETAIL | OEM

Indicates the driver to be installed. <PointingDeviceDriver Information> is replaced by a string that matches those in the right side of the [Mouse] section of txtsetup.sif (for retail drivers) or txtsetup.oem (for OEM drivers). More than one driver can be listed in this section.

RETAIL implies that the driver is supplied as part of the Windows 2000 installation.

OEM indicates that the driver is OEM-supplied. The driver must also be listed in the [OemBootFiles] section of the unattend.txt file as well.

[OEMBootFiles]  [OEMBootFiles] contains a list of OEM-supplied boot files. It will be ignored unless OemPreinstall = Yes and the files have also been put into the $OEM$\ Textmode directory of the OEM’s distribution share. The files are listed below:

•  Txtsetup.oem

This file has descriptions of all OEM-supplied drivers in this section. It also includes instructions on how to install drivers that are listed in this section. If this section exists, this file must be listed.

•  <HALFileName>

Maps to a HAL description that has been indicated by the ComputerType key in the [Unattended] section of the unattend.txt file.

•  <SCSIDriverFileName>

Maps to a mass storage driver description listed in the [Mass StorageDriver] section of the unattend.txt file. More than one <SCSIDriverFileName> can be listed in this section.

•  DetachedProgram = <DetachedProgramString>

Indicates the path of a custom program that should run concurrently with Setup.

•  Arguments = <ArgumentsString>

Indicates any arguments or parameters required by the detached program given in the DetachedProgram key.
