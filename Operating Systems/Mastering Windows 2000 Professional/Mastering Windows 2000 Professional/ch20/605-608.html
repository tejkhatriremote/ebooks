Installing Windows 2000 is a fairly straightforward procedure. It is even easier if you know your machine and think about the logistics for a moment before you begin. Although the installation procedure is more time-consuming than it is for some applications, you do have an alternative to standing over the computer and responding to dozens of installation questions. You can write a setup batch script that will automatically answer all those questions. But first, this chapter provides a step-by-step guide to installing Windows 2000 on its own and also as a link to the server. Then we explain how Windows 2000 interacts with the server and other operating systems, how to install it efficiently to several machines using the batch routine, and how to install fonts.

Knowing Your Computer Needs

First and foremost, make sure your computer has the CPU, disk space, and RAM necessary for the operating system. And just as important, be sure you have all the hardware you need to install and operate Windows 2000 Professional smoothly. Next, ascertain whether the computer will be a stand-alone workstation or part of a domain. Determine also if Windows 2000 Professional will be the only operating system on the machine or if another OS, such as Windows 9x, will be present as well. Let’s talk about each of these steps.

Reviewing Your Hardware Requirements

You need to make sure you have all the hardware necessary to install and operate Windows 2000 effectively. Hardware requirements are outlined in the following sections.

CPU

The CPU must be at least a 166MHz Pentium-class processor or compatible (Windows 2000 doesn’t run on 486 machines anymore). This requirement shouldn’t be too much of a problem now, as it has become difficult to buy anything less than a Pentium II.

RAM

The amount of RAM you need to run Windows 2000 Professional depends on your patience. The minimum recommendation is 32MB, which works but is very slow; 64MB of RAM provides a more satisfying speed. Don’t worry, though. If you don’t have 64MB of RAM and haven’t worked with a machine that does, you won’t know what you are missing. You don’t have to rush out and get more RAM. Do remember, however, that Windows 2000 gets faster when you add more RAM. With today’s memory prices, it’s easy to consider running Windows 2000 on 128MB or even 256MB.

Hard Disk

Windows 2000 is more demanding in this area as well. To install Windows 2000 Professional, you will need at least a 2GB disk (don’t worry, though, you won’t need all of it), with at least 650MB free. Windows 2000 will pre-install nearly all of its drivers, so a lot of that space will be storage for those drivers.

CD-ROM

You can install Windows 2000 from a CD-ROM in either of two ways: from three floppy boot disks and a CD-ROM, or from a CD-ROM by itself. For installation on a single workstation, the three-floppy method doesn’t sound too bad, but that step just gets the installation process going. You still need the CD-ROM. The Setup program copies files from the CD to the floppies to start the installation. In a CD-only installation, however, Setup instead copies the files to your hard drive. If you have enough space on your hard drive, you don’t need to use floppies.

NOTE:  Many computers now support booting from the CD-ROM. Windows 2000 CD-ROMs are bootable on machines that support this feature. This means you can insert the CD-ROM, boot the machine, and enter directly into Setup. You may have to change the boot order settings in your computer’s BIOS to activate this feature.

Installing Windows 2000 as a Stand-Alone Workstation or Domain Member

Determine whether this computer will be a stand-alone workstation or part of a domain. If it is to be part of a domain, the installation program will ask you several questions. The domain installation process is faster and easier if you know the following:

•  The type of network card installed in the computer. Some network cards require the user to adjust the manufacturer’s default settings. Before installing, run a diagnostic tool such as WinMSD to determine what these settings should be, and then write them down so that you can refer to them later.
•  The protocol(s) the machine uses to connect to the domain controller (server).
•  Your computer’s set Internet Protocol (IP) address, unless you have a Dynamic Host Configuration Protocol (DHCP) server to determine your computer’s IP address. If you do have a DHCP server on your network, a static IP address isn’t required. Ask your system administrator if you don’t know whether your network has a DHCP server.

As you can see, making the machine part of a domain necessitates the addition of a network card. If you do not have a network card set up when you install Windows 2000 Professional, you can still continue the installation process. Just set up the computer as a stand-alone machine and make it part of a domain after you install the network card.

Yes, you read that correctly. Changing your mind about having the workstation as part of domain does not mean completely reinstalling Windows 2000. One great feature of Windows 2000 Professional is that you can work through those kinds of administrative details quite simply by running a program. The option of quickly adding a workstation to a domain at any time can be rather handy. You’ll find out how to connect a workstation to a domain in the “Setting Up Domain and Administrator Accounts” section later in this chapter.
