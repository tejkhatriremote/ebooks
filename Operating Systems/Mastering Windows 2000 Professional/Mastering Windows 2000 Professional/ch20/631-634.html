Making a Dual Boot Work

A dual boot refers to the boot.ini file that creates a Startup menu, which enables you to choose an operating system for that session. Windows 2000 creates the boot.ini during the installation process when it locates one or more preexisting operating systems.

Every time you install Windows 2000 on your system, Windows 2000 creates a new menu entry. If you have installed Windows 2000 into more than one directory on your machine, you will encounter identical menu entries for Windows 2000. For example, one user created two Windows 2000 installations on one workstation: One version was installed with a FAT file system; the other uses an NTFS system. Even though the entries may look alike, they point to different directory trees on the hard drive. To distinguish the difference between entries, you’ll probably want to edit the boot.ini file.

Editing the file is simple. Using any text editor, you can make the changes necessary to distinguish between the menu entries. Why edit the file? Well, as mentioned previously, there may be more than one entry in the menu. Also, you may have deleted or renamed a Windows 2000 installation. If so, then you need to either change the directory name in the boot.ini file or delete the entry entirely. As long as boot.ini has at least one valid entry, you will be able to load an operating system.

The boot.ini file has two sections: boot loader and operating systems. The first item you’ll enter into the boot loader section is the time-out, which specifies the amount of time you are given to pick an operating system. Windows 2000 creates a time-out of 30 seconds by default, but you can change this item to any length of time that you like. The other item in this section is the default operating system, which is automatically chosen if the user doesn’t pick one during the time-out period.

NOTE:  If Windows 2000 is the only operating system present on the computer, it will not display the boot menu very long, usually less than five seconds. In this case, the delay is only there to enable you to get to the troubleshooting boot options by pressing the F8 key.

In the operating systems section, you need to provide information in order for the system to locate where Windows 2000 and other operating system files reside. Knowing on which hard disk and partition the Windows 2000 files are located is very important here. Without this information, Windows 2000 can’t locate the rest the files it needs in order to boot.

Each menu entry for Windows 2000 has four parts:

•  multi(#), with # starting at 0; designates the hard disk controller for the disk where Windows 2000 is located
•  disk(#), with # starting at 0; designates the hard disk on which Windows 2000 is located
•  rdisk(#), with # starting at 0; designates the hard disk on which Windows 2000 is located
•  partition(#), with # starting at 1; designates the hard disk partition on which Windows 2000 is located

For example, you might have two hard disks and two partitions on the second hard disk. Let’s say that you installed Windows 2000 three times (once on the first hard disk and once in each partition). The entries in the boot.ini file would look like this:

   multi(0)disk(0)rdisk(0)partition(1)\WINNT="Microsoft Windows 2000 Professional"
   multi(0)disk(0)rdisk(1)partition(1)\WINNT="Microsoft Windows 2000 Professional"
   multi(0)disk(0)rdisk(1)partition(2)\WINNT="Microsoft Windows 2000 Professional"

You may also designate any other systems that may exist. For example, if you have Windows 95 residing on your system, you’ll want to create an entry like this: 

   C:\="Microsoft Windows 95"

When boot.ini is finished, it should look like this:

   [boot loader]
   timeout=30
   default=multi(0)disk(0)rdisk(0)partition(1)\WINNT[operating systems]
   multi(0)disk(0)rdisk(0)partition(1)\WINNT="Microsoft Windows 2000 Professional"
   multi(0)disk(0)rdisk(1)partition(1)\WINNT="Microsoft Windows 2000 Professional"
   multi(0)disk(0)rdisk(1)partition(2)\WINNT="Microsoft Windows 2000 Professional"
   C:\="Microsoft Windows 95"

Save the file, reboot your machine, and you should be able to load Windows 2000 or any other operating system by clicking one of the options you just created. The path to the operating system files is called an Advanced RISC Computing (ARC) path. It came originally from the RISC platforms and was chosen to facilitate the porting of Windows NT to those platforms. Table 20.2 has more information about the sections of an ARC path.

Table 20.2 Sections of an Arc Path
Keyword Description
multi(#) Identifies the hard disk controller and is used for either IDE or SCSI with an active BIOS. Numbers start from 0.
scsi(#) Identifies the hard disk controller and is used only when Windows 2000 is installed on a SCSI disk without an active BIOS. Numbers start at 0.
signature(#) This keyword indicates a disk controller by an internal serial number. The number portion is represented by an eight-digit hexadecimal value. It only appears when Windows 2000 is installed on a FAT32 partition.
disk(#) Identifies the hard disk and is used only with the scsi(#) keyword. It will be ignored otherwise. Numbers from 0.
rdisk(#) Identifies the hard disk when the path begins with the multi(#) keyword. It will be ignored otherwise. Numbers start at 0.
partition(#) Identifies the partition number where Windows 2000 is installed. Numbers start at 1.
\<folder> This is the name of the root of the folder structure where the Windows 2000 system files are located.

Examples of valid ARC paths would be: 

   Multi(0)disk(0)rdisk(3)partition(2)\winnt
   Scsi(2)disk(2)rdisk(0)partition(1)\winnt4
   Signature(ea9b9c97)disk(0)rdisk(0)partition(1)\winnt
