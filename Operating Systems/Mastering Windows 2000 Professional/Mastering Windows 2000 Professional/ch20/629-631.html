Creating a Repair Disk on Windows 2000

If you want to create a repair disk after you’ve already installed Windows 2000, then you’ll need to run the Backup utility while you are logged on. Start at the command prompt and follow these steps to create an Emergency Repair Disk:

1.  Open the Backup utility by clicking Start Programs Accessories System Tools Backup. The Backup utility starts with a wizard screen like the one in Figure 20.6. You can reach this page at any time by clicking the Welcome tab.


Figure 20.6  The Backup utility in Windows 2000 lets you create an emergency repair disk.

2.  Click the Emergency Repair Disk button and follow the prompt. You will need one formatted 1.44MB floppy disk. This process also updates the repair information on your hard drive that is located in the %SystemRoot%\Repair folder.

Repairing a Damaged Version of Windows 2000 Professional

Repairing a damaged version of Windows 2000 begins the same way an installation does: with the WINNT32 command. The Setup program first copies files from the I386 directory to a temporary directory and restarts. Then you are asked if you want to set up Windows 2000 or repair a damaged version of Windows 2000.

NOTE:  To repair a damaged Windows 2000 installation, you must have both the emergency repair information and a Windows 2000 CD. If you don’t have both of these items, you’re going to have to reinstall Windows 2000. The repair information can be obtained from either the emergency repair disk or from the Repair folder on the hard disk. NT allowed you to perform a repair with only the Setup floppies and a repair disk, but Windows 2000 requires the presence of the original CD-ROM in a local CD-ROM drive.

Press R to repair the damaged version.

At this point, Setup gives you the option to do one of the following:

•  Inspect the Registry files.
•  Inspect the start-up environment
•  Verify the Windows 2000 system files.
•  Inspect the boot sector.

Next, you are prompted to insert your repair disk into the A drive.

After you’ve done that, Setup then asks if you want to repair the following Registry files:

•  SYSTEM (system configuration)
•  SOFTWARE (software information)
•  DEFAULT (default user profile)
•  SECURITY (security policy)
•  SAM (user account database)

Check the boxes next to the files you want to repair, highlight Continue, and press Enter. Setup searches for files in the Registry that have either been changed or damaged. When it locates such a file, it asks if you want to replace it with an original version. If you do, then you need to use your CD-ROM. If you don’t want to replace the file, then go on to the next file. After Setup has finished searching for damaged files, you need to restart your computer to load your repaired version of Windows 2000 Professional.

Using Dual Operating Systems

What if you have Windows 95 (or any other operating system) already installed on your machine and you want to add Windows 2000 (or vice versa)? Having more than one operating system on a machine can be a tricky business. Problems can arise when you install an application that has already been installed on one operating system on another. Separate operating systems and their applications need to stay separate. Having two operating systems generally means installing your applications twice. There is no easy way out, although individual software specs can vary.

Consider this example: One user installed Lotus SmartSuite in NT but put it in the Windows 95 system directory, hoping that somehow NT and Windows 95 could share the files. Unfortunately, most of the applications crashed in Windows 95 after installing them with NT. Here’s what happened: Some of the files used by SmartSuite and its applications were originally configured for Windows 95; being replaced with NT files made them unreadable by Windows 95. This problem should be resolved for the most part in Windows 2000, but the only real way to know with your applications is to test them in a dual boot scenario.

TIP:  What do you do if you use the same application on multiple operating systems? The only answer that we can see is to install your applications on each operating system, because applications can’t be shared. The applications can usually be installed in their original folders because they will place their operating-system-specific files in the proper locations within the operating system folders.

Of course, some applications will not work at all while running under Windows 2000. You’ll want to check to be sure that an application will work under Windows 2000 before you shell out big bucks for an expensive software package. Try to find someone who is already running that software package that you want to buy and ask their opinion of its performance on Windows 2000.
