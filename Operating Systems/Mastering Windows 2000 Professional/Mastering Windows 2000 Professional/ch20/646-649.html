[UserData]

•  FullName = Your Full Name

Indicates the full name of the user. If the FullName key is empty, the user is prompted to enter a name.

•  OrgName = Your Organization Name

Implies the name of your organization. If the OrgName key is empty, the user is prompted to enter an organization name.

•  ComputerName = MYCOMPUTER

Indicates what the computer will be named. If the ComputerName key is empty, the user is prompted to enter a computer name.

•  ProductId = 123456789012345

This specifies the Microsoft product identification (productID) number. This number will be requested by Microsoft if you call for technical support. The ProductID specified takes precedence over any other ProductID specified in the initial.inf file. The productID must be specified in order to perform an unattended upgrade.

[Display]  The [Display] section lists the display settings for the display device defined in the [DisplayDrivers] section. In order for your display to be set up properly, you must know the correct settings for the graphics device. If the settings are not valid, the unattended installation will halt and prompt the user to input the correct settings. The settings are as follows:

•  ConfigureAtLogon = 0 | 1

Indicates when the display device is configured.

0 indicates that the device will be configured during Setup. 1 implies that the device will be configured by the user after the first logon.

•  BitsPerPel = <ValidBitsPerPixel>

Indicates the number of bits per pixel for the device being installed.

•  Xresolution = <ValidXResolution>

Indicates the horizontal resolution of the display. Typically, this is 640.

•  YResolution = <ValidYResolution>

Indicates the vertical resolution of the display. Typically, this is set at 480.

•  VRefresh = <ValidRefreshRate>

Indicates the speed at which the display screen is replaced.

•  Flags = <ValidFlags>

Indicates any valid flags for the driver being installed.

•  AutoConfirm = 0 | 1

Indicates whether the device should be configured with the above settings.

0 means not to use the settings listed above. 1 indicates that these settings should be used. If 1 is used, then all the above parameters need to be specified. If 0 is used, then the above settings do not need to be included. However, if you do not specify any settings, you will be prompted for them during the installation.

[Modem]  The [Modem] section indicates that a modem will be installed. Remote Access Service (RAS) uses this section to install a modem if DeviceType = Modem is located in the list of RAS parameters.

•  InstallModem = <ModemParameterSection>

Defines the section where the parameters to install modems are located. This key must exist if you want to install a modem.

[<ModemParameterSection>]  [<ModemParameterSection>] lists the parameters needed to install a modem on a specific COM port. If this section is empty, RAS will attempt to detect and install a modem.

•  <COMPortNumber> = “<ModemDescription>” [, <Manufacturer>,<Provider>]

<COMPortNumber> indicates the COM port on which a modem will be installed. This number must match ports configured by the RAS installation.

<ModemDescription> matches the modem description in an mdmxxxxx.inf file that corresponds to the modem.

<Manufacturer> and <Provider> identify the manufacturer and provider of a given modem. This statement is used when the modem description is not unique.

[Network]  [Network] is used to specify that networking will be installed. If this section is empty, you will receive a number of error messages. If this section is missing, Setup will not install networking.

•  Attend = yes | no

You’ll specify this key when you want to install networking manually, which is generally a good idea. With an unattended installation, if you’re installing Windows 2000 on hundreds of machines, you may install the same error on every computer. Also, if a computer is configured incorrectly, it may cause problems with other computers on the network. Another reason to install networking manually is that you won’t be able to install many network adapter cards in an unattended installation. For example, if you have a 3Com PCI Bus Master card in your system—a card that is widely used—you can’t perform an unattended installation without installing the network card manually. See the InstallAdapters key listed below to see what we mean.

You’ll have to install networking manually regardless of whether the key is set at yes or no. If you don’t want to do so, do not include this key in the [Network] section.

•  JoinWorkgroup = <WorkgroupName>

This key names the workgroup that the computer will be a member of.

•  JoinDomain = <DomainName>

Indicates the name of the domain you want the computer to be a member of.

•  DetectAdapters = <DetectAdaptersSection> | “”

Indicates that you want Windows 2000 to attempt to detect your network card. Setup will install the first card detected if the value of this key equals “”.

•  InstallAdapters = <InstallAdaptersSection>

Defines the section in which the network adapter cards will be listed. Either this key or the DetectAdapters key must exist if you want to install a network card.

•  InstallProtocols = <ProtocolsSection>

Defines the section in which the protocols will be listed.

•  InstallServices = <ServicesSection>

Defines the section in which the services will be listed.

[<DetectAdaptersSection>]  [<DetectAdaptersSection>] is defined by the DetectAdapters key mentioned previously. You can name this section as you wish.

•  DetectCount = <number of detection attempts>

Determines the number of attempts to make to detect a network card.

•  LimitTo = <NetcardInfOption>

Indicates a list of options to limit the network card detection. These options can be found in the [Options] section of the network adapter card’s oemnadxx.inf file. For instance, a 3Com Etherlink III card is named oemnadxx.inf. You may have to do some searching to find the right file, as you can see that these .inf files are not clearly named.

•  <NetcardInfOption> = <NetcardParameterSection>

Defines the section that has descriptions for a specific network card. These descriptions can be found in the [Options] section of the card’s oemnadxx.inf file.
