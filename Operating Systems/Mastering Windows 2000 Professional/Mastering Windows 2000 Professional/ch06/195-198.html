Working with the Print Queue

If you print more than a few files at a time, or if you share your printer for network use, you’ll sometimes want to check the status of a printer’s print jobs. You also might want to see how many jobs need to print before you turn off your local computer and printer if others are using it. Or you might want to know how many other jobs are ahead of yours.

You can check on these items by opening a printer’s window. You’ll see this information:

Document Name
Name of the file being printed and possibly the source program
Status
Whether the job is printing, being deleted, or paused
Owner
Who sent each print job to the printer
Progress
How large each job is and how much of the current job has been printed
Start at
When each print job was sent to the print queue

NOTE:  If the print job originated from a DOS program, the document name will not be known. It’s listed as Remote Downlevel Document, meaning that it came from a workstation that doesn’t support Microsoft’s RPC (Remote Procedure Call) print support. This is also true of print jobs from Windows for Workgroups, LAN Manager, Unix, and NetWare.

To see the queue on a printer, open the Printers folder and double-click the printer in question. Adjust the window size if necessary so you can see all the columns. You can resize the columns in the display to see more data in a small window. Move the pointer to the dividing line in the header display and drag the line left or right.

If the printer in question is a network printer, and the printer is offline for some reason, such as its computer isn’t turned on, you’ll be forced to work offline. An error message will alert you to this, and the top line of the printer’s window will read, “User intervention required—Work Offline.” Until the issue is resolved, you won’t be able to view the queue for that printer. You can still print to it, however.

When an error occurs during a print job, Windows 2000 tries to determine the cause. For example, the printer might be out of paper, or the printer might be offline or unplugged from the AC outlet. You may be forced to work offline until the problem is resolved. Opening the printer’s queue should display an error message approximating the nature of the problem to the best of Print Manager’s capabilities. Check the printer’s File menu to see if the Work Offline setting has been activated. When you think the problem has been solved, turn off this setting to begin printing again.

Refreshing the Network Queue Information

The network cabling that connects workstations and servers often is busy, so Windows 2000 usually doesn’t bother to add more traffic to the network by polling each workstation for printer-queue information. This is done when necessary, such as when a document is deleted from a queue.

If you want to refresh the window for a printer to get the absolute latest information, press F5. This immediately updates the queue information.

Deleting a File from the Queue

After sending a file to the queue, you might change your mind about printing it. For example, you might discover that you need to make some changes before printing the document. If so, you can simply remove the file from the queue. Here’s the procedure:

1.  Open the printer’s window.
2.  Select the file by clicking it in the queue.
3.  Choose Document Cancel Printing, press Delete, or right-click and choose Cancel Printing. The document item is removed from the printer’s window. If you’re trying to delete the job that’s printing, you might have some trouble. At the very least, the system might take some time to respond.

You can delete other people’s print jobs if the printer in question is connected to your computer. However, normally, you can’t delete someone else’s print jobs on a remote printer. If you try to, you are told that this is beyond your privileges and that you should contact your system administrator. If you want to be able to delete jobs on a remote computer, someone must alter the security settings on the remote computer to enable remote administration of the printer. Remote administration is covered in Part V.

You may find, especially with PostScript laser-type printers, that after deleting a file while printing, you need to reset the printer to clear its buffer or at least eject the current page (if you have a page-eject button). To reset, you typically need to push a button on the printer’s front panel or turn off the printer for a few seconds, then on again.

NOTE:  Because print jobs in Windows 2000 are stored on the hard drive of the print server as files, pending print jobs are not lost when computers are powered down. Any documents in the queue when the system goes down reappear in the queue when you power up. When you turn on a computer that is the host for a shared printer that has an unfinished print queue, you are alerted to the number of jobs in the queue and asked whether to delete or print them.

Canceling All Pending Print Jobs on a Printer

Sometimes, because of a megalithic meltdown or some other catastrophe, you’ll decide to bail out of all the print jobs that are stacked up for a printer. Normally, you don’t need to do this, even if the printer has gone wacky. You can just pause the queue (as described in the next section) and continue printing after the problem is solved. But sometimes you’ll want to resend everything to another printer and remove the queue on the current one. It’s easy:

1.  Select the printer’s icon or window.
2.  Right-click and choose Cancel All Documents, as shown in the following graphic. Or, from the printer’s window choose Printer Cancel All Documents. All queued jobs for the printer are canceled.

WARNING:  Make sure you really want to cancel the jobs before you choose Cancel All Documents. This is a good way to make enemies if people on the network were counting on their print jobs being finished anytime soon.
