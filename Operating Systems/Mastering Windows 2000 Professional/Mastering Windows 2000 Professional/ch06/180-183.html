Spooling

By default, documents for printing are spooled to the printer before they are printed. When a document is spooled, it is sent first to your hard drive, then across the network to the print server’s hard disk, and then sent to the printer from there. This allows users to get back to what they are doing as soon as possible and is called printing in the background. You can choose whether printing starts during or after spooling and turn spooling off entirely (Print Directly to the Printer), through the Advanced tab.

Windows 2000 always wants to spool the documents so that you can get control of your application faster. In other words, the hourglass goes away sooner. If you find that your printer has trouble processing print jobs with this spooling enabled, you can select the option to Print Directly to the Printer. This will turn off the spooling for the printer and send the job directly to the print device. This could solve your printing problem, but it will result in slower printing.

Setting Up Separator Pages

Another organizational tool is a separator page. Separator pages are extra pages printed before the main document. These extra pages can be used to identify the owner of the print job, record the print time and date, print a message to users of the printer, or record the job number. They’re useful for keeping documents sent to the printer separate from each other. If a number of people are using the same networked printer, separator pages help them keep their documents sorted. Several separator page files are included with Windows 2000. You can also create your own using Notepad.

Choosing a Separator Page

To specify a particular separator page, choose the Advanced tab of the Printer Properties dialog box and click Separator Page. You’ll see the following dialog box.

If no separator file is listed, you can select one by typing the full path of the file you want to use or by browsing for the correct file by clicking Browse. After you’ve selected a separator file, the page containing that information will print before every print job. To stop using a separator page, go to this dialog box and delete the entry in the text box.

When specifying a separator page, you can type a filename (if you’re already in the proper path to find the file) or the filename and path (if you’re in another path). You must, however, use a file that is physically located on the computer that controls the printer for which you’re specifying the separator page. Why can’t you use just any file that’s accessible from the network? The computer that controls the printer stores separator page information in its Registry, and so it needs that information available locally. If you tell the printer to use a separator file that is not located on its hard disk or one that is not in the path that you’ve indicated, you’ll see an error message that says, “Could not set printer: The specified separator file is invalid.”

Creating a Separator Page

To make your own separator page file, begin a new document in Notepad. (See Chapter 11, “The Other Windows 2000 Accessories,” for details on using Notepad.) On the first line, type a single character and then press Enter. This character will now be the escape character that tells the system that you’re performing a function, not entering text. You should use a character that you don’t anticipate needing for anything else, such as a dollar sign ($) or pound sign (#). Here, we’ve used a dollar sign as the escape character. Now that you’ve established your escape code, you can customize your separator page with the variables listed in Table 6.2.

Table 6.2 Printer Separator Page Variables
Variable Effect
$N Prints the user name of the person who submitted the job.
$I Prints the job number.
$D Prints the date the job was printed. The representation of the date is the same as the Date Format in the International section in Control Panel.
$T Prints the time the job was printed. The representation of the time is the same as the Time Format in the International section in Control Panel.
$Lxxxx Prints all the characters (xxxx) following it until another escape code is encountered. You can use this code to enter text exhorting people not to waste paper on unnecessary print jobs, to have a nice day, to save the planet, or anything else that you like.
$Fpathname Prints the contents of the file specified by path, starting on an empty line. The contents of this file are copied directly to the printer without any processing.
$Hnn Sets a printer-specific control sequence, where nn is a hexadecimal ASCII code sent directly to the printer. To determine the specific numbers, see your printer manual.
$Wnn Sets the width of the separator page. The default width is 80; the maximum width is 256. Any printable characters beyond this width are truncated.
$U Turns off block character printing.
$B$S Prints text in single-width block characters until $U is encountered.
$E Ejects a page from the printer. Use this code to start a new separator page or to end the separator page file. If an extra blank separator page prints, remove this code from your separator page file.
$n Skips n number of lines (from 0 through 9). Skipping 0 lines moves printing to the next line.
$B$M Prints text in double-width block characters until $U is encountered.

For example, consider the following Notepad separator page file: 

$
$N
$D
$L This is a test separator file.
$T
$E

The file results in this output: 

Mark 4/11/1994 This is a test separator file. 9:21:22 AM

Notice that, even though Enter was pressed after each entry, the output is all on one line. This is because we didn’t use the $n character to tell the separator page to skip lines between entries.
