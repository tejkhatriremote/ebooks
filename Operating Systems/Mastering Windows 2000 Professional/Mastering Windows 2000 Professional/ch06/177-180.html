To retrieve a printer port:

1.  Start the Registry Editor (regedt32.exe) and go to the following subkey in the handle key HKEY_LOCAL_MACHINE on Local Machine subtree: 
Software\Microsoft\Windows NT\CurrentVersion\Ports
2.  From the Edit menu, choose Add Value.
3.  In the Value Name box, type the printer port that you deleted (for example, COM1: or LPT1:). Keep the default value in the Data box (REG_SZ), and click OK.
4.  In the String box, type the appropriate settings that correspond to the port you typed in the Value Name box. For example, you would type 9600,n,8,1 for any COM ports. Table 6.1 tells you exactly what the values in each section should be for each port.
Table 6.1 String Values For Different Ports
Port Name Data Type String Value
COM1 REG SZ 9600,n,8,1
COM2 REG SZ 9600,n,8,1
COM3 REG SZ 9600,n,8,1
COM4 REG SZ 9600,n,8,1
FILE REG SZ empty
LPT1 REG SZ empty
LPT2 REG SZ empty
LPT3 REG SZ empty
5.  Exit the Registry Editor. Log off and then log back on, and the port should be back.

Configuring a Port

If the printer you are setting up is connected to a parallel port, you can specify the time lapse before the Print Manager decides that the printer is not responding and notifies you (as the user) of an error. Setting the Transmission Retry number higher or lower adjusts the amount of time that the Print Manager will wait for a printer to prepare itself to accept data.

This setting affects not only the printer that you’ve selected, but also any other local printers that use the same printer driver. Click the Configure Port button in the Ports tab to set the time-out value.

Setting Availability Times

You can choose not to have the printer always available (the default) by clicking the Available From radio button next to Available and filling in the hours the printer can produce output. To adjust print times, click the up and down arrows of the From and To boxes, or type in the times that you want the printer to be available. If a print job is sent to a printer during its “off” hours, that job won’t disappear, but it will sit in the queue until the printer is authorized to print again.

Although you can set user logon hours and printer hours, you must set the same printer hours for all groups and users who have access to a particular printer. For example, you can’t restrict a group of users to accessing a printer only between 9 A.M. and 5 P.M. if the administrators can access it at any time. There are exceptions to this rule, however: You can adjust users’ logon times so that they are forcibly disconnected when their logon hours expire. You can also set up two printer names: one name for the window of time when the printer is open for use and the other name with a different time window. Those who connect to the printer name with the limited hours will be able to use the printer only during those hours.

Using Scheduling

Let’s look at some of the ways you can use scheduling. Imagine that two people (Jose and Fred) need different access rights to different printers. You, as the printer administrator, want Jose to be able to use the ink-jet printer all the time, but to use the laser printer only on Fridays when the graphics department goes home early. You want Fred, who is in the graphics department, to go home early instead of racking up overtime pay and therefore you don’t want him to use the laser printer at all on Fridays. Fred still needs access to the ink-jet printer on Friday mornings and the rest of the week.

Because both Jose and Fred can have access to the ink-jet printer all week, no special restrictions need to be made. The laser printer is another matter. Here, you need to set up two printers with different access times for the laser printer. You call one Week Use Laser Printer, share it with the network, restrict access times to Monday through Thursday, and connect Fred’s workstation to that printer. You then set up another printer with the same specifications, except it’s called Friday Use Laser Printer, restrict the hours of use to Fridays and connect Jose’s workstation to that printer (which is physically the same printer as Fred’s laser printer).

Setting Printer Priorities

If you want to share your printer with the network but don’t want everyone else’s print jobs crowding out your own, you can give the printer two names: one name for you to use that has a high priority and a name for everyone else to use that has a lower priority. To further hone print priorities for different user groups, you can give the printer three or more names, each with its own priority attached, and then assign each group access to a printer name according to their priority.

Priorities decide where the document goes in line for the printer; it won’t interrupt a document that is already printing. This means that when a high priority document enters the queue, it jumps over the lower priority jobs that are waiting in line. As soon as the print device is available, the high priority job will be printed. When you’re setting the printer priorities, don’t forget that higher numbers (up to 99) have priority over lower ones. The default value is 1.

If you like, you can send print jobs directly to the port connected to the printer. If you do this, however, you won’t be able to use your application until the print job is done.
