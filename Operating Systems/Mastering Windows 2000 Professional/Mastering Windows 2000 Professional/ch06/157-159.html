About Adding Printers

Before running the Add Printer Wizard, let’s consider when you would need to add a new printer to your Windows 2000 configuration:

•  You’re connecting a new printer directly to your computer.
•  Someone has connected a new printer elsewhere on the network, and you would like to use it from your computer.
•  You want to print to disk files that can later be sent to a particular type of printer.
•  You want to set up multiple printer configurations (preferences) for a single physical printer so you can easily switch between them without changing your printer setup between print jobs.

Notice that a great deal of flexibility exists here, especially in the case of the last item. Because of the modularity of Windows 2000’s internal design, even though you might have only one physical printer, you can create any number of printer definitions for it, each with different characteristics (and different descriptive names).

NOTE:  These printer definitions are actually called printers, but you can think of them as printer names, printer aliases, or named virtual devices.

For example, you might want to set up one definition to print on legal-sized paper in landscape orientation and another to print with standard-sized paper in portrait orientation. Each of these two “printers” would use the same physical printer to print. While you’re working with Windows 2000’s manual, Online Help, and this book, keep this terminology in mind. The word printer often doesn’t refer to a physical printer. It usually means a printer setup that you’ve created with the wizard. It’s a collection of settings that typically points to a physical printer, but it could just as well create a print file instead of a paper printout.

About Printer Drivers

Also consider that a printer can’t just connect to your computer and mysteriously print a fancy page of graphics or even a boring page of text. You need a printer driver. The printer driver (a file on your hard disk) translates your text file to commands that tell your printer how to print your file. Because different brands and models of printers use different commands for such actions as “move up a line, print a circle in the middle of the page, print the letter A,” and so on, a specialized printer driver is needed for each type of printer.

DOS programs running under Windows 2000 also require a print driver for applications. For instance, WordPerfect 5.1 running in a DOS session under Windows 2000 uses a DOS printer driver and a Windows 2000 printer driver to print.

NOTE:  Because some printers are functionally equivalent, a driver for a popular brand and model of printer (for example, an Epson or a Hewlett-Packard) often masquerades under different names for other printers.

When you add a printer, you may be asked to choose the brand and model of printer. Windows 2000 has the benefit of Plug-and-Play technology, and can usually detect your printer correctly. If you own a printer that cannot be detected by Windows 2000, you will need to tell the system what kind of printer you have so the correct driver can be installed.

A good printer driver takes advantage of all your printer’s capabilities, such as its built-in fonts and graphics features. A poor printer driver might succeed in printing only draft-quality text, even from a sophisticated printer.

If you’re the proud owner of some offbeat brand of printer, you may be alarmed when you can’t find your printer listed in the box when you run the wizard. But don’t worry, the printer’s manufacturer might be able to supply a driver. The procedure for installing manufacturer-supplied drivers is covered later in this chapter, in the section titled “Adding a Printer That Is Not on Windows 2000’s List.”

About the Add Printer Wizard

Microsoft has made the previously arduous chore of adding a printer somewhat simpler, though it is still a multistep process. You may be adding a network print server or a local computer, which is one that is attached to your computer. We’ll start with the procedure for adding a network printer.

Adding a Network Printer

Assuming your Windows 2000 network system is successfully cabled and running, network printing should be possible. Before a network user can access a network printer, the following must be true:

•  The printer must be cabled to the sharing computer. If the printer has a direct network interface, then the sharing computer must be on the same network as the printer.
•  The printer must be created; that is, you need to complete all the steps in the Add Printer Wizard to create the printer, and it must be working properly for local use.
•  The printer must be shared.
•  The security settings for the printer and network users wishing access must match.

By default, new printer shares are given a security setting that gives all users (Everyone) access for printing. Only the creator, power users, and administrators can manage the printer, however. Managing the printer means rearranging the print-job queue: starting, stopping, and deleting print jobs.

Connecting to a shared printer using Windows 2000 is so easy that it’s tempting to believe that you didn’t do it right until you print successfully. To connect to a printer on the network, all you do, in essence, is point to the printer you want to use and say: “Print here.” Of course, the print server administrator has control over who can use the printer you just pointed to, but apart from giving the print server administrator flowers and candy, you have no control over his printer settings. This makes connecting to a networked printer easy, and it also makes Windows 2000 flexible and controllable (for the printer administrators).

Because Windows 2000 workstations use the printer drivers stored on the print server, even if you install a newer version of a driver on the print server, the Windows 2000 workstations will use it automatically. You don’t need to tweak the workstation connection.
