What’s special about this wiring system? Token Ring has the following special features that make it a good choice for many offices:

•  It’s easy to troubleshoot.
•  It can use a smart hub (not an IBM standard, but can work with Token Ring) to remove faulty nodes from the network without disrupting transmissions.
•  It has a transmission design that ensures that one node can’t tie up the network.

One of the potential problems with Token Ring is that although it’s better equipped to deal with a broken cable connection than Ethernet is, if one adapter in a Token Ring network fails, so does the entire segment. This doesn’t happen often, but it’s a big enough pain if it does, so it might be worth the extra money to get a smart hub. A smart hub can tell if there’s trouble and can remove the problem node from the network without disrupting traffic (IBM’s standard MAUs aren’t capable of this). Token Ring is also expensive to install. Because Token Ring is less popular than Ethernet, technology for Token Ring typically arrives on the market anywhere from three to six months after it arrives for Ethernet. This can be a disadvantage if your organization must always have the latest and greatest.

Ethernet Options

10BaseT and 10Base2 have the same maximum transmission rate—about 10Mbps—but, as always, you should remember that a lot of factors affect network speed besides transmission rate over the cable. If your network seems slow, don’t blame your wiring system first.

10Base2

This wiring system uses both the logical and physical bus topologies. Using thin coaxial (RG-58A/U) cable, it connects the nodes of the network in a daisy chain.

Why use the 10Base2 standard? It has two major advantages. First, it uses significantly less cable than a network using a star topology, such as 10BaseT. Second, it doesn’t require a hub to operate, so you don’t need to allocate room, power, or money to connect all the network nodes—they connect directly to each other.

10BaseT

When discussing the bus physical and logical topologies, we will mention 10BaseT as an example of a network type that uses a bus logical topology and a connected-star physical topology. 10BaseT is more commonly used in physical star topologies. 10BaseT is a special type of Ethernet, as it runs on twisted-pair cable rather than coax; 10Base2 is the more traditional bus-style Ethernet.

By way of review, the 10 stands for 10Mbps, the Base stands for baseband transmissions (one pathway for data to go across), and the T stands for twisted-pair. Keep in mind that the distance limitation for twisted-pair cable is 300 feet without a repeater.

Theoretically, the advantage of the 10BaseT standard is that it could enable you to use existing twisted-pair telephone wire and thus save on cabling costs. Practically, most LAN planners find that they have to pull any other type of wire you may have (wrongly) anticipated they might need. In any case, when installing 10BaseT, consider that more people are familiar with twisted-pair than with coax, so you could save money on hiring the technicians you need. A bigger potential advantage of 10BaseT is that troubleshooting is easier than it is in a physical-bus LAN. Like the spokes of a wheel, the LAN’s cables extend from a wiring closet to the nodes of the network. If one cable breaks or shorts, that node is out of commission, but the rest of the LAN is unaffected.

Other Ethernet cabling options, such as 10Base5 and 10Broad36, appear from time to time but are much less common than the two options described above.

100BaseT

Because 100BaseT has become so popular, it bears mentioning here. 100BaseT is functionally just like 10BaseT in that it uses Category 5 UTP cable and works with the same topologies and similar hardware. What’s the difference? Speed. 100BaseT functions at 100Mbps, 10 times faster than 10BaseT networks. It does require Category 5 UTP as a minimum, and Network Interface Cards, hubs, and routers that are specially designed to work at these speeds.
