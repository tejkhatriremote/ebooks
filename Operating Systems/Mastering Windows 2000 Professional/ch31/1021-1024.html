Device Manager

The Device Manager applet assists you in managing the many device drivers Windows 2000 uses to communicate with the hardware installed in your system. A quick look at the Device list can tell you whether a particular device driver is loaded and running.

If a device is malfunctioning, its console tree will be expanded and the icon will have either a red splat or yellow bang on it. The red splat is a common slang term for a device displaying the red X through the icon, and the yellow bang means a device with the yellow symbol containing an exclamation point. If you see one of these, you’ll know there is a problem with either the device or its driver.

In Figure 31.14, a yellow exclamation point shows the Multimedia Video Controller device, or its driver, has a problem. If you highlight that device, there will be a button on the right side of the toolbar that will let you re-enable the device, or uninstall its driver entirely.


Figure 31.14  Device Manager lets you enable or disable devices.

Device drivers have specific start-up values that determine when they should be loaded and started during the boot process. In earlier versions of Windows NT, you could determine those start-up values manually. In Windows 2000, that’s a bit more difficult since this is normally handled through Plug and Play. The device reports what it needs from the operating system, and Windows 2000 tries to provide that. If you ever find that you need to change these values, you will have to do so through the Registry. The following list describes the start-up values.

Boot
Indicates that device drivers load first; this setting should be used only for devices that are necessary during the boot process.
System
Indicates that device drivers load when the system loads; this setting should be used for devices needed to get the system up and running.
Automatic
Indicates that device drivers get loaded after the boot and system devices; they are not considered necessary for system start-up.
Manual
Stipulates that a device will be started only on demand, either by explicit user request or by another device calling for the use of this device.
Disabled
Means that the device cannot be started by users; however, the operating system itself may start the device when necessary.

If you happen to be using multiple hardware profiles, then each profile will be listed by name in the Properties sheet for a device. Simply check the box beside each profile name where the device should be used.

Services Console

The main difference between a service and an application is that a service loads whenever the system is booted, regardless of whether any user logs into the system. Examples of services include tape backup services, Web server services, and mail services. You can get involved, however, and start, stop, pause, or resume specific services by means of the Start, Stop, Pause, and Restart buttons in the Services console (see Figure 31.15). As with devices, services can be also be enabled or disabled automatically for a given hardware profile.


Figure 31.15  The Services console lets you specify when various services should be started, and which logon account they will use.

Because services can start prior to or irrespective of who logs in, there needs to be some way to specify which user account a service should log in with. Normally, services will default to use the System account, but in some cases this creates a security risk. For example, if a Web server starts up using administrator privileges and allows server-side includes, it is entirely possible for someone out on the Internet to gain complete access to the system, because any program started via the Web server gains the same security rights as the Web server itself.

So how do you change the login account for a service? Follow these steps:

1.  Start by selecting the service in the Services applet.
2.  Double-click the service. This will bring up the Computer Browser Properties dialog box shown in Figure 31.16.


Figure 31.16  Services usually default to logging in via the system.

3.  Click the This Account radio button, then click the Browse button beside it. This brings up the Add User dialog box.
4.  Within the new dialog box, select the user account the service should use when it logs in, then click OK.
5.  As a final security measure, you should type the password for the user account you’ve selected. This prevents other users from giving access rights to that service.
