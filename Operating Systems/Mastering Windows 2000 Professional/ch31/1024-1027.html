The Recycle Bin’s Properties

Recall that when you delete files, they aren’t actually erased (unless you remembered to press the Shift key at the same time as the Delete key when you deleted them). They get moved to the Recycle Bin, which is a directory on your hard disk. If your hard disk is modest in size or is getting crammed, decreasing the size of the Recycle Bin might be in order. If you decrease its size, though, remember that you’ll be cutting down on the number (and/or the size) of the files that can be recovered. In some cases, you might even want to increase the size of the Recycle Bin to keep yourself from worrying about whether you’re making the right decision when clearing off certain projects (for instance, if you’ve ever worked for clients who change their mind after giving their “final” instructions). The options in the Recycle Bin’s properties sheet are therefore worth a look.

To get to the properties sheet, right-click the Recycle Bin icon, either on the Desktop or in Explorer, and then choose Properties. The Recycle Bin settings appear as shown in Figure 31.17.


Figure 31.17  You can set specifics of the Recycle Bin’s behavior from its properties sheet.

There are a number of settings here to contemplate. First, notice that in addition to a Global tab, there is also a tab for each of your local hard disks. If you choose Configure Drives Independently on the Global page, you’ll have to go to each tab as needed to make individual drive settings. Otherwise you can just use the Global page.

If you don’t want to be able to reclaim deleted files, check the Do Not Move Files to the Recycle Bin option. Why would you do this? For one thing, it speeds up the deletion process. For another, it frees up disk space immediately on deletion. Finally, if you’re paranoid that someone else is going to access your system and expose your old mistakes, it destroys the evidence.

As a default, 10 percent of each drive is used for the Recycle Bin. When you delete a file on drive C, it goes to drive C’s Recycle Bin until you empty the Bin. When you delete from drive D (if you have a second hard drive), it goes into drive D’s Recycle Bin. As a Recycle Bin reaches capacity, newly deleted files will push older files off the bottom of the list, deleting them permanently. In practical terms, the Recycle Bin’s size determines how long a file will be recoverable before it’s pushed off the list.

WARNING:  The size of a drive’s Recycle Bin unfortunately limits the size of the largest file that can be recovered after an accidental deletion. If a file is larger than the maximum size of the Recycle Bin, then it won’t be recyclable. As of this writing, you won’t be warned of this either; it will just be erased.

If you want to alter the size of your Recycle Bins, use the percentage slider. Usually, the default of 10 percent works just fine, but if you work with very large files, you may want to increase this percentage to accommodate them. If you’re short on disk space and don’t tend to make deletion mistakes, decrease it.

The last check box in the Recycle Bin properties sheet, Display Delete Confirmation Dialog, is normally checked so that you can confirm before Windows 2000 will empty the bin. If you find the confirmation boxes annoying, turn off this check box.

Here are a few other points you might want to keep in mind concerning the Recycle Bin in Windows 2000:

•  If you delete a folder, only the files within that folder appear in the Recycle Bin—the folder itself will not appear. However, if you restore a file that was originally located in a deleted folder, Windows 2000 recreates the folder and then restores the file to it.
•  Windows 2000 maintains separate Recycle Bins for each user on an NTFS partition. If you use Explorer to list the folders on an NTFS drive, you will find a folder called the Recycler. The Recycler contains individual Recycle Bins for each user that has logged onto the computer list by the Security ID of the user.
•  When you empty the Recycle Bin, all the bins are flushed. You can’t empty them on a drive-by-drive basis unless you use a file-management program other than Explorer, such as the old Windows File Manager (which at last report was still shipping in Windows 2000—to run File Manager, click Start Run and type winfile). You can use Explorer to delete specific files from the Recycle Bin if you select and delete the files individually, however.
•  If you delete files via the command prompt (typing del or erase), these files are immediately erased. If you ever need to restore these files (or files deleted from the Recycle Bin), there are third-party tools available that will work with Windows 2000, but Windows 2000 itself provides no such means of “undeleting” files that were deleted via a DOS command.
