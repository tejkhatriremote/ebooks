The Management Console

We’ve spent a lot of time in this book referring to the Microsoft Management Console (MMC), it seems like a good idea to look at this new tool a little closer. The MMC is the answer to a common complaint in Windows NT, namely “Why can’t we have one tool to do all of our administration?” Well, someone at Microsoft was listening, though whether this turns out to be a good solution remains to be seen.

You can get to the MMC in a variety of ways. The easiest way, if you just want to look at the tool itself, is to use the Run command and type in mmc. This opens an empty console. This brings up the question, empty of what? Well, the MMC normally contains things called snap-ins, which enable you to actually do some work with Windows 2000. To add a snap-in to the MMC use the following steps:

1.  Open the MMC by clicking the Start button and selecting Run. Type mmc, and click OK.
2.  From the Console menu select Add/Remove Snap-in. This will open the Add/ Remove Snap-in dialog box. Click Add.
3.  Select the snap-in you want to load from the list, and then click the Add button.
4.  To add additional snap-ins, repeat steps 2 and 3 as many times as necessary.

If you need help when using any snap-in in the MMC, click the Action menu and select the Help command.

So, what happens if you’ve gone through all the trouble to really customize the snap-ins in the MMC and then you close it? Do you lose all of your work? Not if you save the settings when you exit. To save your workspace in the MMC, click the Console menu and select Save As. You will be prompted to provide a name for the saved file. All saved MMC files end with a .msc extension.

You can also open your saved MMC files from the command prompt using the following syntax:

   MMC path\filename /a /s

The syntax is defined as follows:

MMC
The command to open the Microsoft Management Console.
path\filename
Defines the path to the saved MMC file and its name.
/a
Opens the console in Author mode. This will enable you to add or remove snap-ins from the saved console. This doesn’t change the default mode for the file, it simply overrides it for one session.
/s
Prevents the display of the splash screen. This switch is a holdover from the first version of the MMC, which would display a splash screen every time it opened. This switch is not necessary in Windows 2000.

The MMC can be used to monitor both locally and remotely. If you choose to open a saved MMC file for a remote computer, you do it the same way. Simply enter the path to the remote .msc file at a command prompt and press enter. You must have network connectivity to the remote computer as well as the necessary permissions to run the MMC on that computer.

Final Suggestions

In Chapter 27, “Supporting New Hardware and Installing Drivers,” and Chapter 28, “Running Windows 3.x and DOS Programs,” we touched on some basic Windows 2000 software and hardware compatibility issues, some of which affect system throughput and overall performance. Although Windows 2000 Professional has evolved through many changes and improvements, unfortunately a genie in a bottle isn’t included. The old tried-and-true performance essentials are still applicable. We could write a hundred pages about keeping your hardware doodads up to snuff, your hard disk organized, and other tricks for squeezing the most out of every CPU cycle. But in the end, most people don’t bother, and it doesn’t really net you much. In reality only a few truly beneficial performance tricks are worth the hassle or price:

•  First off, always keep a lot of free space available on your hard disk. This is essential. You should have at least 200MB (some people say as much as 500MB) free on your disk for use by Windows for virtual memory. If you have too little, you’ll start getting messages about running out of memory when you try to switch between programs. If you use only a few small programs at a time, this is less of an issue.
•  If you are unhappy with the performance of your system and it’s a pretty fast Pentium II-level system already, then increase the amount of RAM. Most Pentium computers today come with 64MB. You will find that 128MB (or more!) will dramatically improve things.
•  If you don’t have a local-bus hard-disk controller, switch to one. SCSI III is best; IDE on the VLB or PCI bus is good. You may be able to keep your existing drive, but make sure it’s large enough and has a fast step rate (12 milliseconds or lower).
•  Your video controller should be on the local bus too. If you can live with only 16 colors, use the Display applet in Control Panel to reduce the colors. Screen redrawing will usually speed up significantly.
