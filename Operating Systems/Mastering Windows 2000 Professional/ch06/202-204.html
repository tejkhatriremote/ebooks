Troubleshooting

If your print job isn’t coming out quite like you expected (or at all), make sure that you’ve got the printer set up correctly. Have you specified an existing printer port? What about the printer driver—did you install the right one for your printer? If so, you may want to try deleting and reinstalling the printer driver. (Remove the printer from the system and re-create it.)

The problem could also be in the application. Can you print to another printer? Can you print to that printer from another application? If it’s a DOS application, try copying the driver file to the application’s directory and then try to print again.

It’s also possible that something could go wrong with the printer itself, or the cabling, especially if you can’t print from DOS. After you’ve checked the printer to make sure that it’s online, plugged in, and has paper, check the connection. Will the printer work when attached to another parallel or serial port? If so, you could have a network problem. Try printing the file directly from the print server. If nothing else works, try removing extraneous cards from the print server to see if it’s a hardware conflict. Windows 2000’s capability to network printers is one of its great strengths. With a firm grasp of the Print Manager, you can make sure that all users have easy access to printing services.

If you’re printing through a serial port, you’ll see a Printer Out of Paper message when any one of the following things is wrong:

•  The printing device is out of paper.
•  The printing device is offline.
•  The printing device is switched off.
•  The cable to the printing device is disconnected.

You get the message because serial connections don’t have the status lines available to communicate exactly what the problem is. When something has gone wrong, you know only that there’s a problem, not what the problem is. The default message is that the printer is out of paper, but you need to check all the other options if you’re using a serial connection.

Always remember to check the simple things first. There are two questions that should always be asked (gently and carefully) when your print job will not come out on the printer. The first is, “Is it plugged in?” Check the cable from the computer to the printer and the power cord. The second question is, “Is it turned on?” Some printers have their power switch in hard-to-find locations. Asking yourself these questions can often fix the problem. You’d be amazed how often this solves printing issues. If your document won’t print from your favorite application, can you print from Notepad? Notepad is the simplest printing that you can get in Windows 2000. If you can print from Notepad, but not from your application, then your application is to blame. For more information on troubleshooting printing, please refer to Chapter 32, “Advanced Troubleshooting Methodology.”

Summary

Printing is an everyday task we often take for granted. Windows 2000 Professional makes this task easy once you have learned some of the techniques covered in this chapter.

Windows 2000 provides an easy-to-use wizard to add printers to your computer and to share them for use on the network. It also provides the printer drivers to other users automatically. After a printer has been shared for network use, a person with the correct permissions can adjust the queue, manage documents, and even manage the printer itself.
