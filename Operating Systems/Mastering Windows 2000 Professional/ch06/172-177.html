Creating a Second Printer for the Same Print Device

It’s often desirable to have two (or more) names for the same print device you are sharing with others. Having different people access the same device from different names enables you to assign different printing priorities to different users, specify different hours that the printer is available for printing, make one printer for network use and another for local use, and so forth. Having two or more names enables you to fine-tune the network’s access to the printer.

The process of creating a second printer for the same print device is identical to that of creating the first one, but make sure that you select a new name for the additional printer. Windows 2000 will remember the last printer you installed and automatically pick the same manufacturer and model when you install the second printer. Be sure to give the new printer a different name or Windows 2000 won’t actually create the printer.

Choose the same printer driver for the printer that the other printer on this print device uses, and make sure that all other settings are as they should be. You don’t need to share all printers you create, even if they’re attached to the same printing device. If you do choose to share the new printer, it will be available to other workstations on the network.

Creating a Printer Pool

Just because you send a print job to a particular printer name doesn’t mean that that print job must print to one particular printer. To save time for print jobs, you can use the Details dialog box to pool several identical printers into one logical one. If you do this, when you send a print job to that printer name, the first available printer will do the job. This is called printer pooling.

Printer pooling will not work unless the pooled printers are physically identical—the same make and model and the same amount of memory.

To set up printer pooling, follow these steps:

1.  In the Printers folder, right-click the printer’s icon and choose Properties.
2.  Select the Ports tab and click Enable Printer Pooling.
3.  Select the ports that correspond to the ports where you’ve plugged in the other printers. If the ports you need aren’t on the list, you can add them by clicking the Add Port button (discussed in the next section).
4.  When you have added all of the ports, click OK to apply the changes. Any job sent to this printer will now go to the first available port.

TIP:  You can actually create a printer pool with dissimilar print devices if they emulate a common printer. Install the driver for the emulation and adjust the physical printers to use the emulation. If they aren’t identical physically, this may work, but you may receive varying levels of success.

Customizing Your Printer

The settings pertaining to a printer are called properties. As we’ve discussed earlier, properties abound in Windows 2000. Almost every object in Windows 2000 has properties that you can examine and change at will. When you add a printer, the wizard makes life easy for you by giving the printer default properties that usually work fine and don’t need to be tampered with. You can change them later, but only if you need to. However, it may be worth looking at the properties for your printer, especially if the printer’s acting up in some way when you try to print from Windows 2000. Here are the steps:

1.  Open the Printers folder.
2.  Right-click the printer’s icon and choose Properties.

TIP:  You can also press Alt+Enter to open the Properties dialog box. This is true with many Windows 2000 objects.
3.  Click the various tab pages of the Printer Properties dialog box to view or alter the great variety of settings. It may not be clear to you what some of these settings do, but you can usually get help by clicking the ? button in the upper-right corner and then clicking the setting or button whose function you don’t understand.

The following sections describe some of the options in the tabs of the Printer Properties dialog box.

Using the General Tab

The first tab of the Properties sheets for any printer is the General tab. On this tab, you will find the basic information that is published to the Directory if the printer is shared as well as buttons to set the printing preferences and to print a test page. Adding a Port

Adding a port is relatively straightforward. Just click the Add Port button, and you’ll see the following dialog box.

Here, you can choose from two types of ports: Local Port or Standard TCP/IP Port. Double-click the type of port to add the driver. If you want to add a different type of port, click the New Port Type button. You will need to provide an appropriate information file for the new port type.

Deleting a Port

The Delete Port option is self-explanatory. After you select a port and choose this option, Windows 2000 lets you change your mind by asking if you really mean it (see the following illustration). Windows 2000 won’t allow you to delete a system port such as the physical parallel port LPT1. If you do manage to delete a port that you need, you should be able to add it again by using the previous procedure to add a new port. If that doesn’t work you will need to edit the Registry directly to recover the port.

Retrieving a Port

To retrieve a printer port (perhaps after accidentally deleting it), you must edit the Registry. Don’t take on this task unless you feel like a true NT power user. This is not a task for 99.9 percent of end users.

WARNING:  Be careful when using the Registry Editor! It’s very possible to completely screw up your Windows 2000 installation with it. In fact, you could screw it up enough that you have to reinstall NT to fix the problem. See Chapter 21, “The Registry,” for more information about the Registry Editor.
