Adding a Printer That Is Not on Windows 2000’s List

When you’re adding a local printer, you need to supply the brand name and model of the printer because Windows 2000 needs to know which driver to load into your Windows 2000 setup to use the printer correctly. When you are adding a network printer and the correct printer driver is installed on the print server, you will get the driver automatically. If the print server doesn’t have a driver for your printer, you need to follow the steps on the next page to install a copy of the driver locally on your computer.

What if your printer isn’t on the list of Windows 2000-recognized printers? Many off-brand printers are designed to be compatible with one of the popular printer types, such as the Apple LaserWriters, Hewlett-Packard LaserJets, or the Epson line of printers. Refer to the manual that came with your printer to see whether it’s compatible with one of the printers that is listed in Windows 2000. Some printers require that you set the printer in compatibility mode using switches or software. Again, check the printer’s manual for instructions.

Finally, if it looks like there’s no mention of compatibility anywhere, contact the manufacturer for the Windows 2000-compatible driver. If you’re lucky, the manufacturer will have one. It’s also possible that Microsoft has a new driver for your printer that wasn’t available when your copy of Windows 2000 was shipped. Contact Microsoft at (206) 882-8080 and ask for the Windows 2000 Driver Library Disk, which contains all the latest drivers. Another option is to look for the latest drivers on the Internet, either at Microsoft’s site (http://www.microsoft.com) or at your printer manufacturer’s Web site.

Assuming you do obtain a printer driver, do the following to install it:

1.  Follow the instructions in the preceding section for adding a local printer.
2.  Instead of selecting one of the printers in the Driver list (it isn’t in the list, of course), click the Have Disk button. You’ll see a dialog box asking you to enter the path where the driver is located (typically a floppy disk). The dialog is shown in the following illustration.

3.  Insert the disk (or make sure the files are available somewhere), enter the path, and click OK. Enter the correct source of the driver. Typically, it will be in the A or B disk drive. Then click OK. You might need to choose a driver from a list if multiple options exist.

NOTE:  The Add Printer Wizard is looking for a file with a .inf extension. This is the standard file extension for manufacturer-supplied driver-information files.
4.  Continue with the wizard dialog boxes, as explained in the preceding section.

If none of the drivers you can lay your hands on will work with your printer, try choosing the Generic text-only driver. This driver prints only text—no fancy formatting and no graphics—but it works in a pinch with many printers. Make sure the printer is capable of or is set to an ASCII or ANSI text-only mode; otherwise, your printout may be a mess. Note that PostScript printers don’t have such a text-only mode. For a Postscript laser printer, you can try the Apple LaserWriter for emulation. Most Postscript printers can use the LaserWriter’s drivers to print.

Sharing a Printer for Network Use

You need to share a printer before it becomes available to other network users. Sharing is pretty simple:

1.  Add the printer as a local printer, as described earlier in this chapter.
2.  Right-click the printer’s icon and choose Properties or Sharing, as shown in the following illustration.

3.  You’ll see the Printer Properties dialog box. Click the Sharing tab if it’s not selected. You’ll see the dialog box shown in Figure 6.8.


Figure 6.8  The Sharing tab of the Printer Properties dialog box lets you turn on and off printer sharing.

4.  On the Sharing tab, click Shared As.
5.  A share name based on the printer name is automatically generated. You can leave the share name as is or change it. DOS-based network users see this name, which must conform to DOS file-naming rules. Other users will see this name too, although some may see the other comments about the printer as well. If there are DOS-based clients on your network, this name can’t be more than 12 characters and cannot contain spaces or characters not acceptable in DOS filenames, such as ?, *, #, +, |, \, /, =, >, <, or %.
6.  Now click the General tab. Notice that there is a place for a comment. This is normally blank after you add a printer. If you share the printer on the network, any text that you add to this box will be seen by other users who browse the network for a printer.
7.  While on the General tab, fill in some information for the Location of the printer. This information, along with the comment entered in step 6, will be published to the Directory for other users to see when browsing for a printer.

TIP:  You have the opportunity to preinstall printer drivers for other Microsoft operating systems (Windows 9x and all flavors of NT). This can be a great time-saver for the other printer users because they will not need to supply the driver files from their computer. For the same reason, this helps eliminate one more thing that could go wrong for the users on the network.
8.  Click the OK button, and your printer is shared! Its icon now has a hand under it, as shown in the following graphic. Pretty soon you’ll start to see unexpected print jobs rolling out of your printer.

Preventing a Printer from Being Shared

To prevent the sharing of your printer, open the Printer Properties dialog box, choose the Sharing tab, and click Not Shared.

Another approach to preventing overuse of a given printer (at least temporarily) is to right-click its icon and choose Pause Printing. It may not be a very elegant way to deter people from printing to your printer, but it will stop print jobs. Of course, anyone viewing the print queue will be able to resume printing, so this technique may not buy you much time. Not sharing the printer, as mentioned above, will prevent other users from printing to your printer.
