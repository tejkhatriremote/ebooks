Auditing Printer Usage

To get a good idea of what is happening with your printer, you can audit its use. Perhaps the printer is overused, or perhaps you want to know who is using the printer after regular work hours. Auditing these events gives you the answer. To set up auditing, first go to the User Manager and enable File and object access auditing. (To do this, go to the Policies dialog box and choose Audit Policy.) After you’ve done that, you can set up printer auditing for individuals and groups. If you are unsure if auditing is turned on, you’ll find when you attempt to use the feature and a message telling you that auditing policy is not turned on is generated.

After you’ve turned on auditing, you can choose what to audit in the Printer Auditing dialog box, shown in Figure 6.15. To get to this dialog box, choose the Security tab in the Printer Properties dialog box, click the Advanced button and select the Auditing tab of the Advanced Properties dialog.


Figure 6.15  The Printer Auditing dialog box

You will need to add a user or group account to audit. To do this, click the Add button, and you’ll see the dialog box shown in Figure 6.16.


Figure 6.16  The Select User, Computer, or Group dialog box

To select a group for auditing, double-click it or click once and then click OK. When you’ve selected a user or group, the Auditing Entry dialog should open as shown in Figure 6.17. When you’ve chosen all the appropriate options to audit, click OK to return to the Printer Auditing dialog box. It will now look something like Figure 6.18.


Figure 6.17  The Auditing Entry dialog lets you set the individual success or failure audits for the selected user or group.


Figure 6.18  The Printer Auditing dialog box after choosing the users or groups to audit

For each group or user that you’ve chosen to audit, you can select different items to keep track of by checking the appropriate check boxes. You can choose to audit successful and/or failed attempts by that user or group to

•  Print a document
•  Manage the printer’s settings
•  Manage someone’s documents
•  Read the permissions for the printer
•  Change a user’s permissions for the printer
•  Take ownership of the printer

When you highlight an account in the Auditing Entries box, you’ll see some basic information for the type of audit (success or failure), the account’s permission level, and whether the audit applies to the documents or printer or both. To see the actual settings, highlight the account you want to view then press the View/Edit button to display the Audit Entry dialog. To view the auditing information garnered by this method, use the Event Viewer in the Administrative Tools program group. You can read more about interpreting the Event Viewer data in Chapter 30, “Administrative and Diagnostic Tools.”

Deleting a Printer from Your Printers Folder

You might want to decommission a printer after you’ve added it for several reasons, including the following:

•  You’ve connected a new type of printer to your computer and you want to delete the old setup and create a new one with the correct driver for the new printer.
•  You want to disconnect from a network printer that you’re finished using.
•  You’ve created several slightly different setups for the same physical printer and you want to delete the ones that you don’t use.

In any of these cases, the steps are the same:

1.  Open the Printers folder (the easiest way is to use Start Settings Printers).
2.  Right-click the icon for the printer setup you want to delete and choose Delete (or press the Delete key).

3.  You will see at least one confirmation box before the printer is deleted. You may see another warning if print jobs are in the queue for the printer. Verify that you want to remove this printer.

Because Windows 2000 stores the information for network printers according to the user’s profile, if you delete a network printer, you are deleting it only for yourself. Be careful though; the configuration for local printers is universal to all users on the local computer. If you delete a local printer, you delete it for everyone who logs on to your computer.

When you delete a printer by following the preceding steps, the related driver file and font files are not deleted from the disk. Therefore, if you want to re-create the printer, you don’t need to insert disks, and you won’t be prompted for the location of driver files. This is convenient, but if you’re tight on disk space, you might want to remove the printer fonts and drivers. To remove fonts, use the Fonts applet in the Control Panel, as described in Chapter 5, “Customizing Your Desktop via the Control Panel.”

Printing Documents from Your Programs

By now your printer(s) are added to your system and ready to go. The procedure for printing documents from your programs in Windows 2000 is no different from Windows 3.x, Windows 95, or even Mac programs. Just open the document, choose File Print, and set a few options, such as which pages to print. You might also need to set the print area first or make some other settings, depending on the program.

If you’re already happy with the ways in which you print, you might want to skim over this section. However, there are a couple of conveniences you might not know about, such as using drag-and-drop techniques to print or right-clicking a document to print it without opening the program that created it.
