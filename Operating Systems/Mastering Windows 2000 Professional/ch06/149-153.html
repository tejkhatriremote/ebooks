With the advent of computers came the notion of the “paperless office.” In fact, the goal of the paperless office was heralded as practically the eighth wonder of the world, and computers were to be the key to this promised miracle. My, how times change.

Doing without paper is a wild fantasy in most businesses today. Working completely without a printer to produce most of that paper is totally unthinkable. Go into any office and you’ll hear printers churning out reports, receipts, payroll checks, and even the boss’s daughter’s eighth-grade term paper. No, the paperless office is a miracle that is far too good to be true. The speed of the printer, color of the page, and clarity of the graphics are products much more important these days than a lack of file drawers.

As necessary as they are, however, printers are still big and expensive and printers aren’t used every minute that a computer is used. Sharing a printer between two or even twenty workstations can be a practical solution to money and space constraints. Windows 2000 Professional provides easy, efficient, and helpful tools for sharing a printer among workstations.

At times, however, not sharing a printer is important as well. For example, when the weekly newsletter is printed, all other printing tasks must take a back seat. Windows 2000 comes with security measures that can be easily arranged to give one person or group of people particular access rights and to give people access at different times of the day. If your printer seems overused, the auditing feature can provide detailed accounts of who used the printer when, so that resources can be allocated more efficiently.

Printing in Windows 2000 Professional

The first thing to get out of the way is some basic Windows 2000 terminology. In the language of Windows 2000 printing, a printer is the software that controls a print device, which is the hardware. This can be confusing, but knowing this terminology can be valuable when trying to describe printing issues. The printer is the combination of the printer driver, support files, and the print queue. The print device is the part that we can’t seem to live without and that costs all the money.

As in previous versions of Windows NT, when you choose to print something in Windows 2000, unless you indicate otherwise, Windows programs hand off data to the operating system, which in turn spools the data to a specified printer. Spooling means temporarily putting information that’s headed for the printer on the hard disk. Your document is then sent to the printer at the slow speed that the printer can handle. This lets you get back to work with your program(s) sooner. You can even print additional documents, stacking up a load of jobs for the printer to print. This stack is called a queue.

In Windows NT 3.x (and Windows 3.x), a program called Print Manager was responsible for spooling and managing the print jobs. Windows NT Workstation 4 nomenclature dispensed with the term Print Manager, even though the same functionality is provided. Now you simply look at what’s “inside” a printer by clicking the printer’s icon. This opens a window and displays the print queue for that printer. In reality, however, there is a spooler program and a Print Manager-like utility in Windows 2000, and in this chapter, that is how we’ll refer to the window that enables you to see and work with information about the print queue.

NOTE:  Print jobs from MS-DOS programs can also be spooled, so you can get back to work with your DOS or Windows programs while printing happens in the background.

When you print from a Windows program, Print Manager receives the data, queues up the jobs, routes them to the correct printer, and, when necessary, issues error or other appropriate messages to print-job originators. As in earlier versions of Windows NT, you can use the Print Manager user interface to manage your print jobs. Print Manager makes it easy to check on what’s printing and see where your job(s) are in the print queue relative to other people’s print jobs. You may also be permitted to rearrange the print queue, delete print jobs, or pause and resume a print job so you can reload or otherwise service the printer.

In Windows 3.x, printer-driver installation and configuration (making connections) are done through the Printers applet in Control Panel, and print-job management is done through the Print Manager program. Windows 2000 Professional (like Windows 9x) combines both functions in the Printers folder. (The Control Panel still has a Printers icon, but it simply opens the Printers folder.) Each printer you’ve installed appears in the Printers folder, along with an additional icon called Add Printer that lets you set up new printers. Printer icons in the folder appear and behave like any other objects. You can delete them at will, create new ones, and set their properties. Double-clicking a printer in the folder displays its print queue and lets you manipulate the queue. Commands on the menus let you install, configure, connect, disconnect, and remove printers and drivers.
