Here are the steps for connecting to a network print server:

1.  Open the Printers folder by clicking the Start button and choosing Settings Printers. Two other paths to this folder are from My Computer and from Control Panel.
2.  Double-click Add Printer, as shown below.

3.  A window like the one shown in Figure 6.2 appears. Click Next. The dialog that appears asks if the printer is connected to your computer or is a network printer. Select Network Printer and click Next. (If the printer is directly attached to your computer, skip to the next section, “Adding a Local Printer.”)


Figure 6.2  Beginning the process of adding the printer

4.  Next you’ll be given a choice of how you want to find the shared printer as shown in Figure 6.3. The three options are: Find a printer in the Directory; Type the Printer Name or Click Next to Browse for a Printer; and Connect to a Printer on the Internet or Your intranet.


Figure 6.3  When adding a network printer, select the desired server and printer.

NOTE:  You won’t see the first of these options in Figure 6.3. The option for finding a shared printer in the Directory only appears if your computer is in a native Windows 2000 network. This means that all of the Domain Controllers maintaining the security are Windows 2000 Servers using Active Directory.

If your network is using the Active Directory and the printer has been published to the Directory, the Find option is the best. To use the traditional methods of browsing for a shared printer or connecting to it using its NetBIOS name, use Type the Printer Name.

5.  If the printer you want to connect to isn’t listed in the browse list, don’t despair. You can easily type the path to the printer in the Printer box. Simply type two backslashes (\\) followed by the name of the server, another backslash (\) and the name of the printer. Browse lists are often incomplete and you can’t change that. (See Chapter 24, “Essentials of Windows 2000 Networking,” for more on browsing.) Of course this method requires that you know the name of the server and the printer, but you really should try this before calling your administrator for help. This path is referred to as a Universal Naming Convention (UNC) path, and can be entered directly from the Run command if you really want a shortcut.

NOTE:  For a printer to appear in the network listing, it must have been added to the host computer’s setup (the computer the printer is directly attached to) using the steps for adding a local printer. It must also be set as Shared for use on the network. See “Sharing a Printer for Network Use” for details.
6.  If the network printer is currently offline, you are told that you need to wait until it comes back online before you can use it but that you can go ahead and install it if you want. In this case, you may be asked to specify the brand and model of the printer, because the wizard can’t figure out the type of printer when it is unavailable for your computer to question. If you can alert the owner of the printer to put it online (that user can right-click the printer’s icon at his or her computer and turn off the Work Offline setting), you may not need to specify these settings. If this isn’t possible, just select the brand name and model from the list.
7.  Figure 6.4 shows the next dialog box, where you indicate whether you want the printer to be the default printer. The default printer is the one that programs will assume you want to print to. Some programs don’t let you choose which printer a document will print on, so setting the default printer can be important. If you want this network printer to be the default printer, remember that the computer the printer is directly connected to must be up and running for the printer to work. So, if you have a local printer on your machine as well, it might be better to make that the default printer.


Figure 6.4  The Add Printer Wizard asks if the new printer will be the default printer for Windows 2000.

8.  Finally, you’re asked if you want to print a test page. It’s a good idea to do this. Turn on the printer, make sure it has paper in it, and click Finish. You may be prompted to supply a printer driver for the printer, depending on the type of network you’re connecting to and whether a driver is already on your machine. If you’re told that a driver file for the printer is already on your machine, you are asked if you want to use it or load a new one from the Windows 2000 CD-ROM. Using the existing driver is usually easier. If the driver isn’t on your hard disk, you are instructed to insert the disk containing the driver. The last thing the wizard will do for you is display a list of the options you’ve chosen to install for the printer, as shown in Figure 6.5.


Figure 6.5  Windows 2000 displays a summary of the options you selected for your printer.

NOTE:  Test pages are useful troubleshooting tools. Even though they appear simple, they prove that you can print graphics and text. The text they print is also useful because it contains the list of files supporting the printer and their versions. It’s a good idea to print a test page after installing a new printer and then to keep the page on file in case of problems later.
9.  The test page is sent to the printer; it should print in a few minutes. Then you are asked if it printed correctly. If it didn’t print correctly, click No, and you will be shown some troubleshooting information containing questions and answers. The most likely fixes for the malady will be described. If the page printed properly, click Yes, and you’re finished.

NOTE:  If the print server recognizes the printer and has a driver already installed for your Windows 2000 computer, then steps 5 through 9 above will be skipped and the printer will be created automatically. These last steps will only be necessary if your Windows 2000 computer cannot detect what kind of printer is being installed.

If all went well, you now have a network printer set up and ready to go. The new printer appears in your Printers folder, as shown in the following illustration.
