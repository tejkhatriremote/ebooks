Windows 2000’s print feature mix is quite rich. Here are the highlights:

•  You can add, modify, and remove printers right from Printers folder (available from My Computer, Explorer, the Start button, or Control Panel).
•  You can set up multiple printers for one physical print device, or vice versa. This enables you to set different schedules, priorities, and permissions for the device.
•  You don’t need to think about the relationship of printer drivers, connections, and physical printers because an object-oriented interface using printer icons eliminates that abstraction. You can simply add a printer and set its properties. Once added, it appears as a named printer in the Printers folder.
•  After you set up a printer, you can easily choose to share a printer on the network so others can print to it. You can give the device a useful name, such as LaserJet in Fred’s Office, so people on the network know what it is.
•  When sharing a printer on the network, you can choose to also install one or more Windows 2000 printer drivers for other Windows 2000 platforms, for earlier versions of NT, and/or for Windows 9x stations. These drivers are then provided “automagically” to other printer users, sparing them the task of loading their own printer driver manually (and ensuring that the correct printer driver will always be used by the others accessing this printer). If you’re on a network, you can manage network-printer connections by displaying available printers, sharing your local printer, and connecting to and disconnecting from network printers.
•  When sharing a printer on the network, you can conduct printer-access auditing, set ownership, set which time periods the printer is available for use on the network, and restrict who has access to the printer.
•  You can start printing and immediately go back to work; you don’t need to wait until spooling for Print Manager is done. While one document is printing, other programs can start print jobs. Additional documents are simply added to the queue and print in turn. Windows 2000’s multithreading and preemptive multitasking make 32-bit Windows programs print more smoothly than under Windows 3.x.
•  You don’t need to manually set options such as number of copies, paper tray, page orientation, and so forth. Defaults are automatically used during print jobs.
•  To quell your curiosity, you can open a window displaying jobs being printed or jobs in the queue waiting to be printed, along with an indication of the current print job’s progress.
•  You can easily rearrange the order of the print queue and delete print jobs.
•  You can choose whether printing begins as soon as the first page is spooled to the hard disk or after the last page of a document is spooled.
•  You can temporarily pause or resume printing without causing printer time-out problems.

In fact, if you’re interested in nothing more than printing from one of your programs without viewing the queue, printing to a network printer, or adjusting your current printer’s settings, skip to the “Printing from a Program” section. About Printer Installation

Before installing hardware, including printers, you should read any last-minute printed or on-screen material that comes with Windows 2000. Often, such material is full of useful information about specific types of hardware, including printers. Because these are last-minute details, we can’t include them in this book. Look for a readme file (or something similar) on your Desktop that you can open by double-clicking. Then look through the file for information about your printer.

With that said, here is the overall game plan for adding a new printer. It’s an easy process thanks to the Add Printer Wizard that walks you through it.

1.  Choose Add Printer from the Printers folder.
2.  Specify whether the printer is local (directly connected to your computer) or on the network.
3.  Specify what kind of printer it is.
4.  Select the printer’s port and relevant port settings.
5.  Give the printer a name.
6.  Print a test page.
7.  Check and possibly alter the default printer settings, such as the dots per inch (dpi) and memory settings.

After these steps are complete, your printer should work as expected. After a printer is installed, you can customize its setup by modifying its properties as shown in Figure 6.1; for example, you can

•  Specify the amount of time you want Windows to keep trying to print a document before alerting you to a printer problem.
•  Specify the share name for the printer so other network users can find it when they search the network for printers.
•  Set job defaults pertaining to paper tray, two-sided printing, and paper orientation.
•  Stipulate a separator file (a file, usually one page long, that prints between each print job).
•  Select the default printer if you have more than one printer installed.
•  Choose whether your printer should substitute its own fonts for certain Windows TrueType fonts.
•  Select printer settings relevant to page orientation, color matching, grayscaling, size scaling, type of paper feed, halftone imaging, and when file header information (such as a PostScript preamble) is sent to the printer.
•  Set whether you want to share the printer for use by others on the network.
•  Set whether documents will go directly to the printer or will go through the spooler.


Figure 6.1  The Device Settings tab enables you to fine tune the options of your printer

The good news is that normally you won’t need to fuss with any of these settings. The other good news is that, like the Windows NT 4 interface, with Windows 2000, getting to any one of these settings is a piece of cake. You don’t need to wind your way through a bevy of dialog boxes to target a given setting. The Properties dialog box has tabs, so it’s a cinch to find the one you want. We cover these options in the “Customizing Your Printer” section.
