Idle Sensitivity

Idle Sensitivity affects the length of time that a DOS application must be idle before some of its CPU cycles are allocated elsewhere. The higher the setting, the sooner an application will lose CPU cycles, although it will not lose all of them unless Always Suspend is checked.

Adjusting this setting can make DOS multitasking smoother. A simple multitasking system would perform basic time slicing, a process whereby X percent of the time goes to program 1, Y percent to program 2, and so on. But we can do better than that. It turns out that DOS programs tend to have one thing in common: They’re interactive. As we said a minute ago, an application spends a lot of time waiting for input.

So if three programs are running and two are basically waiting for input, then Windows 2000 should give all the CPU time to the one program that isn’t waiting for input. The only problem is figuring out which programs are waiting for keyboard input. This process is not an exact science, so Windows 2000 has some rules of thumb that it uses to guess which programs are waiting for input and which aren’t. Try fiddling with these settings to help Windows 2000 use its rules. Depending on where you set the slider, Windows 2000 will temporarily shut down any program needing input, automatically reactivating that program when you next give it a keystroke.

The downside (you knew there’d be one) is that it takes some time for Windows 2000 Professional to monitor a program for inactivity, and that time may really slow down a program. So adjust each program’s PIF to the length of idle time that you’re willing to allow, but if the system seems slow, move the slider down so that more idle time is permitted. (Moving a slider down to get more of something seems counterintuitive, but then, do you turn air conditioning up or down to get colder air?) Idle sensitivity is just one of a number of PIF options that really requires some experimentation to get just right.

Here are some programs for which you should probably set the idle sensitivity as low as possible:

•  Background communications
•  Background printing
•  3270 emulation

Once again, if you find that an application is not getting enough resources to work in the background, adjust the setting downward.

Mouse

You can adjust how the mouse works with DOS applications in two ways:

•  Permit the user to select text for copying and cutting without using the Mark Text tool in the DOS toolbar (which is a viable option only if you turn on the toolbar under the Appearance properties page).
•  Restrict the mouse to just the active DOS application.

Activating QuickEdit means that you can select data within the DOS window without needing to mark it. From there, you can copy the data to another application or to Notepad. You can still copy data from an application without QuickEdit activated, but to do so, you must first click the Mark button (a rectangle with a dotted border) so you can select the data to be copied. QuickEdit saves you a step.

If you set the mouse to Exclusive Mode, you can get the mouse back for Windows 2000 use by pressing Alt+Tab or Alt+Esc to cycle to another application. However, you can use the mouse only while in that active application. Move the mouse from the window and the cursor disappears.

Termination

Warn If Still Active alerts you that a DOS program is still running if you attempt to close a DOS window without first exiting the program running in that window. Try it, and you’ll see the following warning.

You might as well keep this option enabled (it is by default) because Windows 2000 Professional won’t prevent you from closing the window without exiting first, but only warns you that doing so could cause problems.

Other

You know that Windows 2000 didn’t know how to categorize something when you see it in the Other box in the Misc tab. Fast Pasting refers to how data can be pasted from the Clipboard. With this option enabled, if you have data in the Clipboard, you can paste it into your application just as if it were an NT app—maybe.

If you select the Fast Pasting check box, the application will paste information as quickly as Windows 2000 can transfer the data. Because some applications cannot paste properly using this method, test it before relying on it—try to paste information from the Clipboard to the application. If the paste is successful, you can use the option.

Windows Shortcut Keys

If you uncheck any of these shortcut key combinations, then you’re telling Windows 2000 to ignore them and reserve them for the use of that particular DOS application. You can’t use this dialog box to assign new shortcut keys to an application. This dialog box is for helping you get full functionality from programs designed before Windows took over some key combinations.

Managing DOS Memory under Windows 2000

DOS memory management under Windows 2000 Professional isn’t very different from DOS memory management under DOS. Keep the number of programs loaded in conventional memory to a minimum, load DOS high, and that should prevent many memory problems.

Maximizing Conventional Memory

DOS can see only the lowest 640K of memory, so if you’re running a DOS program you’re going to want to squeeze all the good from that 640K that you can. And that means that you want as much of that memory free as possible.

Load Fewer Programs (TSRs and Device Drivers)

The first step in maximizing conventional memory has been done for you in Windows 2000 Professional. Rather than relying on external device drivers (like the one controlling your mouse) that must use part of the lowest 640K, Windows 2000 includes 32-bit device drivers in the operating system that can reside in memory above 640K, thus freeing up conventional memory. You can follow in that tradition by not loading TSRs unless you really need them.

Load DOS High

By default, another step for maximizing conventional memory has already been done for you as well. If you turn to the Program tab in a DOS program’s Properties dialog box, you’ll notice that, by default, DOS is loaded high so that most of it isn’t cluttering up conventional memory.
